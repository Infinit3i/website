## Overview

**Sudoking** is an Easy **Coding** challenge — an online judge rather than a hacking target.
The page is a Flask app with a Monaco editor; you `POST` a program to `/run`, the server runs
it against a Sudoku puzzle by feeding the puzzle to your program's **stdin** and comparing its
**stdout** to the expected solution, and a correct run returns the flag.

## The technique

The *Run* button shows the protocol:

```js
fetch("/run", { method:"POST",
  body: JSON.stringify({ code: editorValue, language: "python" }) })
  .then(r => r.json())
  .then(data => { /* data.input, data.result, data.expected, data.stderr, data.flag */ });
```

**Step 1 — learn the exact format.** Submit a passthrough program so the response reveals the
real input/output layout:

```python
import sys
print(sys.stdin.read())
```

The reply shows the grid is drawn with box separators (`.` = empty), and `expected` is the
same layout fully solved:

```
+-------+-------+-------+
| . . . | . . 6 | . . . |
| . 6 . | 9 . . | . . . |
...
+-------+-------+-------+
```

So the program must print the solved grid **byte-for-byte** in that format.

## Solution

Submit a backtracking solver. The key robustness trick is to parse the grid by **filtering
the cell charset** (`123456789.`) out of the whole stdin — that ignores the pipes and
separators automatically — then re-emit the exact box layout.

Create `sudoku_solver.py` (this is the `code` we submit):

```python
import sys
data = sys.stdin.read()
cells = [c for c in data if c in '123456789.']
g = [[0]*9 for _ in range(9)]
for i, c in enumerate(cells[:81]):
    g[i//9][i%9] = 0 if c == '.' else int(c)

def ok(r, c, v):
    for i in range(9):
        if g[r][i] == v or g[i][c] == v: return False
    br, bc = 3*(r//3), 3*(c//3)
    for i in range(br, br+3):
        for j in range(bc, bc+3):
            if g[i][j] == v: return False
    return True

def solve():
    for r in range(9):
        for c in range(9):
            if g[r][c] == 0:
                for v in range(1, 10):
                    if ok(r, c, v):
                        g[r][c] = v
                        if solve(): return True
                        g[r][c] = 0
                return False
    return True

solve()
sep = '+-------+-------+-------+'
out = [sep]
for r in range(9):
    out.append('| ' + ' | '.join(' '.join(str(g[r][c]) for c in range(b*3, b*3+3)) for b in range(3)) + ' |')
    if r % 3 == 2: out.append(sep)
print('\n'.join(out))
```

POST it to `/run` and read the flag from the JSON response:

```python
import json, urllib.request
URL = 'http://<host>:<port>/run'
code = open('sudoku_solver.py').read()
req = urllib.request.Request(URL, data=json.dumps({'code': code, 'language': 'python'}).encode(),
                             headers={'Content-Type': 'application/json'})
print(json.load(urllib.request.urlopen(req))['flag'])   # HTB{...}
```

## Why it worked

The only real steps are recognizing the Flask + Monaco `/run` judge (so you submit a
*program*, not a single answer), and emitting output that matches the expected layout exactly.
Reading the grid by charset-filter rather than table-parsing makes the input handling immune
to the separator spacing.

## Takeaway

For any HTB *Coding* challenge: `curl` the root; if it's Werkzeug + Monaco, probe `/run` with
a passthrough program to capture the exact I/O format, then submit a solver and read
`data.flag` (some variants instead put the flag in `data.stdout` or gate it behind
`data.challengeCompleted`).
