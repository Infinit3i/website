## Overview

MSS is an easy HackTheBox **Crypto** challenge. A networked server protects a 256-bit AES key with a Shamir Secret Sharing scheme built on a degree-30 polynomial, but only ever issues **19** shares — fewer than the 31 points classical interpolation requires. The catch the title hints at ("really small threshold") is a trap: the polynomial is computed **over the integers, with no modular reduction**, which turns every share into a clean congruence on the secret. Nineteen shares taken at prime points, combined with the **Chinese Remainder Theorem**, recover the key and decrypt the flag.

## The technique

Real Shamir Secret Sharing lives in a finite field `GF(p)`: shares are `y = f(x) mod p`. Here the server's polynomial is evaluated in plain `Z`:

```python
self.key    = bytes_to_long(os.urandom(BITS//8))                 # 256-bit secret
self.coeffs = [self.key] + [random 256-bit for _ in range(d)]    # d = 30  ->  31 coeffs
def poly(self, x):
    return sum(self.coeffs[i] * x**i for i in range(self.d+1))    # over Z, NOT mod p
```

Because there is no modulus, every term `c_i * x^i` for `i >= 1` is a multiple of `x`. Therefore:

```
y(x) mod x  ==  c0 mod x  ==  key mod x
```

A single share at `x` directly leaks `key mod x`, completely independent of the other 30 unknown coefficients. The polynomial's high degree — the whole point of a threshold scheme — becomes irrelevant. This is a missing-required-cryptographic-step weakness, [CWE-325](https://cwe.mitre.org/data/definitions/325.html): omitting the prime-field reduction that the threshold security depends on.

## Solution

The server caps `x` at `2**15` and allows 19 `get_share` calls. Pick the 19 largest **primes** `<= 2**15`, collect `key mod p` from each, and CRT them together. Nineteen ~15-bit primes give ~285 bits of combined modulus — comfortably larger than the 256-bit key — so the result is pinned uniquely. The AES key is `sha256(str(key))`, which decrypts the flag.

Create `solve.py`:

```python
import sys, json
from hashlib import sha256
from sympy import prevprime
from sympy.ntheory.modular import crt
from Crypto.Cipher import AES
from pwn import remote, context

context.log_level = 'error'
HOST, PORT = sys.argv[1], int(sys.argv[2])

primes, p = [], 2**15 + 1
while len(primes) < 19:
    p = prevprime(p)
    primes.append(p)

io = remote(HOST, PORT)

def send(obj):
    io.recvuntil(b'query = ')
    io.sendline(json.dumps(obj).encode())

send({'command': 'encrypt_flag'})
line = io.recvline_contains(b'encrypted flag').decode()
enc = json.loads(line.split('encrypted flag : ', 1)[1].rsplit('.', 1)[0])
iv = bytes.fromhex(enc['iv']); ct = bytes.fromhex(enc['enc_flag'])

residues = []
for pr in primes:
    send({'command': 'get_share', 'x': pr})
    share = json.loads(io.recvline_contains(b'"y"').decode())
    residues.append(share['y'] % pr)

key, _ = crt(primes, residues)
aes_key = sha256(str(int(key)).encode()).digest()
flag = AES.new(aes_key, AES.MODE_CBC, iv).decrypt(ct)
print("FLAG:", flag.rstrip(b'\x00').decode(errors='replace'))
```


## Why it worked

The threshold (fewer shares than `degree + 1`) is only a real defense **inside a finite field**, where reducing mod `p` destroys the "divisible by `x`" structure and forces full Lagrange interpolation. Over the integers, each evaluation is just the congruence `key ≡ y (mod x)`, so the secret leaks one prime modulus at a time and the Chinese Remainder Theorem reassembles it — the small threshold provides no protection at all.

## Fix / defense

- **Do Shamir in a finite field.** Choose a prime `p` larger than the secret and compute `poly(x) mod p`. With the reduction in place, `y mod x` no longer equals `key mod x`, and fewer than `degree + 1` shares reveal nothing.
- **Authenticate shares** with a MAC or hash commitment before reconstruction.
- Never roll your own threshold cryptography over `int`; use a vetted library that operates in a prime field or `GF(2^n)`.
