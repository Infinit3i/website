## Overview

`Traces` is an easy HackTheBox **Crypto** challenge. You're given `server.py` for a fantasy
IRC server ("EldoriaNet") and a live instance. Every message is encrypted with AES in CTR mode,
but the counter is re-initialised to zero on *every* call while the key stays fixed for the
connection — so all messages reuse the **same keystream**. That is a classic
[keystream-reuse / many-time pad](https://cwe.mitre.org/data/definitions/323.html) mistake, and
the whole solve is recovering that keystream with known-plaintext crib-dragging.

## The technique

CTR mode turns a block cipher into a stream cipher: it generates a keystream
`KS = AES(key,0), AES(key,1), …` and XORs it with the plaintext. The server does this:

```python
def __init__(self, host, port):
    self.key = os.urandom(32)            # fixed for the whole connection
def encrypt(self, msg):
    return AES.new(self.key, AES.MODE_CTR,
                   counter=Counter.new(128)).encrypt(msg)   # counter ALWAYS starts at 0
```

`Counter.new(128)` restarts the counter at `0` on every `encrypt()` call, and the key never
changes during a connection. So **every message uses the identical keystream**:

```
ct = pt XOR KS          (same KS for every message)
ct_a XOR ct_b = pt_a XOR pt_b      (the keystream cancels)
KS[i] = ct[i] XOR pt[i]            (any known plaintext leaks the keystream)
```

With one pad reused across many messages, any *crib* (a guessed/known piece of plaintext)
directly hands you the keystream at those byte positions, which then decrypts every other
message there.

## Solution

1. **`JOIN #general`** — the server dumps all stored messages as hex. The first three are
   `!nick Doomfang` / `!nick Stormbane` / `!nick Runeblight` and the trailing three are
   `!leave`. These are *known plaintext* → instantly recover `KS[0:16]`.
2. **Crib-drag the chat.** The conversation is English; reconstructing each readable sentence
   (e.g. `"This channel is not safe for long talks. Let's switch to our private room."`) extends
   the keystream. One message reveals:
   `Here is the passphrase for our secure channel: <passphrase>`
3. **`JOIN #secret <passphrase>`** in the *same* connection (same keystream). The flag lives in
   the message starting `"Yes, but we must treat it only as a last resort…"`.
4. **Pin a parallel sentence over the flag's columns.** The flag sits at high byte offsets where
   few messages overlap, so the automatic heuristic is unreliable there. Use one fully-readable
   `#secret` sentence as a crib to nail the keystream across exactly those columns, then
   `flag = ct XOR KS`.
5. **Verify by parallel coherence.** Set `KS = flag_ct XOR flag_guess` and decrypt the *other*
   `#secret` messages across the same columns — because `ct_a XOR ct_b` is exact, a correct flag
   makes every parallel message read as clean English (`"…doom our entire campaign…"`,
   `"…lose access to their strongholds…"`). This is what catches a wrong guess: an off-by-one
   case bit or a missed punctuation char shows up as a garbled/case-flipped parallel.

The working solver — connects, recovers the keystream from the cribs, reads the passphrase,
joins `#secret`, and prints the flag live:

```python
#!/usr/bin/env python3
import socket, re, sys
HOST, PORT = sys.argv[1], int(sys.argv[2])
SECRET_KEY = "<recovered-passphrase>"

def recv_idle(s, t=3.0):
    s.settimeout(t); d = b''
    try:
        while True:
            c = s.recv(4096)
            if not c: break
            d += c
    except socket.timeout:
        pass
    return d

def cts_of(raw):
    clean = re.sub(rb'\x1b\[[0-9;]*m', b'', raw).decode(errors='replace')
    return [bytes.fromhex(h) for n, h in re.findall(r'<([^>]+)> : ([0-9a-f]+)', clean)]

CRIBS = [
    "Understood. Has there been any sign of them regrouping since our last XXXX?",
    "This channel is not safe for long talks. Let's switch to our private room.",
    "If everything is clear, we move to the next stage. Our goal is within reach.",
    "nt out signals to an unknown beacon-one that none of us authorized.",
]

def recover_ks(general, secret):
    allc = general + secret
    ks = bytearray(max(len(c) for c in allc))
    def crib(ct, pt, off=0):
        for i in range(min(len(ct) - off, len(pt))):
            ks[off + i] = ct[off + i] ^ pt[i]
    for ct in allc:
        for nm in (b'Doomfang', b'Stormbane', b'Runeblight'):
            if len(b'!nick ' + nm) == len(ct):
                crib(ct, b'!nick ' + nm)
    crib(general[4], CRIBS[0].encode()); ks[74] = general[4][74] ^ ord('?')
    crib(general[6], CRIBS[1].encode())
    crib(general[13], CRIBS[2].encode())
    if secret:
        crib(secret[5], CRIBS[3].encode(), off=127)
    for p in range(len(ks)):
        if ks[p]:
            continue
        col = [c[p] for c in allc if len(c) > p]
        if len(col) < 2:
            continue
        best, bs = 0, -1e9
        for c in col:
            cand = c ^ 0x20
            s = sum(1 if (x ^ cand) == 32 or 65 <= (x ^ cand) <= 90 or 97 <= (x ^ cand) <= 122 else -1 for x in col)
            if s > bs:
                bs, best = s, cand
        ks[p] = best
    return bytes(ks)

s = socket.socket(); s.connect((HOST, PORT)); recv_idle(s, 2)
s.sendall(b'JOIN #general\n'); gen = cts_of(recv_idle(s, 3))
ks0 = recover_ks(gen, [])
print("[*] passphrase line:", bytes(gen[7][i] ^ ks0[i] for i in range(len(gen[7]))).decode(errors='replace'))
s.sendall(b'!nick guest\n'); recv_idle(s, 1.5)
s.sendall(b'!leave\n');       recv_idle(s, 1.5)
s.sendall(f'JOIN #secret {SECRET_KEY}\n'.encode()); sec = cts_of(recv_idle(s, 3))
s.close()

ks = recover_ks(gen, sec)
for ct in sec:
    pt = bytes(c ^ ks[i] for i, c in enumerate(ct)).decode(errors='replace')
    m = re.search(r'HTB\{.*?\}', pt)
    if m:
        print("[+] FLAG:", m.group(0))
```

```
[+] FLAG: HTB{...}
```

## Why it worked

The cipher itself (AES) is fine — the bug is *operational*: a stream cipher's security depends on
**never reusing a (key, nonce) pair**, and `Counter.new(128)` silently reset the counter to zero
for every message under one fixed key. That collapses AES-CTR into a one-time pad reused dozens of
times, and a reused pad leaks to anyone with a sliver of known plaintext — here, the fixed IRC
protocol strings and the readable chat. No key recovery, factoring, or oracle is needed: just XOR.

## Fix / defense

Use a **unique, random nonce per message** and send it alongside the ciphertext, or use an AEAD
mode that binds the nonce and authenticates the ciphertext:

```python
import os
nonce = os.urandom(12)
ct = nonce + AES.new(key, AES.MODE_GCM, nonce=nonce).encrypt(msg)
```

Never let the counter/nonce repeat under a given key, and prefer AES-GCM-SIV if accidental nonce
reuse is a realistic risk — it degrades gracefully instead of leaking the whole keystream.
