## Overview

Anchored is an HTB Mobile challenge (Easy) featuring an Android APK that POSTs a secret token to a server alongside the user's email. The token is generated entirely inside a native JNI library (`libanchored.so`), which is never visible to the Java decompiler. The technique: extract the x86_64 `.so`, identify the XOR loop in the JNI export, and emulate the function offline with the `unicorn` CPU emulator — no device, no Frida, no certificate pinning bypass required.

---

## The technique

[CWE-321 (Use of Hard-coded Cryptographic Key)](https://cwe.mitre.org/data/definitions/321.html) in a native Android library, obfuscated with XOR. Both the cipher bytes and the XOR key ship together as `uint16_t` wide-char pairs in `.rodata` — the obfuscation provides zero security because both operands are static and co-located. The core insight is that an x86_64 Android `.so` can be emulated directly on a Linux x86_64 host using `unicorn`, bypassing any Android-specific shared library dependencies entirely.

## Solution

### 1. Decompile the APK

```bash
jadx -d decompiled Anchored.apk
```

`MainActivity.java` exposes the shape of the request:

```java
hashMap.put(MainActivity.this.mrm(), MainActivity.this.frf());
// mrm() = parameter name
// frf() = the secret value — both declared `native`
```

Both methods are `native` — the Java layer has no implementation. The body lives in `libanchored.so`.

### 2. Extract and inspect the native library

```bash
unzip -P hackthebox Anchored.apk 'lib/x86_64/*.so' -d apk_extracted
readelf -s apk_extracted/lib/x86_64/libanchored.so | grep 'Java_'
objdump -d -M intel apk_extracted/lib/x86_64/libanchored.so > disasm.txt
```

`Java_com_example_anchored_MainActivity_frf` is the JNI export of `frf()` — it lives at library offset `0xf2f0`. Inside it, a 22-iteration XOR loop at offsets `0xf530`/`0xf533` reads from two pointer tables in `.rodata`: cipher bytes at one offset, key bytes at another, both stored as `uint16_t` wide-char pairs. The result is passed directly to `NewStringUTF`, whose argument is the secret.

### 3. Emulate `frf()` with unicorn

`ctypes.CDLL` fails immediately on Linux because `libanchored.so` depends on `liblog.so` (Android-only). The solution is offline CPU emulation.

Key implementation decisions:

| Problem | Solution |
|---|---|
| ELF segments overlap when mapped separately | Map one contiguous region (`max_vaddr` rounded up); write each segment at `BASE + p_vaddr` |
| TLS `fs:0x28` stack canary | Map fake TLS page; write known canary; set `FS_BASE` to the TLS base |
| Android-only PLT imports crash | Parse `.rela.plt` + `.rela.dyn`; write `RET` (`0xc3`) stubs; patch each GOT entry |
| libc++ string constructor is a PLT import | Redirect its GOT entry to the library's own internal implementation at offset `0xf250` |
| `NewStringUTF` at vtable offset `0x538` | Build a fake `JNIEnv*` with a hookable stub at that vtable slot; pass it as `RDI` |

Create `solve.py`:

```python
#!/usr/bin/env python3
import unicorn as uc
from unicorn import x86_const as x86
import struct

LIB_PATH = "apk_extracted/lib/x86_64/libanchored.so"

with open(LIB_PATH, "rb") as f:
    lib_data = f.read()

def parse_elf(data):
    e_phoff = struct.unpack_from('<Q', data, 0x20)[0]
    e_phentsize = struct.unpack_from('<H', data, 0x36)[0]
    e_phnum = struct.unpack_from('<H', data, 0x38)[0]
    segs = []
    for i in range(e_phnum):
        ph = e_phoff + i * e_phentsize
        if struct.unpack_from('<I', data, ph)[0] == 1:
            segs.append((struct.unpack_from('<Q', data, ph+0x10)[0],
                         struct.unpack_from('<Q', data, ph+8)[0],
                         struct.unpack_from('<Q', data, ph+0x20)[0],
                         struct.unpack_from('<Q', data, ph+0x28)[0]))
    return segs

def find_relocs(data):
    e_shoff = struct.unpack_from('<Q', data, 0x28)[0]
    e_shentsize = struct.unpack_from('<H', data, 0x3a)[0]
    e_shnum = struct.unpack_from('<H', data, 0x3c)[0]
    e_shstrndx = struct.unpack_from('<H', data, 0x3e)[0]
    shstr_sh = e_shoff + e_shstrndx * e_shentsize
    shstr_off = struct.unpack_from('<Q', data, shstr_sh + 0x18)[0]
    strtab = data[shstr_off:shstr_off+0x10000]
    sections = {}
    for i in range(e_shnum):
        sh = e_shoff + i * e_shentsize
        name_idx = struct.unpack_from('<I', data, sh)[0]
        sections[strtab[name_idx:].split(b'\x00')[0].decode()] = sh
    relocs = {}
    for sec_name in ['.rela.plt', '.rela.dyn']:
        if sec_name not in sections: continue
        sh = sections[sec_name]
        sh_off = struct.unpack_from('<Q', data, sh + 0x18)[0]
        sh_sz  = struct.unpack_from('<Q', data, sh + 0x20)[0]
        sh_link = struct.unpack_from('<I', data, sh + 0x28)[0]
        sym_sh = e_shoff + sh_link * e_shentsize
        sym_off = struct.unpack_from('<Q', data, sym_sh + 0x18)[0]
        sym_ent = struct.unpack_from('<Q', data, sym_sh + 0x38)[0]
        sym_stridx = struct.unpack_from('<I', data, sym_sh + 0x28)[0]
        sstr_sh = e_shoff + sym_stridx * e_shentsize
        sstr_off = struct.unpack_from('<Q', data, sstr_sh + 0x18)[0]
        sstr = data[sstr_off:sstr_off+0x100000]
        for j in range(sh_sz // 24):
            r_off = struct.unpack_from('<Q', data, sh_off + j*24)[0]
            r_info = struct.unpack_from('<Q', data, sh_off + j*24 + 8)[0]
            sym_idx = r_info >> 32
            if sym_idx > 0:
                st_name = struct.unpack_from('<I', data, sym_off + sym_idx * sym_ent)[0]
                sym_name = sstr[st_name:].split(b'\x00')[0].decode()
                relocs[sym_name] = r_off
    return relocs

segments = parse_elf(lib_data)
relocs = find_relocs(lib_data)

BASE      = 0x01000000
MEM_STUB  = 0x05000000
MEM_TLS   = 0x06000000
MEM_HEAP  = 0x07000000
MEM_STACK = 0x08000000
MEM_HALT  = 0x09000000
MEM_JNI   = 0x0a000000

max_vaddr = max(v + m for v,o,f,m in segments)
lib_size = ((max_vaddr + 0xfff) & ~0xfff) + 0x1000

mu = uc.Uc(uc.UC_ARCH_X86, uc.UC_MODE_64)
for addr, size in [(BASE, lib_size + 0x10000), (MEM_STUB, 0x1000000),
                   (MEM_TLS, 0x10000), (MEM_HEAP, 0x1000000),
                   (MEM_STACK, 0x400000), (MEM_HALT, 0x1000), (MEM_JNI, 0x10000)]:
    mu.mem_map(addr, size)

for p_vaddr, p_offset, p_filesz, _ in segments:
    mu.mem_write(BASE + p_vaddr, lib_data[p_offset:p_offset+p_filesz])

mu.mem_write(MEM_HALT, bytes([0xf4]))
mu.mem_write(MEM_TLS + 0x28, struct.pack('<Q', 0xdeadbeefcafebabe))
mu.reg_write(x86.UC_X86_REG_FS_BASE, MEM_TLS)

heap_ptr = [MEM_HEAP]
def my_alloc(sz): p = heap_ptr[0]; heap_ptr[0] += (max(sz,1)+0xf)&~0xf; return p

stub_ptr = MEM_STUB
stub_addrs = {}
for name, r_offset in relocs.items():
    stub_addrs[name] = stub_ptr
    mu.mem_write(stub_ptr, bytes([0xc3]))
    mu.mem_write(BASE + r_offset, struct.pack('<Q', stub_ptr))
    stub_ptr += 0x10

str_ctor = '_ZNSt6__ndk112basic_stringIcNS_11char_traitsIcEENS_9allocatorIcEEEC2IDnEEPKc'
if str_ctor in relocs:
    mu.mem_write(BASE + relocs[str_ctor], struct.pack('<Q', BASE + 0xf250))

JNIENV_VTABLE = MEM_JNI + 0x1000
NSU_STUB = stub_ptr; mu.mem_write(stub_ptr, bytes([0xc3])); stub_ptr += 0x10
mu.mem_write(JNIENV_VTABLE + 0x538, struct.pack('<Q', NSU_STUB))
mu.mem_write(MEM_JNI, struct.pack('<Q', JNIENV_VTABLE))

RSP = MEM_STACK + 0x3ff000
mu.reg_write(x86.UC_X86_REG_RSP, RSP - 8)
mu.mem_write(RSP - 8, struct.pack('<Q', MEM_HALT))
mu.reg_write(x86.UC_X86_REG_RDI, MEM_JNI)
mu.reg_write(x86.UC_X86_REG_RSI, 0)

XOR_OFFSET = 0xf530
xor_trace = []
nsu_result = [None]

def hook(mu2, addr, size, ud):
    off = addr - BASE
    if off == XOR_OFFSET:
        rdx = mu2.reg_read(x86.UC_X86_REG_RDX)
        try: b = bytes(mu2.mem_read(rdx, 1))[0]
        except: b = 0
        xor_trace.append([b, None])
    if off == XOR_OFFSET + 3:
        rcx = mu2.reg_read(x86.UC_X86_REG_RCX)
        try: b = bytes(mu2.mem_read(rcx, 1))[0]
        except: b = 0
        if xor_trace: xor_trace[-1][1] = b
    if addr == NSU_STUB:
        rsi = mu2.reg_read(x86.UC_X86_REG_RSI)
        try: nsu_result[0] = bytes(mu2.mem_read(rsi, 100)).split(b'\x00')[0]
        except: pass
    if addr in [stub_addrs.get('malloc'), stub_addrs.get('_Znwm')]:
        mu2.reg_write(x86.UC_X86_REG_RAX, my_alloc(mu2.reg_read(x86.UC_X86_REG_RDI)))
    if addr == stub_addrs.get('strlen'):
        rdi = mu2.reg_read(x86.UC_X86_REG_RDI); n = 0
        try:
            while bytes(mu2.mem_read(rdi + n, 1))[0]: n += 1
        except: pass
        mu2.reg_write(x86.UC_X86_REG_RAX, n)
    if addr == stub_addrs.get('memcpy'):
        dst = mu2.reg_read(x86.UC_X86_REG_RDI)
        src = mu2.reg_read(x86.UC_X86_REG_RSI)
        n = mu2.reg_read(x86.UC_X86_REG_RDX)
        if 0 < n < 0x100000:
            try: mu2.mem_write(dst, bytes(mu2.mem_read(src, n)))
            except: pass
        mu2.reg_write(x86.UC_X86_REG_RAX, dst)

mu.hook_add(uc.UC_HOOK_CODE, hook)
try:
    mu.emu_start(BASE + 0xf2f0, MEM_HALT, timeout=10000000, count=500000)
except: pass

if nsu_result[0]:
    secret = nsu_result[0].decode('latin1')
else:
    secret = bytes(a ^ b for a, b in xor_trace if b is not None).decode('latin1')

print(f"HTB{{{secret}}}")
```

Run it:

```bash
python3 solve.py
```

Output: `HTB{...}`

## Why it worked

The secret is a static 22-byte string embedded in `libanchored.so`'s `.rodata` section. A simple XOR loop with a co-located key is the only protection — since both the ciphertext and the key bytes ship inside the same binary, there is no cryptographic security at all. This is a classic [hardcoded credential](https://cwe.mitre.org/data/definitions/798.html) pattern dressed up with trivial obfuscation.

The Android-native delivery makes it slightly harder to analyse interactively (no `ctypes` on Linux, no Frida without a rooted device), but an x86_64 `.so` built for the Android emulator target runs identically under `unicorn` on a Linux host — no Android framework needed. The emulator only has to survive 22 XOR iterations and one `NewStringUTF` call; the PLT stubs for everything else can safely be `RET` no-ops.

---

## Fix / defense

Never embed secrets in the binary, whether obfuscated or not. The correct architecture:

1. The secret lives server-side only.
2. The client authenticates the user (username + password + proper TLS with certificate pinning against both leaf and chain).
3. Only after authenticated does the server return the per-session token — over a connection the client verified.

XOR, base64, and similar reversible in-binary transforms buy nothing against an analyst who has the binary. Android Keystore can protect cryptographic keys from extraction via the TEE, but even that is not a substitute for keeping secrets off the device entirely.
