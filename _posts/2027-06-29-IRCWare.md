## Overview

IRCWare is an HTB Reversing challenge (Easy) that gives you a stripped x86-64 ELF binary acting as an IRC bot. The bot connects to `127.0.0.1:8000`, joins `#secret`, and exposes three commands — `@pass`, `@flag`, and `@exec` — where `@flag` is gated behind a correct password. The password is stored in the binary's `.data` section as its [ROT17-encoded](https://cwe.mitre.org/data/definitions/261.html) form; recovering it requires `strings`, a quick disassembly pass to identify the shift constant, inverting the cipher with one line of Python, and then running a fake IRC server to deliver the command and receive the flag.

**[CWE-261](https://cwe.mitre.org/data/definitions/261.html) — Weak Encoding for Password**

---

## The Technique

The binary uses a [weakly encoded password](https://cwe.mitre.org/data/definitions/261.html): it applies a ROT17 substitution cipher to each uppercase letter of user input before comparing it to a hardcoded encoded form stored in `.data`. ROT ciphers are completely reversible — the inverse of ROT17 over a 26-letter alphabet is ROT9 — so anyone with `strings` and a disassembler can recover the plaintext in under a minute.

The technique chain:

1. `strings` reveals the encoded blob and the IRC protocol strings that identify the binary as a bot.
2. Disassembly of the password comparison loop reveals the shift constant (`0x11` = 17).
3. ROT9 applied to the stored form recovers the plaintext password.
4. A Python fake IRC server sends the correct `@pass` command and reads the `@flag` response.

## Solution

### Step 1 — Identify the binary

```bash
strings ./ircware
```

Relevant output:

```
NICK ircware_0000
USER ircware 0 * :ircware
JOIN #secret
PRIVMSG #secret :@exec
PRIVMSG #secret :@flag
RJJ3DSCP
PRIVMSG #secret :@pass
Accepted
Rejected
Requires password
```

`RJJ3DSCP` stands out — 8 uppercase characters with no obvious meaning. The IRC strings confirm the binary connects to a server and waits for commands in `#secret`.

### Step 2 — Find the connect target

```bash
strace -e trace=network ./ircware 2>&1 | head
```

```
connect(3, {sa_family=AF_INET, sin_port=htons(8000), sin_addr=inet_addr("127.0.0.1")}, 16)
```

The bot connects to `127.0.0.1:8000`.

### Step 3 — Disassemble the password comparison loop

```bash
objdump -d ./ircware | less
```

The comparison loop at `0x400401` processes each character of the user-supplied password:

```
; for each char c of input:
cmp $0x41, %al       ; c < 'A'?  → pass through unchanged
cmp $0x5a, %al       ; c > 'Z'?  → pass through unchanged
add $0x11, %al       ; uppercase: c += 17
cmp $0x5a, %al       ; result > 'Z'?
jbe done             ;   no  → keep result
sub $0x5a, %al       ;   yes → result -= 90
add $0x40, %al       ;         result += 64  (wraps back into A–Z)
cmp %al, STORED[i]   ; compare transformed char to stored form
```

This is ROT17 over uppercase A–Z (`A→R`, `B→S`, … `I→Z`, `J→A`, … `Z→Q`).

### Step 4 — Invert: ROT9 recovers the plaintext

```bash
python3 -c "enc='RJJ3DSCP'; n=17; print(''.join(chr((ord(c)-65+(26-n))%26+65) if c.isupper() else c for c in enc))"
```

```
ASS3MBLY
```

### Step 5 — Patch binary port and run a fake IRC server

Port 8000 was occupied on the test machine, so the binary was patched to connect to port 9000 instead (the `pushw $0x401f` instruction at offset `0x2bd` was rewritten to `pushw $0x2823`). The Python fake server handles the full IRC handshake:

Create `solve.py`:

```python
import socket, threading, time, subprocess, re

FLAG = None

def fake_irc_server():
    global FLAG
    srv = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    srv.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
    srv.bind(('127.0.0.1', 9000))
    srv.listen(1)
    conn, _ = srv.accept()
    buf = b""
    sent_pass = False

    while True:
        data = conn.recv(4096)
        if not data:
            break
        buf += data
        while b"\n" in buf:
            line, buf = buf.split(b"\n", 1)
            line = line.rstrip(b"\r").decode("utf-8", errors="replace")

            if line.startswith("PING "):
                conn.send(("PONG " + line[5:] + "\r\n").encode())

            if line.startswith("NICK") and not sent_pass:
                conn.send(b":irc.htb 001 ircware_0000 :Welcome\r\n")
                time.sleep(0.2)
                conn.send(b":h!h@127.0.0.1 PRIVMSG #secret :@pass ASS3MBLY\r\n")
                sent_pass = True

            if "Accepted" in line:
                conn.send(b":h!h@127.0.0.1 PRIVMSG #secret :@flag\r\n")

            m = re.search(r'HTB\{[^}]+\}', line)
            if m:
                FLAG = m.group(0)
                conn.close(); srv.close(); return
    conn.close(); srv.close()

binary = bytearray(open('./files/ircware', 'rb').read())
idx = binary.find(b'\x66\x68\x1f\x40')
binary[idx:idx+4] = b'\x66\x68\x23\x28'
open('./ircware_patched', 'wb').write(binary)

t = threading.Thread(target=fake_irc_server)
t.daemon = True
t.start()
time.sleep(0.3)
proc = subprocess.Popen(['./ircware_patched'], stdout=subprocess.PIPE, stderr=subprocess.PIPE)
t.join(timeout=10)
proc.terminate()
print(f"FLAG: {FLAG}")
```

## Why It Worked

The binary used a [weakly encoded password](https://cwe.mitre.org/data/definitions/261.html) ([CWE-261](https://cwe.mitre.org/data/definitions/261.html)): a ROT17 substitution cipher is a bijection — every output maps back to exactly one input — so storing the encoded form alongside the encoding algorithm (both inside the same binary) provides no security. An attacker who can read the binary can always recover the plaintext.

The same issue arises with XOR-encoded strings, Base64-wrapped passwords, or any transform whose key is embedded in the code: the "encoding" is effectively public.

---

## Fix / Defense

Never store credentials in a binary, even encoded. If a local authentication check is necessary, store a proper one-way hash (`bcrypt`, Argon2) with a per-user salt — even if the hash leaks, it cannot be reversed to the plaintext in polynomial time.

```python
import bcrypt

STORED_HASH = bcrypt.hashpw(b"ASS3MBLY", bcrypt.gensalt())

def check_password(user_input: bytes) -> bool:
    return bcrypt.checkpw(user_input, STORED_HASH)
```
