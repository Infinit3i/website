## Overview

AHS512 is an Easy crypto challenge. You connect over `nc`, receive the digest of a fixed message, and must submit a **different** message that hashes to the same value. The hash isn't plain SHA-512 — it's a homemade construction that mixes the bytes first. That mixing step is where it breaks: it loses entropy, which turns "find a SHA-512 collision" (infeasible) into "flip one bit" (trivial).

## The technique

The server's hash is:

```
digest = sha512( rotate( transpose(message) ) )
```

- **transpose** — a columnar transpose whose column count is driven by a *random* key: a divisor of `len(message)` in `[2, len-1]`. For the 20-byte target message, the key is randomly one of `{2, 4, 5, 10}`, re-rolled on every evaluation.
- **rotate** — applied to each byte: `R(b) = ((b >> 4) | (b << 3)) & 0xff`.

SHA-512 is collision-resistant, so the only way two messages collide is if `rotate(transpose(...))` produces **identical bytes** for both. That means the weakness has to be in `transpose` or `rotate`.

Trace `rotate` at the bit level. For a byte `b7 b6 b5 b4 b3 b2 b1 b0`:

- `b >> 4` drops `b7 b6 b5 b4` into output positions 3..0.
- `(b << 3) & 0xff` drops `b4 b3 b2 b1 b0` into output positions 7..3.

OR them together and the only place input **bit 7** ever lands is output **bit 3**, where it is OR'd with input **bit 0**:

```
output_bit3 = b7 | b0
```

If `b0 == 1` (the byte is odd), that output bit is already `1`, so toggling `b7` changes nothing:

```
R(b) == R(b ^ 0x80)   for every byte whose LSB is set
```

`rotate` maps 256 inputs onto just 128 outputs — it is exactly **2-to-1**, not injective. This is a [weak-hash](https://cwe.mitre.org/data/definitions/328.html) construction ([CWE-328](https://cwe.mitre.org/data/definitions/328.html)): a lossy custom transform feeding a strong hash makes a [second preimage](https://cwe.mitre.org/data/definitions/328.html) trivial.

## Solution

The target message is `pumpkin_spice_latte!` (20 bytes). Flip the high bit of any **odd** byte — e.g. `u` (`0x75`) → `0xf5`. The new message differs at exactly one position, but `rotate` equalizes that position, so the post-rotate bytes are byte-for-byte identical → identical SHA-512 → collision, with no work done against SHA-512 itself.

The single-byte change is also **invariant to the transpose key** — it differs at one position, and `rotate` makes that position equal no matter where the permutation moves it — so the forged message collides under *any* key. The server fixes the original's random key per connection but draws a fresh key for each submission, so we just resubmit the same forged hex until our draw matches (≈1/4 per send for keys `{2,4,5,10}`).

Create `solve.py`:

```python
import socket, sys, time

def main():
    host, port = sys.argv[1], int(sys.argv[2])
    orig = b"pumpkin_spice_latte!"
    m = bytearray(orig); m[1] ^= 0x80    # 'u' 0x75 -> 0xf5, both odd -> same rotate output
    m = bytes(m); assert m != orig
    payload = m.hex().encode()
    s = socket.create_connection((host, port), timeout=15)
    print(s.recv(8192).decode(errors="replace"), end="")
    for _ in range(200):                  # resubmit until our random key == the connection's key
        s.sendall(payload + b"\n")
        time.sleep(0.05)
        resp = s.recv(8192).decode(errors="replace")
        if "HTB{" in resp:
            print(resp); return
    print("no flag after 200 rounds")

if __name__ == "__main__":
    main()
```

Run it against the live instance:

```bash
python3 solve.py <host> <port>
```

Within a few sends the forged key matches and the server prints the flag (`HTB{...}`, redacted).

## Why it worked

SHA-512 is sound. The bug is the homemade `rotate` wrapper, which silently isn't a bijection (2-to-1). Treating `sha512(MIX(x))` as collision-resistant is only valid when `MIX` is injective. Once `MIX` collides, second preimages fall out of a single XOR and the strength of the downstream hash is irrelevant.

## Fix / defense

- Never roll a custom per-byte "mixing" step that isn't a bijection. If you must transform input, prove the transform is injective — otherwise you've manufactured collisions.
- Hash the raw message with a standard hash: `digest = sha512(message)`. No transpose, no rotate, no lossy domain map.

The general recipe applies to any `H(x) = StrongHash(custom_transform(x))`: tabulate the per-byte transform over all 256 inputs, group by output to find a colliding pair, then substitute that delta into one position of a known message for an instant second preimage.
