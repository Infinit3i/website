## Overview

Signals is an Easy HackTheBox **Hardware** challenge. You get a single download, `Signal.wav` — a 16-bit mono, 48 kHz, ~128-second audio clip described as a "strange signal from space" with "similarities to signals transmitted by the ISS." That hint is the whole solution: the ISS famously transmits **SSTV** (Slow-Scan Television), an analog mode that paints a still picture into audio tones. Decode the picture and the flag is written across it.

## The technique

SSTV encodes an image one scan line at a time as audio frequency changes (typically 1500–2300 Hz for pixel brightness). Every frame begins with a fixed **VIS header**: a 1900 Hz leader tone, a 1200 Hz break, then a string of calibration bits at 1100/1300 Hz that spell out a 7-bit **VIS code** identifying which SSTV mode was used. Decoders read that VIS code to auto-select the mode (Martin 1, Scottie 1, Robot 36, …).

The trick in this challenge: the transmitted VIS code is **non-standard** (it reads as `95`, which maps to no known mode). Standards-based decoders abort with "VIS unsupported" and every auto-detect fails. The picture itself is a perfectly normal 256-line frame, though — you just have to stop trusting the VIS byte and **force the mode** instead. Brute the handful of 256-line modes; whichever renders a readable picture wins. Here it's **Martin 1**.

## Solution

First, confirm it really is SSTV without any tooling — read the audio's own preamble tones. Sweeping a short FFT window across the first second shows the unmistakable `1900 → 1200 → 1500/2300 Hz` signature. Then decode, forcing the mode.

The decoder needs a `--mode` override (the standard `colaclanth/sstv` has none and dies on the bad VIS); [`jbirby/sstv-codec`](https://github.com/jbirby/sstv-codec) is a pure-Python decoder that accepts one.

Create `solve.py`:

```python
import sys, wave, subprocess
import numpy as np

WAV = sys.argv[1] if len(sys.argv) > 1 else "Signal.wav"

w = wave.open(WAV); rate = w.getframerate()
d = np.frombuffer(w.readframes(w.getnframes()), dtype=np.int16).astype(float)

def peak_freq(seg):
    seg = seg * np.hanning(len(seg))
    return np.fft.rfftfreq(len(seg), 1/rate)[np.argmax(np.abs(np.fft.rfft(seg)))]

ws = int(0.010 * rate)
head = [round(peak_freq(d[i:i+ws])) for i in range(0, int(0.4*rate), ws)]
assert any(abs(f-1900) < 50 for f in head), "no 1900 Hz SSTV leader"
print("[+] 1900 Hz leader present -> SSTV")

subprocess.run([sys.executable, "sstv_decode.py", WAV, "signal_decoded.png",
                "--mode", "martin1"], check=True)
print("[+] image -> signal_decoded.png ; flag is painted across the picture")
```

Run it against the WAV:

```bash
git clone https://github.com/jbirby/sstv-codec
cp sstv-codec/*.py .
python3 solve.py Signal.wav
```

The decoded `signal_decoded.png` renders the picture with mild colour-shift and slant artefacts (the 48 kHz capture rate isn't the mode's nominal rate), but the flag is painted across it in large leet text — convert to grayscale and crop the text band to read it unambiguously:

```
HTB{...}
```

## Why it worked

SSTV is just frequency-shift modulation of an image into the audio band; the picture is recoverable byte-for-byte as long as you decode with the right line timing — i.e. the right mode. The challenge weaponises the one piece of the protocol you'd normally automate: the VIS mode-identifier. By transmitting a bogus VIS code, it breaks lazy "feed it to the decoder and let it auto-detect" solves, forcing you to recognise the signal from its leader tone and supply the mode yourself.

## Fix / defense

SSTV isn't a vulnerability — it's a transmission format, so there's nothing to "patch." The takeaway is operational: triage unknown analog/RF captures by their **preamble tones and duration**, not by guessing. A 1900 Hz leader + 1200 Hz sync = SSTV; a ~110–128 s run means a 256-line mode. When a standards-based tool chokes on a malformed mode byte, fall back to forcing the mode by hand (`qsstv` / `RX-SSTV` do this interactively; a `--mode` CLI does it headlessly). Keep a tone-to-mode cheat-sheet and a forced-mode decoder in your kit.
