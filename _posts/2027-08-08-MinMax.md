## Overview

**MinMax** is a Hack The Box **Coding** challenge. Unlike a `nc` puzzle or a binary to reverse, the Coding category serves a **Flask web app** with an in-browser Monaco code editor: you write a program, the server runs it against a hidden stdin test case, and returns the flag once your output matches. The whole challenge is trivial arithmetic — print the smallest and largest number in a list — but it hides a sharp gotcha in the *output format*.

## The technique

The challenge brief:

> "Most of the numbers are garbage, but the biggest and smallest numbers in the file form co-ordinates of the group's next attack location. Identify these 2 numbers, then print out first the minimum and then the maximum."

```
Example Input : 3.29 3.09 1.34 2.89
Example Output: 1.34 3.29
```

Two pieces of recon matter:

1. **The "silent" docker port is actually HTTP.** Connecting with `nc`/pwntools shows the port open but sending no banner — it just waits, which looks like a broken or input-first protocol. A plain `curl` reveals it's a Werkzeug/Flask server hosting a Monaco editor titled *MiniMax*.

   ```bash
   curl -s http://<host>:<port>/        # Flask page + Monaco editor
   ```

2. **The submit endpoint is `POST /run`.** Reading the page's inline JavaScript shows the editor POSTs your source as JSON; the response JSON carries `input` (the test case it fed), `result` (your stdout), `stderr` (the grader verdict), and `flag` on success.

   ```javascript
   fetch("/run", { method: "POST",
     headers: { "Content-Type": "application/json" },
     body: JSON.stringify({ code: editor.getValue(), language: "python" }) })
   ```

So we never need a browser — we can submit solutions programmatically and iterate output formats in seconds.

## Solution

The logic is just `min` and `max` of a list of floats. The trap is the **output delimiter**. The brief's rendered example showed `Output 1.34 3.29`, which looks space-separated. But HTML collapses a newline into a single space, so the shown delimiter is misleading. Submitting space-separated output is numerically correct yet rejected:

```python
print(min(v), max(v))   # -> "12.33 1997.68"  →  "Code is incorrect!"
```

The grader wants each value on its **own line** — minimum first, maximum second:

Create `solve.py`:

```python
#!/usr/bin/env python3
import json, sys, urllib.request

SOLUTION = """import sys
v=[float(x) for x in sys.stdin.read().split()]
print(min(v))
print(max(v))
"""

def main():
    target = sys.argv[1] if len(sys.argv) > 1 else "127.0.0.1:1337"
    body = json.dumps({"code": SOLUTION, "language": "python"}).encode()
    req = urllib.request.Request("http://" + target + "/run", data=body,
                                 headers={"Content-Type": "application/json"})
    resp = json.loads(urllib.request.urlopen(req, timeout=15).read().decode())
    print(resp.get("flag") or ("no flag — " + repr(resp.get("result"))))

if __name__ == "__main__":
    main()
```

Run it against the live instance:

```bash
python3 solve.py <host>:<port>
HTB{...}
```

## Why it worked

The Coding harness compares your program's stdout against the grader's own answer as an exact string. The arithmetic was never the obstacle — the answer `12.33` / `1997.68` was right on the first try. The failure was purely a **format mismatch**: the grader emits its two numbers separated by a newline, and the brief's HTML-rendered example collapsed that newline into a space, so the obvious one-line `print(a, b)` produced the right values with the wrong separator.

## Fix / defense

For challenge authors: specify the exact output format unambiguously (e.g. "one value per line") rather than relying on an HTML-rendered example, where whitespace is not faithfully preserved. For solvers, two durable habits: a silent TCP port may simply be HTTP — always `curl` before assuming a custom protocol; and when a numerically-correct answer is rejected, suspect the delimiter first and try newline-separated versus space-separated output before doubting your logic. Reading all of stdin with `sys.stdin.read().split()` keeps the input parsing robust to either delimiter.
