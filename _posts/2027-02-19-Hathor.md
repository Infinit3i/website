## Overview

Hathor is an Insane-difficulty Windows Active Directory box. The route to `user.txt` is a chain of an unchanged CMS default credential, an IIS upload filter that checks extensions on upload but not on the Copy operation, an AppLocker/Constrained-Language-Mode foothold, a leaked-and-cracked NT hash, Kerberos-only authentication, and a DLL hijack against a periodically-spawned process on a writable share. This post covers recon through the user flag.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 80   | HTTP (IIS) | mojoPortal CMS, "under construction" |
| 88/389/636/3268 | Kerberos / LDAP | Active Directory — domain `WINDCORP.HTB` |
| 139/445 | SMB | shares |
| 5985 | WinRM | remote management |

The open-port spread (Kerberos + LDAP + SMB) immediately marks this as a domain controller. Nmap also gives us the hostname, so we add it to `/etc/hosts`:

```bash
nmap -p- --min-rate 1000 -T4 10.129.230.109
nmap -p80,88,389,445,5985 -sC -sV 10.129.230.109
echo "10.129.230.109 hathor.windcorp.htb windcorp.htb" | sudo tee -a /etc/hosts
```

## Enumeration

Port 80 serves an IIS site that is "under construction." Viewing the page source reveals the CMS in use is **mojoPortal**. mojoPortal is open source, and its installation ships with a well-known default administrator credential that is frequently left unchanged:

```
admin@admin.com : admin
```

Those credentials log straight into the admin panel — the install default was never rotated.

## Foothold

The most promising admin feature is the **File Manager**. Uploading an `.aspx` web shell is rejected, and the **Rename** option is filtered too. The **Copy** option, however, only validates the destination name — not the source extension. So we upload the shell with an allowed extension and copy it back to `.aspx`:

```bash
# 1. upload the ASPX web shell with a .txt extension via the File Manager (passes the upload filter)
cp insomniashell.aspx shell.txt

# 2. use the File Manager "Copy" option to copy shell.txt -> shell.aspx
#    (Copy validates the new name only, not the original extension)
```

The media directory path comes from the WindCorp logo's `img src` on the front page, so the copied shell is reachable at:

```bash
curl -s "http://hathor.windcorp.htb/Data/Sites/1/media/logos/shell.aspx"
```

Triggering the web shell with a connect-back gives a shell as the `windcorp\web` app-pool user:

```bash
rlwrap nc -lvnp 9001
```

PowerShell in this shell lands in **Constrained Language Mode**, indicating AppLocker is enforced:

```powershell
$ExecutionContext.SessionState.LanguageMode   # -> ConstrainedLanguage
Get-AppLockerPolicy -Effective -Xml
```

Enumerating the host turns up a non-default `C:\Get-bADpasswords` directory containing a signed, modified copy of the **Get-bADpasswords** auditing script with `write_hash_to_logs` enabled. Its log files leak an NT hash for the user **BeatriceMill**:

```
9cb01504ba0247ad5c6e08f7ccae7903
```

NTLM authentication is disabled on this domain (`crackmapexec` returns `STATUS_NOT_SUPPORTED`), so we crack the hash offline and then authenticate over Kerberos instead:

```bash
john --format=NT --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
# beatricemill : <redacted>
```

With a cracked password and a `krb5.conf` for `WINDCORP.HTB`, we request a ticket and reach the `share` SMB share. The share holds AutoIt scripts that load a `7-zip64.dll` — the one file on the share we can overwrite. Replacing it with a DLL built on Kali (`x86_64-w64-mingw32-gcc -shared`) gives code execution as the user **ginawild** when the scripts next run, which lands the user shell.

## User flag

```bash
type C:\Users\ginawild\Desktop\user.txt   # HTB{...}
```

Access as `ginawild` achieved — the user flag is captured.

Privilege escalation is left as an exercise — this post stops at user.
