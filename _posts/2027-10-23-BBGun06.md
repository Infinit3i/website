BBGun06 is an Easy crypto challenge modelled on an email-signing server that uses RSA with the small public exponent **e = 3**. The server hands you a signed email and dares you to produce a *different* signature that still verifies over the same sender — a forgery that, in the story, lets a threat actor impersonate the vendor's IT department in phishing mail. The verifier is hand-rolled PKCS#1 v1.5 code that **parses** the recovered block with a regular expression instead of reconstructing and comparing the full padded block. That is the textbook **Bleichenbacher 2006** mistake (hence "BB06"), an [improper verification of a cryptographic signature](https://cwe.mitre.org/data/definitions/347.html), and it lets you forge a valid signature with no private key at all.

## Overview

- **Category:** Crypto · **Difficulty:** Easy
- **Vuln class:** RSA e=3 PKCS#1 v1.5 signature forgery — [CWE-347](https://cwe.mitre.org/data/definitions/347.html)
- **One-line path:** the verifier only pattern-matches the *start* of the cube of the signature, so you build the block you want, take its integer cube root, and send that as the forged signature.

## The technique

A correct PKCS#1 v1.5 verifier rebuilds the entire expected encoded message — `00 01 FF…FF 00 ASN1 H(m)` — for the exact modulus length and compares it byte-for-byte. BBGun06 does this instead:

```python
def verify(self, message, signature):
    decrypted = pow(signature, 3, self.n)        # cube the signature
    clearsig  = decrypted.to_bytes(keylength, "big")
    r = re.compile(b'\x00\x01\xff+?\x00(.{15})(.{20})', re.DOTALL)
    m = r.match(clearsig)                         # <-- the flaw
    if not m or m.group(1) != self.asn1 or m.group(2) != sha1(message).digest():
        raise VerificationError('Verification failed')
    return True
```

Two properties make this forgeable:

1. **`re.match` anchors only at the start.** It does not require the pattern to consume the whole block — every byte after the 20-byte hash is never inspected.
2. **`\xff+?` is non-greedy.** It matches as *few* `FF` bytes as possible, so the match ends right after `…00 ASN1(15) SHA1(20)`. The low ~215 bytes are completely free.

Because the low bytes are unconstrained and `e = 3` is tiny, you can craft a block whose meaningful high bytes are exactly what the regex wants, then take the integer cube root — the cube of that root keeps the high bytes intact while only the unchecked low "garbage" wobbles. Crucially the forge depends only on `e`, the modulus byte-length, and `SHA1(message)`, so it works against the fresh keypair generated on every connection — you never even read the public key.

## Solution

The signed message is fixed — it is the `From:` line of `email.txt`: `IT Department <it@cloudcompany.com>`. Build `00 01 FF 00 || ASN1 || SHA1(user)`, zero-pad the low end to 256 bytes (2048-bit modulus), cube-root rounded up, and send the result as hex.

Create `solve.py`:

```python
import sys, socket, re
from hashlib import sha1

HOST, PORT = sys.argv[1], int(sys.argv[2])
ASN1 = b"\x30\x21\x30\x09\x06\x05\x2b\x0e\x03\x02\x1a\x05\x00\x04\x14"
USER = b"IT Department <it@cloudcompany.com>"
KEYLEN = 256

def icbrt(x):                       # integer cube root, rounded UP
    lo, hi = 0, 1 << ((x.bit_length() // 3) + 2)
    while lo < hi:
        mid = (lo + hi) // 2
        if mid ** 3 < x:
            lo = mid + 1
        else:
            hi = mid
    return lo

def forge():
    h = sha1(USER).digest()
    prefix = b"\x00\x01\xff\x00" + ASN1 + h          # one \xff satisfies \xff+?
    block = prefix + b"\x00" * (KEYLEN - len(prefix))  # garbage = zeros, low end
    sig = icbrt(int.from_bytes(block, "big"))
    return sig.to_bytes((sig.bit_length() + 7) // 8, "big")

forged = forge()
s = socket.create_connection((HOST, PORT), timeout=15)
buf = b""
while b"Enter the signature as hex:" not in buf:
    buf += s.recv(4096)
s.sendall(forged.hex().encode() + b"\n")
resp = b""
while True:
    c = s.recv(4096)
    if not c:
        break
    resp += c
print(re.search(r"HTB\{[^}]+\}", (buf + resp).decode("latin1")).group(0))
```

Run it against the live instance:

```bash
python3 solve.py <host> <port>
# HTB{...}
```

The server accepts the forged signature, confirms it differs from the real one (`forged % n != real` — trivially true, since the forged value is ~2^680 against a real signature of ~2^2040), and returns the flag.

## Why it worked

With `e = 3`, the block `00 01 FF 00 ASN1 SHA1(user) 00…00` is about `2^2040`, well below `n ≈ 2^2048`, so `pow(sig, 3, n)` never wraps the modulus — the cube is computed over the integers. Rounding the cube root **up** guarantees `sig**3 ≥ block`; the increment perturbs only the bottom ~85-byte region, which sits entirely inside the 215 garbage bytes the verifier never checks. So the high bytes the regex matches against survive exactly, and the signature verifies. No factoring, no private key, and the attack is independent of the per-connection modulus.

## Fix / defense

| Cause | Fix |
|-------|-----|
| `e = 3` makes a low-root forgery cheap | Use `e = 65537` |
| `verify` regex-parses a prefix (`re.match`, non-greedy `\xff+?`) | **Reconstruct** the full encoded message and compare the whole block in constant time |
| Trailing bytes after the hash unchecked; `FF`-run length not pinned | Anchor and full-match the entire decrypted block |
| Hand-rolled PKCS#1 v1.5 | Prefer RSASSA-PSS or a vetted library's `verify` |

The defensive verifier reconstructs and compares everything:

```python
em = b'\x00\x01' + b'\xff' * (k - 3 - len(asn1) - 20) + b'\x00' + asn1 + sha1(message).digest()
if not hmac.compare_digest(em, pow(sig, e, n).to_bytes(k, 'big')):
    raise VerificationError
```
