## Overview

`SPG` (Secure Password Generator) is a Very Easy HackTheBox **Crypto** challenge. You get two files and no network target: a `source.py` that generates a password and AES-encrypts the flag, and an `output.txt` holding the generated password plus the base64 ciphertext. The cipher itself is solid AES — the bug is that the "password generator" prints the secret key bit-by-bit, so the public password *is* the key in disguise. Recover the key from the password, derive the AES key, decrypt the flag.

## The technique

This is a [secret-disclosure-through-deterministic-encoding](https://cwe.mitre.org/data/definitions/203.html) ([CWE-203](https://cwe.mitre.org/data/definitions/203.html)) bug. The generator walks the master key one bit at a time and emits one password character per bit, where the character itself reveals the bit:

```python
ALPHABET = string.ascii_letters + string.digits + '~!@#$%^&*'   # 70 chars

def generate_password():
    master_key = int.from_bytes(MASTER_KEY, 'little')
    password = ''
    while master_key:
        bit = master_key & 1
        if bit:
            password += random.choice(ALPHABET[:len(ALPHABET)//2])   # first half  -> bit 1
        else:
            password += random.choice(ALPHABET[len(ALPHABET)//2:])   # second half -> bit 0
        master_key >>= 1
    return password

def main():
    password = generate_password()
    encryption_key = sha256(MASTER_KEY).digest()
    cipher = AES.new(encryption_key, AES.MODE_ECB)
    ciphertext = cipher.encrypt(pad(FLAG, 16))
```

The random choice *within* a half is irrelevant — only **which half** matters. So password character `i` discloses bit `i` of `MASTER_KEY` (least-significant bit first). And `MASTER_KEY` is exactly what derives the AES key via `sha256(MASTER_KEY)`. The secret never had to be brute-forced — it was printed.

## Solution

Map each password character back to its bit, rebuild the integer LSB-first, convert it back to little-endian bytes (inverting `int.from_bytes(..., 'little')`), derive the key, and decrypt.

Create `solve.py`:

```python
from hashlib import sha256
import string
from base64 import b64decode
from Crypto.Cipher import AES
from Crypto.Util.Padding import unpad

ALPHABET = string.ascii_letters + string.digits + '~!@#$%^&*'
FIRST_HALF = set(ALPHABET[:len(ALPHABET)//2])   # char from here => bit 1

PASSWORD = "<password from output.txt>"
CT = b64decode("<base64 ciphertext from output.txt>")

master_key = 0
for i, ch in enumerate(PASSWORD):
    if ch in FIRST_HALF:
        master_key |= 1 << i

nbytes = (master_key.bit_length() + 7) // 8
MASTER_KEY = master_key.to_bytes(nbytes, 'little')

key = sha256(MASTER_KEY).digest()
pt = unpad(AES.new(key, AES.MODE_ECB).decrypt(CT), 16)
print("MASTER_KEY =", MASTER_KEY)
print("FLAG =", pt.decode())
```

Run it:

```bash
python3 solve.py
```

The recovered `MASTER_KEY` comes out as a readable ASCII string — a free sanity check that the bit order and endianness are right (a wrong choice would produce garbage bytes). The flag prints:

```
FLAG = HTB{...}
```

## Why it worked

Any time a public artifact (a password, token, serial, or nonce) is a **1:1 deterministic function of a secret**, every output symbol becomes a read oracle for that secret. Here the mapping was "charset-half = one key bit", but the same flaw shows up as case-of-letter (upper=1/lower=0), parity-of-index, or even/odd-of-byte. Once you spot that the output is a reversible encoding of the key, you just invert it — no cryptanalysis of the cipher required, because the cipher was never the weak link.

## Fix / defense

Never derive a user-facing value from key material with a recoverable mapping. Generate passwords and tokens from a CSPRNG, fully independent of any secret, and derive cipher keys from a high-entropy random source ([CWE-330](https://cwe.mitre.org/data/definitions/330.html)):

```python
import secrets
password = ''.join(secrets.choice(ALPHABET) for _ in range(24))   # independent of the key
encryption_key = secrets.token_bytes(32)                          # CSPRNG, not present in any output
```

If a value genuinely must be both displayed and bound to a secret, commit to it with a salted one-way KDF, not a per-bit encoding.
