## Overview

QuickR is an Easy HackTheBox Misc challenge served over a socket (`nc <host> <port>`).
On connect it greets you — *"you got only 3 seconds!"* — then prints a **QR code drawn
as ANSI terminal art** and waits for a reply. It loops for several rounds; clear them all
and it hands over the flag. The whole thing is a speed test that's trivial to automate once
you spot the two tricks.

## The technique

**1 — the QR is reverse-video text, not an image.** Each QR module is a pair of characters:
a reverse-video cell `\x1b[7m  \x1b[0m` for one polarity, and a plain two-space `  ` for the
other. So you can't feed a PNG to a decoder — you rebuild the 0/1 module matrix straight from
the escape sequences, then hand *that* to a QR decoder.

**2 — the decoded text is a math problem, and it wants the machine value.** The QR doesn't
contain the answer; it contains an expression like `44.6704… x 124.3167… + 285.3189… =`
(`x` means multiply). The banner hint — *"you simply need the true machine value"* — means
compute it in **IEEE-754 double precision** and send the full-precision result, not a rounded
human approximation.

## Solution

Reconstruct each QR row from the escape cells, map it to a pixel grid, upscale it, add a white
quiet-zone border, and decode with OpenCV's built-in `QRCodeDetector` (Kali ships `cv2` but not
`pyzbar`/`zbarimg`). Try **both** polarities — reverse-video may be the light quiet-zone rather
than the dark module. Then `eval` the decoded expression (after `x`→`*`) and reply `repr(result)`.
A round comfortably fits in the 3-second window because `cv2` decodes in well under 100 ms.

Create `solve.py`:

```python
#!/usr/bin/env python3
import socket, re, sys, time
import numpy as np, cv2

HOST, PORT = sys.argv[1], int(sys.argv[2])
ANSI = re.compile(rb'\x1b\[[0-9;]*m')

def recv_until_idle(s, idle=0.5, total=4.0):
    s.setblocking(False); buf=b""; t0=time.time(); last=time.time()
    while time.time()-t0 < total:
        try:
            ch=s.recv(65536)
            if ch: buf+=ch; last=time.time()
            else: break
        except BlockingIOError:
            if buf and time.time()-last>idle: break
            time.sleep(0.02)
    return buf

def parse_qr(raw):
    text=raw.decode('latin1')
    lines=[ln for ln in text.split('\n') if '\t' in ln and ('\x1b[7m' in ln or ln.strip(' \t')=='')]
    rows=[]
    for ln in lines:
        ln=ln.split('\t',1)[1] if '\t' in ln else ln
        tmp=ln.replace('\x1b[7m  \x1b[0m','#')
        tmp=ANSI.sub(b'', tmp.encode('latin1')).decode('latin1')
        cells=[]; i=0
        while i < len(tmp):
            if tmp[i]=='#': cells.append(1); i+=1
            elif tmp[i:i+2]=='  ': cells.append(0); i+=2
            else: i+=1
        if cells: rows.append(cells)
    if not rows: return None
    w=max(len(r) for r in rows)
    rows=[r for r in rows if len(r)==w]
    return np.array(rows,dtype=np.uint8) if len(rows)>=10 else None

def decode(mat):
    det=cv2.QRCodeDetector()
    for inv in (0,1):
        base = mat if not inv else (1-mat)
        px = np.where(base==1, 0, 255).astype(np.uint8)
        img=np.kron(px,np.ones((10,10),np.uint8))
        img=cv2.copyMakeBorder(img,60,60,60,60,cv2.BORDER_CONSTANT,value=255)
        v,_,_=det.detectAndDecode(img)
        if v: return v
    return None

s=socket.socket(); s.connect((HOST,PORT)); s.settimeout(5); rounds=0
while True:
    raw=recv_until_idle(s)
    if not raw: break
    clean=ANSI.sub(b'', raw)
    if b'HTB{' in clean:
        print("FLAG:", re.search(rb'HTB\{[^}]+\}', clean).group().decode()); break
    mat=parse_qr(raw)
    if mat is None: continue
    val=decode(mat)
    if val is None: break
    expr=val.strip().rstrip('=').strip().replace('x','*').replace('X','*')
    out=repr(eval(expr,{"__builtins__":{}},{}))
    s.sendall(out.encode()+b'\n'); rounds+=1
s.close()
```

Run it against the instance:

```bash
python3 solve.py <host> <port>
```

After a couple of rounds the server prints the flag (`HTB{...}`).

## Why it worked

The challenge bets you can't read and answer a QR in three seconds by hand — but a QR rendered as
terminal art is just a 0/1 grid in disguise, and floating-point arithmetic has exactly one correct
answer the server is comparing against. Rebuild the grid, decode it, reproduce the exact IEEE-754
computation, and the timer is irrelevant.

## Fix / defense

This is a CTF speed puzzle rather than a vulnerability. The operational lesson generalizes: when a
service encodes data in a presentation format (ANSI art, ASCII tables, images), treat it as a
parseable structure rather than something only a human can read; and when it asks for a computed
value, reproduce the exact arithmetic semantics (here, IEEE-754 doubles) instead of a rounded guess.
