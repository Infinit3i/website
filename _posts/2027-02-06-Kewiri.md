## Overview

Kewiri is a Very Easy HackTheBox **Crypto** challenge. You connect to a container
that runs a timed "Grand Archives of Eldoria" quiz: six number-theory questions
over one **fixed** 384-bit prime `p`. Answer all six and the flag drops. A wrong
or *slow* answer kills the connection — so the heavy computation is done offline
and the cached answers are replayed. The final question is an elliptic-curve
discrete log, and the whole challenge hinges on noticing the curve is **anomalous**
(`#E(F_p) == p`), which makes the [ECDLP](https://cwe.mitre.org/data/definitions/327.html)
solvable in linear time via **Smart's attack**.

## The technique

The six trials, and how each is answered:

| Q | Asked | Answer |
|---|-------|--------|
| 1 | bit length of `p` | `p.bit_length()` = 384 |
| 2 | factor the order of `F_p*` = `p-1` (format `p0,e0_p1,e1_...`) | `sympy.factorint(p-1)` — ~20 s, so **precompute offline** |
| 3 | is each element a generator of `F_p*`? (`1`/`0`) | `g^((p-1)/q) != 1` for every prime `q | p-1` |
| 4 | order of `E: y² = x³ + ax + b` over `F_p` | PARI `ellcard` → **== p (anomalous!)** |
| 5 | factor `#E(F_{p³})` | trace `t = 1` ⇒ `#E(F_{p³}) = p³ + 3p = p·(p²+3)` |
| 6 | ECDLP: find `d` with `A = d·G` (only x-coords given) | **Smart's attack** |

The key observation is in Q4: `ellcard` returns exactly `p`. That means the trace
of Frobenius is `t = p + 1 - #E = 1`, i.e. the curve is **anomalous**. Two things
follow:

- **Extension-field order (Q5)** is free from the trace. With `α + β = t = 1` and
  `αβ = p`, Newton's identity gives `s_n = α^n + β^n` via `s₀=2, s₁=t,
  s_n = t·s_{n-1} - p·s_{n-2}`, and `#E(F_{p^n}) = p^n + 1 - s_n`. For `n = 3`:
  `#E(F_{p³}) = p³ + 3p = p·(p²+3)` (factor `p²+3` with factordb → `2²·7²·P764`).
- **The ECDLP (Q6) collapses.** On an anomalous curve the reduction map
  `E(Q_p) → E(F_p)` has a formal-group kernel isomorphic to the *additive* group
  `(F_p, +)`. Lift the points into the p-adics, push them into that kernel by
  multiplying by `p`, and read the formal logarithm `t = -x/y`. The discrete log
  is just the ratio of two formal logs mod `p` — **Smart's attack** (SSSA).

## Solution

`p` is constant across instances, so the expensive answers (Q2/Q5 factorizations)
are computed once offline and replayed instantly to beat the per-question timer.
The interesting part is Smart's attack on Q6, done with **PARI via `cypari2`** — no
Sage required (the wheel bundles libpari):

```python
import cypari2
pari = cypari2.Pari()

# E: y^2 = x^3 + a*x + b over F_p, with #E(F_p) == p (anomalous)
def smart_attack(xG, xA, prec=5):
    pad  = lambda v: pari(v) + pari('O(%d^%d)' % (p, prec))   # lift to Q_p
    Ep   = pari.ellinit([pad(a), pad(b)])
    lift = lambda x: [pad(x), pari.ellordinate(Ep, pad(x))[0]]
    pG   = pari.ellmul(Ep, lift(xG), p)                       # p*G -> formal group
    pA   = pari.ellmul(Ep, lift(xA), p)
    d    = (-pA[0]/pA[1]) / (-pG[0]/pG[1])                    # ratio of formal logs
    return int(pari.lift(d + pari('O(%d^1)' % p))) % p
```

The full quiz bot answers Q1–Q5 from cached values, computes the live curve order
for Q4 with `ellcard`, then solves Q6 with `smart_attack`:

```python
import socket, re
import cypari2
pari = cypari2.Pari()

# fixed challenge prime + curve; ANS2/ANS5 are the precomputed factorizations
FPRIMES = [int(x.split(',')[0]) for x in ANS2.split('_')]
is_gen  = lambda g: 0 if any(pow(g, (p-1)//q, p) == 1 for q in FPRIMES) else 1

c.send(p.bit_length())                       # Q1
c.send(ANS2)                                 # Q2  (cached factorization of p-1)
# Q3: answer the generator series until the curve question appears, then Q4:
E = pari.ellinit([aa, bb], p); c.send(int(pari.ellcard(E)))   # Q4 == p
c.send(ANS5)                                 # Q5  (cached factorization of p^3+3p)
c.send(smart_attack(xG, xA))                 # Q6  ECDLP via Smart's attack
```

Only the **x-coordinates** of `G` and `A` are published, so the recovered scalar is

## Why it worked

The challenge author chose an **anomalous** curve — one whose group order over
`F_p` equals the field characteristic exactly (`#E(F_p) == p`). That single property
breaks the hardness assumption the ECDLP relies on: instead of a hard discrete log
in a generic prime-order group, the curve admits an *additive transfer* to `(F_p, +)`
where "discrete log" is just division. Smart's attack exploits this in linear time,
no index calculus, no smooth-order requirement — the tell is simply `ellcard(E) == p`.

## Fix / defense

Never deploy an anomalous curve. At key-generation time, reject any curve where
`#E(F_p) == p` (equivalently `trace != 1`), and prefer a vetted, standardized
prime-order curve whose order differs from the field prime:

```python
from cryptography.hazmat.primitives.asymmetric import ec
priv = ec.generate_private_key(ec.SECP256R1())   # #E != field prime
```

Generating ad-hoc curve parameters without checking the order against the field size
is a [use of a broken/risky cryptographic primitive](https://cwe.mitre.org/data/definitions/327.html);
standard curves (secp256r1, Curve25519) are vetted precisely to avoid anomalous,
low-embedding-degree, and small-subgroup pitfalls.
