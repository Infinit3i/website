## Overview

**Art** is an Easy HackTheBox **Misc** challenge. The download is one small (5.4 KB)
`300x300` PNG full of flat, brightly colored square blocks arranged as concentric rings —
the brief is just *"Can you find the flag?"*. The trap is to reach for steganography tools;
the real answer is that the image **is source code** written in the [Piet](https://www.dangermouse.net/esoteric/piet.html)
esoteric programming language. Running the image through a Piet interpreter prints the flag.

## The technique

Piet is a 2-D esoteric language designed so that programs look like abstract art. Code is
made of **codels** (square blocks of a single color) drawn from a fixed 20-color palette
(6 hues × 3 lightnesses, plus black and white). Operations are encoded by the *change* in
hue and lightness as the program's "direction pointer" walks from one color block to the
next; characters are emitted with Piet's `out` command. So the entire flag is simply the
program's standard output.

The tell that this is Piet rather than stego:

- The file is **too small** (5.4 KB) for meaningful LSB-hidden data, and `zsteg`/`steghide`
  find nothing.
- The palette is the **constrained, flat 20-color Piet set** laid out in clean geometric blocks.

## Solution

First, recover the **codel size** — the edge length in pixels of one logical block. It is the
minimum run-length of identical pixels along the top row / left column:

```python
from PIL import Image
im = Image.open("art.png").convert("RGB"); w, h = im.size; px = im.load()
def minrun(line):
    c, n, runs = line[0], 1, []
    for v in line[1:]:
        if v == c: n += 1
        else: runs.append(n); c, n = v, 1
    runs.append(n); return min(runs)
print(minrun([px[x, 0] for x in range(w)]))   # -> 10
```

Then compile-and-execute the image with the pure-python **repiet** interpreter. The durable
solver:

```python
#!/usr/bin/env python3
# Art (HTB Misc) — the PNG IS a Piet esolang program.
# Concentric colored codel blocks = a Piet image; executing it prints the flag.
import subprocess, os, re
HERE = os.path.dirname(os.path.abspath(__file__))
out = subprocess.run(
    [os.path.join(HERE, "venv/bin/repiet"), "--codel_size", "10", "-x",
     os.path.join(HERE, "files/art.png")],
    input=b"", capture_output=True).stdout.decode(errors="replace")
m = re.search(r"HTB\{[^}]*\}", out)
print(m.group(0) if m else out)
```

Run it:

```bash
python3 -m venv venv && ./venv/bin/pip install repiet numpy pillow
./venv/bin/repiet --codel_size 10 -x art.png </dev/null
# -> HTB{...}   ("piet mondrian")
```

The interpreter prints the flag, then hits an `input()`/`EOFError` — harmless, the flag has
already been emitted (feeding `</dev/null` keeps the output clean).

## Why it worked

Nothing was *exploited* — the challenge is pure recognition. Once you identify the Piet
palette and grid, the image is just a program and the flag is its output. The misdirection
is entirely in the framing: a colorful PNG strongly suggests image steganography, so most of
the difficulty is realizing you should be looking at esoteric languages instead.

## Fix / defense

Not applicable — this is a recognition CTF puzzle, not a vulnerability. The transferable
lesson: when an image presented as "stego" is too small to hide real data and uses a small,
flat, structured palette, check whether the image is itself a program (Piet or another 2-D
esolang) before grinding steganography tooling.
