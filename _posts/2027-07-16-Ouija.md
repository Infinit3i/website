## Overview

Ouija is an HTB Reversing challenge (Easy) featuring a 64-bit ELF binary that stores a [ROT-18](https://cwe.mitre.org/data/definitions/261.html) encoded flag on the stack and deliberately wastes analyst time by inserting a one-second `sleep()` call between every output character. The solve: patch `sleep()` to a noop via `LD_PRELOAD`, run the binary at full speed, and join the single-character output lines to read the flag — or decode it statically from the `objdump` disassembly in one Python line.

---

## The technique

Two layered "anti-analysis" mitigations, both trivially bypassed:

1. **[ROT-18 encoding (CWE-261)](https://cwe.mitre.org/data/definitions/261.html)** — the flag bytes are stored as hardcoded `movb` constants on the stack, shifted by 18 positions in the alphabet. The binary even announces its own key: *"Yes, 18 will do nicely."* ROT-N is a 1-to-1 substitution with no key material — inverting it is a single Python expression.

2. **`sleep()` timing delay ([CWE-656](https://cwe.mitre.org/data/definitions/656.html))** — `sleep(1)` is called before every character print, turning a 31-character output into over 30 seconds of waiting. Because `sleep()` is resolved from `libc` at runtime through the PLT, a tiny `LD_PRELOAD` shared library that overrides it to `return 0` neutralises the delay entirely.

## Solution

### Static path — read the encoded bytes, decode in Python

`objdump -d ouija` shows the flag as 31 `movb` constants initialised on the stack in `main()`. The encoded form is `ZLT{Kdwhafy_ak_fgl_gtxmkuslagf}`. Apply ROT-8 (the inverse of ROT-18) to each alpha character:

```python
enc = "ZLT{Kdwhafy_ak_fgl_gtxmkuslagf}"
n   = 18
flag = "".join(
    chr((ord(c) - 65 + (26 - n)) % 26 + 65) if c.isupper() else
    (chr((ord(c) - 97 + (26 - n)) % 26 + 97) if c.islower() else c)
    for c in enc
)
print(flag)   # HTB{...}
```

### Dynamic path — LD_PRELOAD noop shim

Build a two-function shared library that makes `sleep()` and `nanosleep()` return 0 instantly:

```bash
printf 'unsigned int sleep(unsigned int s){return 0;}\nint nanosleep(const void*r,void*m){return 0;}\n' \
  > ns.c && gcc -shared -fPIC -o ns.so ns.c
LD_PRELOAD=$PWD/ns.so ./ouija | grep -E '^.$' | tr -d '\n' && echo
```

The binary runs at full CPU speed; single-character output lines are the flag characters — `grep` filters them and `tr` joins them.

### Full solve.py

Create `solve.py` next to the binary:

```python
#!/usr/bin/env python3
import subprocess, tempfile, os, re

BINARY = os.path.join(os.path.dirname(__file__), "files/rev_ouija/ouija")

NOSLEEP_SRC = """
#include <time.h>
unsigned int sleep(unsigned int s) { return 0; }
int nanosleep(const struct timespec *req, struct timespec *rem) { return 0; }
"""

def build_nosleep(path):
    src = path + ".c"
    so  = path + ".so"
    with open(src, "w") as f:
        f.write(NOSLEEP_SRC)
    subprocess.run(["gcc", "-shared", "-fPIC", "-o", so, src], check=True)
    return so

def run_binary(binary, nosleep_so):
    result = subprocess.run(
        [binary],
        env={**os.environ, "LD_PRELOAD": nosleep_so},
        capture_output=True, text=True, timeout=30
    )
    return result.stdout

def extract_flag(output):
    chars = [line for line in output.splitlines() if len(line.strip()) == 1]
    flag  = "".join(chars).strip()
    m = re.search(r'HTB\{[^}]+\}', flag)
    return m.group(0) if m else flag

if __name__ == "__main__":
    with tempfile.TemporaryDirectory() as tmp:
        nosleep_so = build_nosleep(os.path.join(tmp, "nosleep"))
        output = run_binary(BINARY, nosleep_so)
        flag   = extract_flag(output)
        print(flag)
```

```bash
python3 solve.py
# HTB{...}
```

## Why it worked

`sleep()` is a `libc` symbol resolved at dynamic-link time. `LD_PRELOAD` inserts a custom shared library before `libc` in the link order, so the loader resolves `sleep` to the attacker's stub instead. This works on any unprivileged dynamically linked ELF — SUID binaries are the one exception (the kernel strips `LD_PRELOAD` across privilege boundaries). The ROT encoding provided no additional protection: the shift constant was printed to stdout by the binary itself, and all constants are visible in the disassembly as plain stack-initialised bytes.

---

## Fix / defense

- **Never store a secret in a binary.** A decompiler or `strings` recovers any hardcoded value regardless of encoding — the binary is fully readable to any analyst.
- **ROT-N / Caesar ciphers are not encryption.** They are keyless 1-to-1 substitutions; any analyst inverts them in one line. Use a one-way hash (bcrypt/Argon2) if a local credential check is truly unavoidable.
- **`sleep()` anti-analysis is bypassed in seconds** via `LD_PRELOAD`, `strace`, or direct binary patching. It imposes a cost on novice analysts and automated pipelines but nothing more. Kernel-mode or hardware attestation is required if timing guarantees actually matter.
